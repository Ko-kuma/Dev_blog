## 보는 방법

- 표는 **실무 사용 빈도 높은 순**으로 정렬
- 빈도: 상 = 거의 매일 씀 / 중 = 자주 씀 / 하 = 알아두면 좋음
- 위치: 해당 태그/속성이 실제 코드에서 들어가는 곳

# 1. HTML 태그

## 문서 구조 태그

| 태그 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `<!DOCTYPE html>` | 상 | 파일 맨 첫 줄 | HTML5 문서임을 선언. 빠지면 브라우저가 옛날 방식으로 해석 |
| `<html>` | 상 | DOCTYPE 바로 아래, 문서 전체를 감쌈 | 최상위 태그. `lang="ko"` 속성으로 언어 지정 |
| `<head>` | 상 | `<html>` 안, `<body>` 위 | 화면에 보이지 않는 메타 정보 영역 |
| `<meta charset>` | 상 | `<head>` 안 첫 번째 줄 | `<meta charset="UTF-8">` — 한글 깨짐 방지. 필수 |
| `<meta viewport>` | 상 | `<head>` 안 | `<meta name="viewport" content="width=device-width, initial-scale=1.0">` — 모바일 반응형 필수 |
| `<title>` | 상 | `<head>` 안 | 브라우저 탭에 표시되는 제목. SEO에도 영향 |
| `<link>` | 상 | `<head>` 안 | 외부 파일 연결. `rel="stylesheet" href="style.css"` — CSS 연결 |
| `<script>` | 상 | `<head>` 안 (defer 사용) 또는 `</body>` 바로 위 | JS 파일 연결. `src="app.js" defer` — defer 속성 권장 |
| `<body>` | 상 | `<html>` 안, `<head>` 아래 | 화면에 실제로 보이는 콘텐츠 전부 |

---

## 시맨틱 레이아웃 태그

의미 없는 `<div>` 대신 **의미 있는 태그**를 우선 사용. SEO + 접근성에 유리.

| 태그 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `<header>` | 상 | `<body>` 안 최상단 | 페이지 또는 섹션의 상단 영역. 로고, 내비게이션 포함 |
| `<nav>` | 상 | `<header>` 안 | 내비게이션 링크 모음. 메뉴, 페이지 이동 링크 |
| `<main>` | 상 | `<body>` 안, `<header>` 아래 | 핵심 콘텐츠 영역. 한 페이지에 하나만 |
| `<section>` | 상 | `<main>` 안 | 주제별 구획. `<h>` 태그와 함께 사용 |
| `<footer>` | 상 | `<body>` 안 최하단 | 페이지 하단. 저작권, 연락처, 사이트맵 |
| `<div>` | 상 | `<body>` 안 어디든 | 의미 없는 블록 컨테이너. 시맨틱 태그가 없을 때 레이아웃용 |
| `<article>` | 중 | `<main>` 또는 `<section>` 안 | 독립적으로 배포 가능한 콘텐츠. 블로그 글, 뉴스 기사 |
| `<aside>` | 중 | `<main>` 옆 또는 `<section>` 안 | 부가 콘텐츠. 사이드바, 광고, 관련 글 |

## 텍스트 태그

| 태그 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `<h1>` ~ `<h6>` | 상 | `<section>`, `<article>`, `<main>` 안 | 제목 태그. h1이 가장 크고 h6이 가장 작음. h1은 페이지에 하나만, h2~h6는 계층 구조로 사용 |
| `<p>` | 상 | `<section>`, `<article>`, `<div>` 안 | 문단(단락). 텍스트 본문에 사용 |
| `<span>` | 상 | 텍스트 안 인라인 위치 어디든 | 인라인 컨테이너. 텍스트 일부에만 스타일 적용할 때 |
| `<strong>` | 중 | `<p>`, `<h>`, `<li>` 등 텍스트 안 | 굵게 + 의미상 중요 강조. 스크린 리더가 강조로 읽음 |
| `<em>` | 중 | `<p>`, `<h>`, `<li>` 등 텍스트 안 | 기울임 + 의미 강조. `<i>`는 시각적 기울임만, `<em>`은 의미 포함 |
| `<br>` | 중 | 텍스트 안 줄바꿈이 필요한 곳 | 줄바꿈. 닫는 태그 없음. 남용 금지 — 여백은 CSS margin으로 |
| `<hr>` | 하 | 섹션과 섹션 사이 | 수평 구분선. 주제 전환 시 사용 |

## 링크 / 이미지 태그

| 태그 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `<a>` | 상 | `<nav>`, `<p>`, `<li>`, `<button>` 대신 | 링크. `href="#섹션id"` — 페이지 내 이동, `href="https://..."` — 외부 링크, `target="_blank"` — 새 탭 |
| `<img>` | 상 | `<section>`, `<div>`, `<figure>` 안 | 이미지. `src="경로"` 필수, `alt="설명"` 접근성 필수. 닫는 태그 없음 |
| `<figure>` | 중 | `<section>`, `<article>` 안 이미지/코드 블록 감쌀 때 | 이미지, 코드, 차트 등 독립 콘텐츠 래퍼. `<figcaption>`과 함께 씀 |
| `<figcaption>` | 중 | `<figure>` 안 | `<figure>`의 설명 캡션 |
| `<video>` | 중 | `<section>`, `<div>` 안 | 영상. `src="경로"` 또는 `<source>` 자식 태그 사용. `controls` 속성으로 재생 컨트롤 표시 |
| `<audio>` | 하 | `<section>`, `<div>` 안 | 오디오. `controls` 속성으로 재생 컨트롤 표시 |

## 목록 태그

| 태그 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `<ul>` | 상 | `<nav>`, `<section>`, `<div>` 안 | 순서 없는 목록. 기본으로 점(•)이 붙음. `nav ul`처럼 메뉴에 자주 사용 |
| `<ol>` | 중 | `<section>`, `<article>` 안 | 순서 있는 목록. 기본으로 숫자(1, 2, 3...)가 붙음 |
| `<li>` | 상 | `<ul>` 또는 `<ol>` 안에서만 | 목록 항목. `<ul>`이나 `<ol>` 밖에서는 사용 불가 |

## 폼 태그

| 태그 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `<form>` | 상 | `<section>`, `<div>`, `<main>` 안 | 폼 양식 컨테이너. `action="경로"`, `method="POST"` |
| `<input>` | 상 | `<form>` 안 또는 독립적으로 | 입력 필드. 닫는 태그 없음. type 속성으로 종류 변경 |
| `<button>` | 상 | `<form>` 안, `<nav>`, `<div>` 안 어디든 | 버튼. `type="submit"` — 폼 제출, `type="button"` — JS 연동용 |
| `<label>` | 중 | `<form>` 안, `<input>` 위 또는 옆 | 입력 필드 설명 라벨. `for="input의 id"`로 연결 |
| `<textarea>` | 중 | `<form>` 안 | 여러 줄 텍스트 입력창 |
| `<select>` | 중 | `<form>` 안 | 드롭다운 선택 목록 |
| `<option>` | 중 | `<select>` 안에서만 | 드롭다운 각 선택지 |

### input type 종류

| type | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `type="text"` | 상 | `<form>` 안 `<input>` 속성 | 일반 텍스트 입력 |
| `type="email"` | 상 | `<form>` 안 `<input>` 속성 | 이메일 형식 검사 자동 포함 |
| `type="password"` | 상 | `<form>` 안 `<input>` 속성 | 입력값을 •••로 숨김 |
| `type="submit"` | 상 | `<form>` 안 `<input>` 속성 | 폼 제출 버튼 |
| `type="checkbox"` | 중 | `<form>` 안 `<input>` 속성 | 체크박스 |
| `type="radio"` | 중 | `<form>` 안 `<input>` 속성 | 라디오 버튼 (하나만 선택) |
| `type="number"` | 중 | `<form>` 안 `<input>` 속성 | 숫자 입력 |
| `type="file"` | 중 | `<form>` 안 `<input>` 속성 | 파일 업로드 |
| `type="hidden"` | 중 | `<form>` 안 `<input>` 속성 | 화면에 보이지 않는 입력값 |

## 자주 쓰는 HTML 속성

| 속성 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `id` | 상 | 모든 태그의 여는 태그 안 | 고유 식별자. 페이지에 하나만. CSS `#id`, JS `querySelector("#id")` |
| `class` | 상 | 모든 태그의 여는 태그 안 | 스타일 클래스. 여러 개 가능. CSS `.class`, JS `querySelector(".class")` |
| `href` | 상 | `<a>` 여는 태그 안 | 이동할 URL. `#섹션id`로 페이지 내 이동 |
| `src` | 상 | `<img>`, `<script>` 여는 태그 안 | 파일 경로 |
| `alt` | 상 | `<img>` 여는 태그 안 | 이미지 설명. 접근성 필수. 이미지 로드 실패 시 대체 텍스트 |
| `type` | 상 | `<input>`, `<button>` 여는 태그 안 | 종류 지정 |
| `placeholder` | 상 | `<input>`, `<textarea>` 여는 태그 안 | 입력 전 안내 텍스트 |
| `for` | 중 | `<label>` 여는 태그 안 | 연결할 `<input>`의 id값. 라벨 클릭 시 해당 input 포커스 |
| `disabled` | 중 | `<input>`, `<button>` 여는 태그 안 | 비활성화. 클릭/입력 불가 |
| `rel` | 상 | `<link>` 여는 태그 안 | 파일 관계 명시. `rel="stylesheet"` |
| `defer` | 상 | `<script>` 여는 태그 안 | HTML 파싱 완료 후 JS 실행 |
| `target="_blank"` | 상 | `<a>` 여는 태그 안 | 새 탭에서 링크 열기 |
| `colspan` | 중 | `<td>`, `<th>` 여는 태그 안 | 열 병합 |
| `rowspan` | 중 | `<td>`, `<th>` 여는 태그 안 | 행 병합 |

# 2. CSS 속성

## 텍스트 / 폰트

| 속성 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `color` | 상 | 텍스트가 있는 모든 선택자 | 글자 색상. `#333`, `rgba(0,0,0,0.5)` |
| `font-size` | 상 | `body`, `h`, `p`, `span` 등 | 글자 크기. 실무에선 `rem` 권장 |
| `font-weight` | 상 | `h`, `strong`, `button`, `label` 등 | 글자 굵기. `bold` 또는 숫자 `400`(기본), `700`(굵게) |
| `font-family` | 상 | `body` (전역 설정) | 글꼴. `body`에 한 번 설정하면 전체 적용 |
| `line-height` | 상 | `body`, `p`, `li` 등 | 줄 간격. 숫자 배수로 지정. `1.6`이 가독성 좋은 기본값 |
| `text-align` | 상 | `h`, `p`, `div`, `footer` 등 | 텍스트 정렬. `left`(기본) / `center` / `right` |
| `text-decoration` | 상 | `a` 태그 선택자 | 텍스트 장식. `none` — 링크 기본 밑줄 제거 |
| `letter-spacing` | 중 | `h`, 강조 텍스트, 버튼 | 글자 사이 간격 |
| `text-transform` | 하 | 영문 텍스트 선택자 | 대소문자 변환. `uppercase` / `lowercase` |
| `white-space` | 중 | 한 줄 고정이 필요한 요소 | `nowrap` — 줄바꿈 금지 |

---

## 색상 / 배경

| 속성 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `background-color` | 상 | `nav`, `header`, `section`, `footer`, `button` 등 | 배경 색상 |
| `opacity` | 상 | 호버 효과, 오버레이, 비활성 요소 | 투명도. `0`(완전 투명) ~ `1`(완전 불투명). 자식도 함께 투명해짐 |
| `background-image` | 중 | 히어로 섹션, 배너 등 | 배경 이미지. `url("경로")` |
| `background-size` | 중 | `background-image`가 있는 선택자 | `cover` — 영역 꽉 채움, `contain` — 비율 유지 |
| `background-position` | 중 | `background-image`가 있는 선택자 | 배경 이미지 위치. `center center` |
| `background-repeat` | 중 | `background-image`가 있는 선택자 | 반복 여부. `no-repeat` 자주 씀 |

## 박스 모델

| 속성 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `box-sizing: border-box` | 상 | `*` 전역 리셋 — CSS 파일 맨 위 | width에 padding, border 포함. 실무 표준. 항상 전역 설정 |
| `width` / `height` | 상 | 레이아웃 요소, 이미지, 버튼 등 | 너비 / 높이. `px`, `%`, `vw`, `vh` |
| `max-width` | 상 | `section`, `.container` 등 콘텐츠 래퍼 | 최대 너비. `max-width: 800px; margin: 0 auto` — 가운데 정렬 조합 |
| `margin` | 상 | 거의 모든 블록 요소 | 바깥 여백. `margin: 0 auto` — 가운데 정렬. `margin: 위 오른쪽 아래 왼쪽` |
| `padding` | 상 | `section`, `button`, `nav`, `div` 등 | 안쪽 여백. margin과 같은 방식으로 지정 |
| `border` | 상 | `input`, `button`, `card` 등 | 테두리. `border: 1px solid #ccc` — 두께 스타일 색상 순 |
| `border-radius` | 상 | `button`, `input`, `img`, 카드 등 | 모서리 둥글게. `8px` 또는 `50%` — 원형 |
| `box-shadow` | 상 | `nav.scrolled`, 카드, 팝업, 버튼 호버 | 그림자. `0 2px 8px rgba(0,0,0,0.2)` — x이동 y이동 흐림 색상 |
| `overflow` | 상 | 콘텐츠가 넘칠 수 있는 컨테이너 | `hidden` — 잘라냄, `scroll` — 스크롤바, `auto` — 자동 |
| `min-width` / `min-height` | 중 | 버튼, 아이콘, 모달 등 | 최소 너비/높이 |
| `aspect-ratio` | 중 | 이미지, 영상, 썸네일 컨테이너 | 비율 고정. `aspect-ratio: 16 / 9` — 16:9 비율 유지 |
| `word-break` | 중 | 긴 텍스트가 들어오는 카드, 말풍선 | 단어 줄바꿈 방식. `break-all` — 글자 단위 강제 줄바꿈 |
| `pointer-events` | 중 | 클릭 막아야 하는 오버레이, 비활성 요소 | `none` — 마우스 이벤트 전부 무시 |
| `user-select` | 중 | 버튼, 아이콘, 드래그 방지 요소 | `none` — 텍스트 드래그 선택 방지 |

## 레이아웃 (display / position)

| 속성 | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `display: block` | 상 | `<div>`, `<p>`, `<h>` 기본값이나 인라인 요소에 블록 적용 시 | 한 줄 전체 차지 |
| `display: inline-block` | 상 | 나란히 배치하면서 width/height도 필요한 요소 | 인라인처럼 나란히 + 블록처럼 크기 적용 |
| `display: flex` | 상 | `nav`, `header`, 카드 목록, 버튼 그룹 등 레이아웃 컨테이너 | Flexbox 활성화. 자식이 가로 정렬됨 |
| `display: grid` | 상 | 전체 페이지 레이아웃, 갤러리, 카드 그리드 | Grid 활성화. 2차원 배치 |
| `display: none` | 상 | JS로 숨기고 보여주는 요소, 모바일에서 숨길 요소 | 완전히 숨김. 공간도 사라짐 |
| `visibility: hidden` | 중 | 공간은 유지하면서 숨겨야 할 요소 | 숨김. 공간은 유지됨 |
| `position: static` | 상 | 기본값이므로 별도 지정 불필요 | 일반 흐름대로 배치 |
| `position: relative` | 상 | absolute 자식의 기준점이 될 부모, 약간 이동이 필요한 요소 | 자기 원래 위치 기준 이동. 주변에 영향 없음 |
| `position: absolute` | 상 | 툴팁, 드롭다운, 배지, 아이콘 오버레이 등 | `relative` 부모 기준 배치. 일반 흐름에서 이탈 |
| `position: fixed` | 상 | `nav`, 플로팅 버튼, 채팅 위젯 | 화면 기준 고정. 스크롤해도 안 움직임 |
| `position: sticky` | 상 | `nav`, 사이드바 목차 | 스크롤 시 특정 지점에서 고정. `top: 0`과 함께 |
| `top` / `right` / `bottom` / `left` | 상 | `position`이 static이 아닌 요소 안 | position과 함께 위치 지정 |
| `z-index` | 상 | 모달, 드롭다운, 툴팁, `nav` 등 겹치는 요소 | 쌓임 순서. 숫자 클수록 앞에. `position`이 있어야 작동 |

| `arr.filter((item) => 조건)` | 상 | 배열에서 특정 조건만 걸러낼 때 | 조건에 맞는 요소만 새 배열로 반환 |
| `arr.find((item) => 조건)` | 중 | 특정 요소 하나를 찾을 때 | 조건에 맞는 첫 번째 요소 하나 반환 |
| `arr.includes(값)` | 상 | 포함 여부 확인이 필요한 조건문 | 값 포함 여부. `true` / `false` |
| `arr.reduce((acc, cur) => acc + cur, 초기값)` | 중 | 배열을 하나의 값으로 합산할 때 | 배열 전체를 순회하며 누적값 반환. 합계, 평균 등 |
| `arr.some((item) => 조건)` | 중 | 하나라도 조건에 맞는지 확인 | 하나라도 true면 true 반환 |
| `arr.every((item) => 조건)` | 중 | 전부 조건에 맞는지 확인 | 전부 true여야 true 반환 |
| `arr.length` | 상 | 배열 길이 확인 | 배열 요소 수 반환 |
| `Object.keys(객체)` | 중 | 객체 키 목록이 필요할 때 | 객체의 키만 배열로 반환. `["name", "age"]` |
| `Object.values(객체)` | 중 | 객체 값 목록이 필요할 때 | 객체의 값만 배열로 반환 |
| `Object.entries(객체)` | 중 | 키-값 쌍을 순회할 때 | `[["name", "희진"], ["age", 28]]` 형태로 반환 |
| `str.split("구분자")` | 상 | 문자열을 배열로 나눌 때 | `"a,b,c".split(",")` → `["a","b","c"]` |
| `str.trim()` | 상 | 입력값 공백 제거 | 앞뒤 공백 제거. form 입력값 처리에 자주 씀 |
| `str.includes("검색어")` | 상 | 문자열 포함 여부 확인 | 검색, 필터링에 자주 씀 |
| `str.replace("찾기", "바꾸기")` | 중 | 문자열 치환 | 첫 번째 일치만 변환. 전체 변환은 정규식 사용 |
| `str.toLowerCase()` / `str.toUpperCase()` | 중 | 대소문자 통일이 필요한 비교, 검색 | 대소문자 변환 |

## 객체 (Object)

| | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `const obj = { key: value }` | 상 | JS 파일 어디든 | 객체 리터럴 선언. 키-값 쌍으로 데이터 구조화 |
| `obj.key` | 상 | 객체 값을 읽거나 쓸 때 | 점 표기법. `user.name` |
| `obj["key"]` | 상 | 키가 변수이거나 동적일 때 | 괄호 표기법. `obj[변수]` |
| `obj.key = 값` | 상 | 객체 속성 수정/추가 | 존재하면 수정, 없으면 추가 |
| `delete obj.key` | 중 | 객체 속성 제거 | 해당 키-값 쌍 삭제 |
| `"key" in obj` | 중 | 객체에 키가 있는지 확인 | `true` / `false` 반환 |

## 비동기

| | 빈도 | 위치 | 설명 |
|---|---|---|---|
| `setTimeout(콜백, ms)` | 상 | 지연 실행이 필요한 곳 | 일정 시간 후 한 번 실행 |
| `setInterval(콜백, ms)` | 중 | 반복 실행이 필요한 곳 | 일정 시간마다 반복 실행 |
| `clearInterval(타이머)` | 중 | setInterval을 중단할 조건 발생 시 | 반복 중단 |
| `async function` | 상 | 서버 요청, 비동기 작업을 하는 함수 선언부 | 함수 안에서 await 사용 가능 |
| `await` | 상 | async 함수 안, Promise 앞 | Promise 완료될 때까지 대기. async 함수 안에서만 |
| `try { } catch(err) { }` | 상 | async 함수 안 | 오류 처리. async/await와 세트 |
| `fetch("url")` | 상 | async 함수 안 | 서버에 HTTP 요청. Promise 반환 |
| `.then()` / `.catch()` | 상 | Promise 뒤 | 성공 / 실패 처리 |
| `window.scrollY` | 상 | scroll 이벤트 핸들러 안 | 현재 스크롤 위치(px) |
| `el.scrollIntoView({ behavior: "smooth" })` | 상 | 클릭 이벤트 핸들러 안 | 해당 요소로 부드럽게 스크롤 이동 |
| `navigator.clipboard.writeText(값)` | 중 | 복사 버튼 클릭 이벤트 안 | 클립보드에 텍스트 복사. Promise 반환 |
