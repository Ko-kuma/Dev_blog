# Python Streamlit 할 일 목록 프로젝트 전체 코드 분석

이 문서는 `app.py`, `todo_List.py`, `task.py`, `storage.py`의 코드를 기반으로,

각 코드가 어떤 역할을 하는지, 어떤 순서로 호출되는지, 그리고 프로그램이 전체적으로 어떤 방식으로 동작하는지를 분석한 기록이다.

아직은 혼자 프로젝트의 전체 구조와 흐름을 완전히 설계하기에는 부족한 부분이 있어, 

LLM의 도움을 받아 코드를 분석하고 이해하면서 필요한 내용을 정리했다.

다만 단순히 코드를 받아 적는 것이 아니라, 왜 이러한 구조로 작성되었는지와 각 코드의 역할을 직접 이해하고 정리하는 것을 목표로 했다.

원본 코드는 그대로 유지했으며, 오타나 오류 가능성이 있는 부분도 수정하지 않았다.

대신 해당 코드를 분석하는 과정에서 오류가 발생하는 원인과 수정 방법을 별도로 정리했다.

#  프로젝트 전체 구조

```text
app.py
├─ Streamlit 화면 표시
├─ 사용자 입력 처리
├─ 추가·완료·삭제 요청 처리
└─ 다른 파일의 클래스와 함수 호출

todo_List.py
├─ 여러 Task 객체 관리
├─ 할 일 추가
├─ 할 일 완료 처리
└─ 할 일 삭제

task.py
├─ 할 일 하나의 제목 저장
├─ 완료 여부 저장
├─ 출력 문자열 생성
└─ JSON 저장용 딕셔너리 변환

storage.py
├─ Task 목록을 tasks.json에 저장
└─ tasks.json을 읽어 Task 목록으로 복원
```

전체 흐름은 다음과 같다.

```text
사용자
↓
app.py
↓
todoList 객체
↓
Task 객체
↓
storage.py
↓
tasks.json
```

# 프로그램 전체 실행 순서

```text
1. 터미널에서 streamlit run app.py 실행
2. app.py가 streamlit을 불러옴
3. app.py가 todo_List.py의 todoList 클래스를 불러옴
4. app.py가 storage.py의 save_tasks와 load_tasks를 불러옴
5. storage.py가 task.py의 Task 클래스를 불러옴
6. Streamlit 페이지 제목과 아이콘 설정
7. session_state에 todo 객체가 있는지 확인
8. 없다면 todoList 객체 생성
9. load_tasks()로 tasks.json 불러오기
10. 입력창과 추가 버튼 표시
11. todo.tasks를 반복하며 체크박스·제목·삭제 버튼 표시
12. 사용자 입력에 따라 추가·완료·삭제 처리
13. 변경될 때마다 save_tasks()로 tasks.json 저장
14. st.rerun()으로 app.py를 다시 실행해 최신 화면 표시
```

## `import streamlit as st`

```python
import streamlit as st
```

`streamlit` 라이브러리를 불러온다.

`as st`는 긴 이름에 짧은 별칭을 붙이는 문법이다. 이후 코드에서는 `streamlit.title()` 대신 `st.title()`처럼 사용할 수 있다.

이 프로젝트에서 Streamlit은 파이썬 코드로 입력창, 버튼, 체크박스, 글자, 열 구조 등을 브라우저 화면에 표시하는 역할을 한다.

쉼표를 사용해 한 모듈에서 여러 이름을 동시에 가져오고 있다.

## session_state 확인

```python
if "todo" not in st.session_state:
```

현재 브라우저 세션의 `st.session_state` 안에 `todo`라는 이름의 값이 없는지 확인한다.

Streamlit은 버튼을 누르거나 체크박스를 바꾸면 `app.py`를 위에서 아래까지 다시 실행한다. 일반 변수만 사용하면 다시 실행될 때 값이 초기화될 수 있으므로, 세션 동안 유지할 값은 `st.session_state`에 저장한다.

`not in`은 특정 값이 컬렉션 안에 존재하지 않는지 확인하는 연산자다.

---

```python
st.session_state.todo = todoList()
```

`todoList` 클래스를 호출해 객체를 만든다.

실행 순서는 다음과 같다.

```text
todoList()
↓
todoList.__init__() 실행
↓
self.tasks = [] 생성
↓
생성된 객체를 st.session_state.todo에 저장
```

---

```python
st.session_state.todo.tasks = load_tasks()
```

`load_tasks()`를 실행해 JSON 파일의 기존 할 일을 불러온다.

반환된 `Task` 객체 리스트를 `todoList` 객체의 `tasks` 속성에 저장한다.

---

```python
todo = st.session_state.todo
```

세션에 저장된 객체를 `todo`라는 짧은 변수로 가리킨다.

새 객체를 만드는 것이 아니라 같은 객체를 참조한다.

## 새 할 일 입력

```python
new_task = st.text_input("새로운 할 일을 입력하세요")
```

텍스트 입력창을 화면에 표시한다.

사용자가 입력한 문자열을 `new_task` 변수에 저장한다. 아무것도 입력하지 않았다면 빈 문자열이 들어간다.

---

```python
if st.button("추가"):
```

`추가` 버튼을 표시한다.

버튼을 클릭한 실행에서는 `True`, 클릭하지 않은 실행에서는 `False`를 반환한다. 따라서 버튼을 눌렀을 때만 아래 들여쓰기된 코드가 실행된다.

---

```python
if new_task:
```

사용자가 실제 내용을 입력했는지 확인한다.

파이썬에서 빈 문자열은 `False`, 내용이 있는 문자열은 `True`로 처리된다. 따라서 빈 문자열은 추가되지 않는다.

---

```python
todo.add_task(new_task)
```

`todoList` 객체의 `add_task()` 메서드를 호출한다.

내부 호출 순서는 다음과 같다.

```text
todo.add_task(new_task)
↓
Task(new_task) 생성
↓
Task.__init__() 실행
↓
새 Task 객체를 self.tasks에 append
```

---

```python
save_tasks(todo.tasks)
```

현재 `todo.tasks` 리스트 전체를 `save_tasks()` 함수에 전달한다.

`save_tasks()`는 각 `Task` 객체를 딕셔너리로 변환한 뒤 `tasks.json`에 저장한다.

---

```python
st.rerun()
```

현재 Streamlit 앱을 즉시 다시 실행한다.

추가된 할 일이 화면에 바로 표시되도록 `app.py`를 처음부터 다시 실행한다.

## 목록 반복

```python
for index, task in enumerate(todo.tasks):
```

`todo.tasks`에 들어 있는 `Task` 객체들을 하나씩 반복한다.

`enumerate()`를 사용하면 현재 위치 번호와 값을 함께 받을 수 있다.

- `index`에는 0부터 시작하는 위치 번호가 들어간다.
- `task`에는 현재 `Task` 객체가 들어간다.

---

```python
col1, col2, col3 = st.columns([1, 6, 2])
```

한 줄을 세 개의 열로 나눈다.

열 비율은 `1 : 6 : 2`다.

- `col1`: 체크박스
- `col2`: 할 일 제목
- `col3`: 삭제 버튼

하나의 함수가 반환한 여러 값을 여러 변수에 나눠 저장하는 다중 할당 문법이다.

## 체크박스 영역

```python
with col1:
```

아래에서 생성되는 Streamlit 요소를 첫 번째 열에 배치한다.

---

```python
checked = st.checkbox("", value=task.done, key=f"done_{index}")
```

체크박스를 생성한다.

- 첫 번째 빈 문자열은 체크박스 옆에 표시할 문구다.
- `value=task.done`은 현재 완료 상태를 기본 체크 상태로 사용한다.
- `key`는 각 체크박스를 구별하는 고유한 이름이다.

`f"done_{index}"`는 f-string이다. 인덱스에 따라 `done_0`, `done_1`, `done_2` 같은 값이 만들어진다.

체크박스의 현재 상태는 `checked` 변수에 저장된다.

---

```python
if checked != task.done:
```

화면의 체크 상태와 객체에 저장된 기존 완료 상태가 다른지 비교한다.

`!=`는 두 값이 서로 다르다는 뜻이다.

상태가 바뀌었을 때만 아래 코드가 실행된다.

---

```python
task.done = checked
```

현재 `Task` 객체의 완료 상태를 체크박스 값으로 변경한다.

체크하면 `True`, 체크를 해제하면 `False`가 저장된다.

---

```python
save_tasks(todo.tasks)
```

변경된 완료 상태를 JSON 파일에 저장한다.

---

```python
st.rerun()
```

변경된 완료 상태를 기준으로 화면을 다시 그린다.

# task.py 역할 및 분석

[task.py](https://github.com/Ko-kuma/Python_Portfolio/blob/main/%ED%95%A0%EC%9D%BC%EB%AA%A9%EB%A1%9D(%EC%9B%B9)/task.py)

`task.py`는 할 일 하나를 나타내는 `Task` 클래스를 정의한다.

각 객체는 다음 정보를 가진다.

```text
title = 할 일 제목
done = 완료 여부
```

## 클래스 정의

```python
class Task :
```

`Task`라는 클래스를 정의한다.

클래스는 관련 데이터와 기능을 하나로 묶는 설계도다.

---

```python
def __init__(self, title, done=False) :
```

`Task` 객체가 만들어질 때 자동으로 실행되는 초기화 메서드다.

- `self`: 현재 만들어지는 객체 자신
- `title`: 할 일 제목
- `done`: 완료 여부

`done=False`는 기본값이다. 값을 따로 전달하지 않으면 미완료 상태로 시작한다.

---

```python
self.title = title
```

매개변수로 받은 제목을 현재 객체의 `title` 속성에 저장한다.

왼쪽의 `self.title`은 객체 속성이고, 오른쪽의 `title`은 전달받은 매개변수다.

---

```python
self.done = done
```

매개변수로 받은 완료 여부를 현재 객체의 `done` 속성에 저장한다.

- `False`: 미완료
- `True`: 완료

---

## 문자열 표현

```python
def __str__(self):
```

`Task` 객체를 문자열로 표현하는 특수 메서드다.

`print(task)`처럼 객체를 출력할 때 이 메서드가 반환한 문자열이 사용된다.

---

```python
status = "v" if self.done else " "
```

조건 표현식이다.

```text
self.done이 True이면 status = "v"
self.done이 False이면 status = " "
```

---

```python
return f"[{status}] {self.title}"
```

# 20. 전체 데이터 동작 원리

## 메모리 안의 데이터

할 일 하나는 `Task` 객체다.

```text
Task 객체
├─ title
└─ done
```

여러 할 일은 `todo.tasks` 리스트 안에 저장된다.

```text
todo.tasks
├─ Task 객체
├─ Task 객체
└─ Task 객체
```

---

## 저장할 때

`Task` 객체는 JSON이 직접 저장할 수 있는 기본 자료형이 아니다.

따라서 `to_dict()`를 통해 딕셔너리로 바꾼다.

```text
Task 객체
↓
to_dict()
↓
딕셔너리
↓
json.dump()
↓
tasks.json
```

---

## 불러올 때

`json.load()`는 JSON 데이터를 리스트와 딕셔너리로 바꾼다.

그 딕셔너리 값을 다시 `Task` 생성자에 전달해 객체로 복원한다.

```text
tasks.json
↓
json.load()
↓
딕셔너리
↓
Task(title, done)
↓
Task 객체
```

# 21. 최종 정리

이 프로젝트는 화면, 목록 관리, 개별 데이터, 파일 저장 기능을 서로 다른 파일로 분리한 구조다.

```text
app.py
= 웹 화면과 사용자 입력 처리

todo_List.py
= 여러 할 일의 추가·완료·삭제 관리

task.py
= 할 일 하나의 제목과 완료 상태 표현

storage.py
= JSON 파일 저장과 불러오기
```

사용자가 화면에서 동작하면 `app.py`가 `todoList`의 메서드나 `storage.py`의 함수를 호출한다.

`todoList`는 여러 `Task` 객체를 리스트로 관리한다.

`Task`는 할 일 제목과 완료 여부를 저장한다.

`storage.py`는 객체를 딕셔너리로 바꿔 JSON에 저장하고, JSON의 딕셔너리를 다시 객체로 복원한다.

Streamlit은 입력이나 버튼 동작이 발생할 때 `app.py`를 다시 실행한다. 따라서 브라우저 세션 동안 객체를 유지하기 위해 `st.session_state`를 사용하고, 프로그램 재실행이나 종료 뒤에도 데이터를 남기기 위해 `tasks.json`을 사용한다.
