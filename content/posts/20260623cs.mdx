## CS 용어 정리

### HTML (HyperText Markup Language)
웹 페이지의 논리적 구조와 뼈대를 정의하는 마크업 언어입니다.

### CSS (Cascading Style Sheets)
구조화된 웹 페이지에 시각적 스타일(레이아웃, 색상, 폰트 등)을 입히는 스타일시트 언어입니다.

### JavaScript (JS)
웹 페이지에 동적 상호작용 및 비즈니스 로직 제어를 부여하는 프로그래밍 언어입니다.

### API (Application Programming Interface)
프론트엔드와 백엔드처럼 서로 다른 애플리케이션이 정해진 규칙에 따라 데이터를 교환할 수 있도록 돕는 통신 인터페이스입니다.

### JWT (JSON Web Token)
로그인 성공 시 발급받는 디지털 신분증. 당사자 간 정보를 JSON 형태로 안전하게 전송하기 위한 보안 토큰 규격입니다.

### httpOnly Cookie
JWT를 브라우저에 저장할 때 JavaScript로 접근하지 못하도록 막는 옵션.
이 설정이 없으면 XSS 공격으로 토큰을 탈취당할 수 있습니다.

### CORS (Cross-Origin Resource Sharing)
백엔드가 **"이 주소에서 오는 요청만 허용할게"** 라고 브라우저에게 알려주는 규칙입니다.
허가되지 않은 다른 사이트에서 내 백엔드에 몰래 요청 보내는 것을 막습니다.

<Callout>
핵심은 "허가된 주소만 통과시키는 규칙"입니다.
</Callout>

### Docker
애플리케이션을 컨테이너 박스에 포장해, 어떤 컴퓨터·서버에서든 동일하게 실행되도록 지원하는 플랫폼입니다.

### MDX (Markdown eXtended)
마크다운에 React 컴포넌트를 함께 사용할 수 있도록 확장한 포맷입니다.
Next.js 블로그에서 글을 작성할 때 사용합니다.

### ORM (Object-Relational Mapping)
객체지향 코드와 관계형 데이터베이스 테이블을 자동으로 매핑해주는 도구입니다.
SQL 쿼리 없이 Python 코드로 데이터를 조작할 수 있게 해줍니다. (예: SQLAlchemy)

## 복습

### Q1. 인증 vs 인가 차이는?
- **인증 (Authentication):** 이 사람이 누구인가? → 로그인
- **인가 (Authorization):** 이 사람이 이걸 할 권한이 있나? → 일반회원이 관리자 페이지 접근 시도

### Q2. httpOnly 없이 JWT를 로컬스토리지에 저장하면?
XSS 공격으로 JavaScript가 토큰을 읽어서 탈취 가능합니다.

### Q3. Prettier / ESLint를 프로젝트 초기에 적용해야 하는 이유는?
코드 가독성 향상, 스타일 충돌 방지, 팀 전체 코드 품질 통일.

### Q4. ORM(SQLAlchemy)이 raw SQL보다 나은 점은?
DB 종류가 바뀌어도 코드 수정이 최소화되고, 코드 가독성과 유지보수가 편합니다.
