## 1. 연산자

### 산술 연산자 (Arithmetic Operator)

**정의**: 숫자를 계산하는 연산자입니다. `+`, `-`, `*`, `/`, `//`, `%`, `**`가 있습니다.

**CS적 의의**:

- **일반 나눗셈 vs 정수 나눗셈**: `/`는 항상 float를 반환하지만, `//`(Floor Division, 몫만 취하는 나눗셈)는 소수점을 버린 값을 반환합니다.
- **모듈로 연산(Modulo Operation)**: `%`는 나머지를 구하는 연산으로, 짝수/홀수 판별이나 순환 로직에 쓰입니다.
- **언어 간 차이**: JS에는 `//` 전용 연산자가 없어서 `Math.floor(a / b)`로 대신 계산해야 합니다.

```python
print(7 // 2)   # 3 (몫)
print(7 % 2)    # 1 (나머지)
```

---

### 비교 연산자 (Comparison Operator)

**정의**: 두 값을 비교해서 True/False를 반환하는 연산자입니다. `==`, `!=`, `>`, `<`, `>=`, `<=`가 있습니다.

**CS적 의의**:

- **대입(Assignment) vs 비교(Comparison)**: `=`는 값을 저장하는 대입 연산자이고, `==`는 두 값이 같은지 확인하는 비교 연산자입니다. 기호가 비슷해서 초보자가 자주 혼동합니다.
- **느슨한 비교 vs 엄격한 비교**: JS는 타입을 변환해서 비교하는 `==`와, 타입까지 비교하는 `===`를 구분합니다. Python은 `==` 하나만 있고, 이는 JS의 `===`처럼 값과 타입을 함께 비교합니다.

```python
print(1 == "1")   # False (Python은 타입이 다르면 다른 값으로 봄)
```

---

### 논리 연산자 (Logical Operator)

**정의**: 여러 개의 참/거짓 값을 조합해서 하나의 참/거짓 결과로 만드는 연산자입니다. `and`, `or`, `not`이 있습니다.

**CS적 의의**:

- **불리언 대수(Boolean Algebra)**: 논리 연산자는 수학의 불리언 대수(참/거짓 값의 연산 체계)를 그대로 코드에 옮긴 것입니다. `and`는 논리곱, `or`는 논리합, `not`은 부정에 해당합니다.
- **표기 방식의 언어 차이**: Python은 영어 단어(and, or, not)를, JS는 기호(&&, ||, !)를 사용합니다.

```python
can_ride = height >= 120 and weight <= 100
```

---

### 멤버십 연산자 (Membership Operator)

**정의**: 어떤 값이 리스트, 문자열 같은 데이터 묶음 안에 포함되어 있는지 확인하는 연산자입니다. `in`, `not in`이 있습니다.

**CS적 의의**:

- **포함 관계 검사**: 값이 컬렉션(모음) 안에 있는지 순회하며 확인하는 연산입니다.
- **언어 간 의미 차이**: JS의 `in`은 배열의 값이 아니라 객체의 키(key)가 있는지 확인하는 용도로 쓰입니다. 배열 값 확인은 `includes()`를 따로 써야 해서, Python의 `in`과 이름은 같지만 동작이 다릅니다.

```python
fruits = ["사과", "바나나"]
print("사과" in fruits)   # True
```

## 2. 조건문

### 조건문 (Conditional Statement)

**정의**: 특정 조건이 참인지 거짓인지에 따라 서로 다른 코드를 실행하게 하는 문법입니다. `if`, `elif`, `else`로 구성됩니다.

**CS적 의의**:

- **제어 흐름(Control Flow)**: 조건문은 프로그램이 위에서 아래로만 순서대로 실행되는 게 아니라, 상황에 따라 실행 경로가 갈라지게 만드는 대표적인 제어 흐름 도구입니다.
- **순차 평가와 단락(Short-circuit)**: `if` → `elif` → `else` 순서로 위에서부터 확인하다가, 처음으로 참이 되는 조건 하나만 실행하고 나머지는 평가하지 않습니다.
- **문법 표현의 언어 차이**: JS는 조건을 괄호로, 블록을 중괄호로 감싸지만(`if (조건) { }`), Python은 콜론과 들여쓰기만으로 표현합니다.

```python
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
else:
    grade = "C"
```

---

### 중첩 조건문 (Nested Conditional)

**정의**: 조건문 안에 또 다른 조건문을 넣어, 단계별로 조건을 확인하는 구조입니다.

**CS적 의의**:

- **단계적 검증(Sequential Validation)**: 1차 조건을 통과한 경우에만 2차 조건을 확인하도록 만들어, "여러 단계의 검증"을 표현할 수 있습니다.
- **가독성과의 트레이드오프**: 중첩이 깊어질수록(3~4단계 이상) 코드를 읽기 어려워지는데, 이를 "화살표 안티패턴(Arrow Anti-pattern)"이라 부르기도 합니다. `and`로 조건을 합치거나 함수로 분리해서 중첩을 얕게 유지하는 것이 권장됩니다.

```python
if has_ticket:
    if height >= 120:
        print("입장 가능")
```

---

### 조건부 표현식 (Conditional Expression / Ternary Operator)

**정의**: if/else 문을 한 줄로 줄여서, 조건에 따라 다른 값을 즉시 만들어내는 표현식입니다.

**CS적 의의**:

- **문(Statement)과 표현식(Expression)의 차이**: 일반 if문은 "실행"만 하는 문(statement)이지만, 조건부 표현식은 "값"을 만들어내는 표현식(expression)입니다. 이 차이 때문에 조건부 표현식은 변수에 바로 대입할 수 있습니다.
- **어순의 언어 차이**: JS는 `조건 ? 참일때값 : 거짓일때값` 순서(기호 사용)이고, Python은 `참일때값 if 조건 else 거짓일때값` 순서(단어 사용)입니다. 같은 개념이지만 읽는 순서가 정반대입니다.

```python
status = "성인" if age >= 19 else "미성년자"
```

---

### Truthy / Falsy

**정의**: True/False가 아닌 값도 조건문 안에서 자동으로 참 또는 거짓으로 취급되는 성질입니다. 참으로 취급되면 Truthy, 거짓으로 취급되면 Falsy라고 부릅니다.

**CS적 의의**:

- **암시적 타입 변환(Implicit Type Conversion)**: 조건문은 값을 bool로 자동 변환해서 판단합니다. `0`, `""`, `None`, `[]`, `{}`는 Falsy로, 그 외 대부분의 값은 Truthy로 취급됩니다.
- **언어 간 가장 흔한 함정**: Python은 빈 리스트(`[]`)와 빈 딕셔너리(`{}`)가 Falsy지만, JS는 빈 배열과 빈 객체가 오히려 Truthy입니다. 두 언어를 오가며 코드를 짤 때 가장 자주 발생하는 실수 중 하나입니다.

```python
if []:
    print("실행 안 됨")   # Python에서는 빈 리스트가 Falsy라 이 줄은 실행되지 않음
```

## Key Questions

**Q1.** `=`와 `==`를 헷갈리면 왜 위험한가요?

> **A.** 둘은 하는 일이 완전히 다르기 때문입니다.
> `=`는 오른쪽 값을 왼쪽 변수에 저장하는 대입 연산자이고, `==`는 두 값이 같은지 확인만 하고 아무것도 저장하지 않는 비교 연산자입니다.
> `if age = 20:`처럼 실수로 `=`를 쓰면 Python에서는 아예 `SyntaxError`가 발생해서 바로 알아챌 수 있지만, 언어에 따라 이런 실수가 조용히 버그로 남는 경우도 있어서 항상 구분해서 써야 합니다.

---

**Q2.** 중첩 조건문과 `and`로 합친 조건문은 결과가 같은데 왜 구분해서 배우나요?

> **A.** 결과는 같아도 "의도를 표현하는 방식"이 다르기 때문입니다.
> `if a: if b:` 형태는 "a를 통과한 경우에만 b를 별도로 확인한다"는 단계적 검증의 뉘앙스를 표현합니다.
> `if a and b:` 형태는 "a와 b를 동시에 만족해야 한다"는 하나의 조건 묶음으로 표현합니다.
> 실무에서는 조건이 2개 정도면 `and`로 합치는 게 더 읽기 쉽고, 각 단계마다 서로 다른 처리(예: 다른 에러 메시지)가 필요하면 중첩 조건문을 씁니다.

---

**Q3.** Truthy/Falsy 개념이 왜 중요한가요?

> **A.** 조건문에서 값을 bool로 매번 명시적으로 변환하지 않아도 되게 해주지만, 언어마다 규칙이 달라 착각하기 쉽기 때문입니다.
> 예를 들어 API에서 받은 리스트가 비어있는지 확인할 때 `if response_data:`라고 짧게 쓸 수 있는 건 Truthy/Falsy 덕분입니다.
> 하지만 Python에서 빈 리스트는 Falsy인 반면 JS에서 빈 배열은 Truthy이기 때문에, 두 언어를 오가며 작업할 때는 이 차이를 반드시 기억해야 합니다.
