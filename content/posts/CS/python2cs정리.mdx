## 1. 변수와 이름 규칙

### 변수 (Variable)

**정의**: 값을 저장해두고 이름으로 불러 쓸 수 있게 만든 저장 공간입니다.

**CS적 의의**:

- **동적 타이핑과의 관계**: Python은 변수에 타입을 미리 선언하지 않는 동적 타이핑 언어라, 하나의 변수 이름에 어떤 타입의 값이든 재할당할 수 있습니다.
- **변수 vs 값**: 타입은 변수 자체가 아니라 변수가 가리키는 값(객체)에 속합니다. 그래서 같은 이름에 다른 타입의 값을 넣어도 문법 오류가 나지 않습니다.

```python
x = 5        # x는 int 타입 값을 가리킴
x = "다섯"    # 같은 이름 x가 이제 str 타입 값을 가리킴 (에러 아님)
```

---

### 예약어 (Reserved Word)

**정의**: 프로그래밍 언어 문법이 이미 특별한 의미로 사용하고 있어서, 변수명으로 쓸 수 없는 단어입니다.

**CS적 의의**:

- **파서(Parser)와의 충돌**: 인터프리터가 코드를 해석할 때, 예약어는 이미 문법 구조를 나타내는 신호로 약속되어 있습니다. 이 단어를 변수명으로 쓰면 파서가 문법 구조와 변수명을 구분하지 못해 `SyntaxError`가 발생합니다.
- **내장 이름(Built-in Name)과의 차이**: `list`, `dict`, `type`처럼 예약어는 아니지만 Python이 기본 제공하는 함수/자료형 이름도 있습니다. 이런 이름은 변수명으로 써도 에러는 안 나지만, 원래 기능을 그 스코프 안에서 덮어써버리는 부작용이 생깁니다.

```python
class = "수학"     # SyntaxError (class는 예약어)
list = [1, 2, 3]   # 에러는 안 나지만, 이 스코프에서 list() 내장 함수를 못 쓰게 됨
```

> `import keyword; print(keyword.kwlist)`로 현재 Python 버전의 예약어 전체 목록을 확인할 수 있습니다.

### 네이밍 컨벤션 (Naming Convention)

**정의**: 여러 단어를 하나의 이름으로 이어붙일 때, 대소문자와 구분 기호를 어떻게 쓸지에 대한 코드 작성 관례입니다.

**CS적 의의**:

- **camelCase**: 첫 단어는 소문자, 이후 단어는 첫 글자만 대문자로 잇는 방식 (`userName`). JavaScript의 변수/함수 명명에 쓰입니다.
- **PascalCase**: 모든 단어의 첫 글자를 대문자로 잇는 방식 (`UserProfile`). Python과 JS 모두 클래스/컴포넌트 이름에 공통으로 씁니다.
- **snake_case**: 전부 소문자 + 언더스코어로 잇는 방식 (`user_name`). PEP 8이 권장하는 Python의 변수/함수 명명 방식입니다.
- **SCREAMING_SNAKE_CASE**: snake_case를 전부 대문자로 쓴 방식 (`MAX_SIZE`). 값이 바뀌지 않는 상수에 관례적으로 씁니다.
- **kebab-case**: 전부 소문자 + 하이픈으로 잇는 방식 (`user-name`). URL 경로나 파일명에 주로 씁니다.

| 대상 | JavaScript | Python |
|---|---|---|
| 변수/함수 | camelCase | snake_case |
| 클래스 | PascalCase | PascalCase |
| 상수 | SCREAMING_SNAKE_CASE | SCREAMING_SNAKE_CASE |

## 2. 자료형과 형변환

### 기본 자료형 (Primitive Data Type)

**정의**: Python이 기본으로 제공하는 값의 종류입니다. int(정수), float(실수), str(문자열), bool(참/거짓), NoneType(값 없음)이 있습니다.

**CS적 의의**:

- **플래그(Flag)**: bool 타입 변수 중, 어떤 상태의 켜짐/꺼짐을 나타내는 데 쓰이는 변수를 관례적으로 플래그라고 부릅니다 (`is_active`, `is_deleted`).
- **None과 다른 언어의 "없음"**: Python은 값이 없음을 `None` 하나로 통일해서 표현합니다. JavaScript는 `null`(의도적으로 비움)과 `undefined`(아예 할당되지 않음)로 나누어 표현한다는 점이 다릅니다.

```python
is_active = True     # 플래그: 계정 활성 상태 여부
nickname = None       # 아직 값이 없음
```

---

### 형변환 (Type Casting)

**정의**: 하나의 자료형 값을 다른 자료형으로 바꾸는 것입니다.

**CS적 의의**:

- **명시적 형변환(Explicit Casting)**: `int()`, `str()`, `float()`처럼 개발자가 직접 함수를 호출해서 타입을 바꾸는 것을 말합니다. Python의 형변환은 대부분 이 방식입니다.
- **ValueError**: 변환이 불가능한 값을 형변환하려 할 때 발생하는 에러입니다 (`int("혜진")`). "값 자체가 잘못됐다"는 의미로, 문법 오류(SyntaxError)와는 다른 종류의 에러입니다.

```python
age_str = "25"
age_int = int(age_str)   # 명시적 형변환
```

## 3. 부동소수점과 진법

### 부동소수점 (Floating Point)

**정의**: 컴퓨터가 소수(실수)를 저장하는 방식이며, 저장 방식의 한계로 미세한 오차가 생길 수 있는 현상을 통틀어 부르는 말입니다.

**CS적 의의**:

- **2진법(Binary) vs 10진법(Decimal)**: 사람은 0~9를 쓰는 10진법을, 컴퓨터는 0과 1만 쓰는 2진법을 사용합니다. 10진법에서 딱 떨어지는 0.1 같은 수가 2진법으로는 무한소수가 되어, 저장 단계에서부터 오차를 안고 시작합니다.
- **부동소수점 오차의 실무 영향**: `0.1 + 0.2 == 0.3`이 `False`로 나오는 것이 대표적 사례입니다. 금액 계산처럼 정확도가 중요한 로직에서는 float 대신 `decimal` 모듈의 `Decimal` 타입을 사용해 오차를 없앱니다.

```python
print(0.1 + 0.2)             # 0.30000000000000004
print(0.1 + 0.2 == 0.3)      # False

from decimal import Decimal
print(Decimal("0.1") + Decimal("0.2"))  # 0.3 (정확)
```

> 부동소수점 오차는 Python만의 문제가 아니라, 2진법으로 실수를 저장하는 대부분의 프로그래밍 언어(JavaScript 포함)에서 공통으로 발생하는 현상입니다.

## 4. 연산자와 숫자 표기

### 정수 나눗셈과 나머지 연산 (Floor Division & Modulo)

**정의**: `//`는 나눗셈의 몫만 정수로 반환하고, `%`는 나눗셈의 나머지를 반환하는 연산자입니다.

**CS적 의의**:

- **일반 나눗셈(`/`)과의 차이**: `/`는 항상 float를 반환하지만, `//`는 소수점 이하를 버린 정수(또는 자료형에 맞는 값)를 반환합니다.
- **모듈로 연산(Modulo Operation)의 활용**: `%`는 "일정한 간격으로 반복되는" 로직에 널리 쓰입니다. 예를 들어 `n % 2 == 0`으로 짝수 여부를 판별하거나, 순환 목록(리스트를 순서대로 돌려쓰기)을 구현할 때 씁니다.

```python
print(7 / 2)    # 3.5
print(7 // 2)   # 3
print(7 % 2)    # 1
```

## 5. 개발 환경

### 환경변수 (Environment Variable)

**정의**: 프로그램 코드 밖에 저장해두고, 실행 시점에 코드가 불러와서 사용하는 값입니다.

**CS적 의의**:

- **보안과의 관계**: API 키, 데이터베이스 비밀번호처럼 코드에 직접 적으면 안 되는 민감한 값을 관리하는 표준적인 방법입니다. 코드는 GitHub 등에 공개될 수 있지만, 환경변수 파일(`.env`)은 별도로 관리해 노출을 막습니다.
- **`os.getenv()`**: Python에서 환경변수 값을 코드로 불러올 때 쓰는 표준 라이브러리 함수입니다.

```python
import os
database_url = os.getenv("DATABASE_URL")
```

## Key Questions

**Q1.** 예약어와 내장 이름(built-in name)은 어떻게 다른가요?

> **A.** 예약어는 변수명으로 쓰는 순간 `SyntaxError`가 나서 아예 실행이 안 됩니다.
> 반면 `list`, `dict`, `type` 같은 내장 이름은 변수명으로 써도 문법 오류는 나지 않습니다.
> 다만 그 순간부터 해당 스코프 안에서는 원래의 내장 기능(`list()` 등)을 쓸 수 없게 되므로, 예약어는 아니어도 실무에서는 피하는 것이 안전합니다.

---

**Q2.** 0.1 + 0.2가 왜 정확히 0.3이 아닌가요?

> **A.** 컴퓨터가 실수를 2진법으로 저장하기 때문입니다.
> 10진법에서는 0.1이 딱 떨어지는 수지만, 2진법으로 변환하면 끝없이 반복되는 무한소수가 됩니다.
> 컴퓨터는 유한한 저장 공간만 쓸 수 있어서 이 무한소수를 어느 지점에서 잘라 저장하고, 그 과정에서 아주 작은 오차가 생깁니다.
> 이 오차가 누적되면 `0.1 + 0.2`의 결과가 `0.30000000000000004`처럼 나오게 됩니다.

---

**Q3.** `/`와 `//`는 언제 다르게 써야 하나요?

> **A.** 결과값의 형태와 용도가 다르기 때문입니다.
> `/`(일반 나눗셈)는 항상 정확한 값을 float로 반환해서, 평균값이나 비율 계산처럼 소수점이 필요한 경우에 씁니다.
> `//`(정수 나눗셈)는 몫만 필요할 때 씁니다. 예를 들어 "100개의 물건을 한 박스에 12개씩 담으면 몇 박스가 나오는가"처럼, 딱 떨어지는 개수만 필요한 경우에 적합합니다.
>
> ```python
> total_items = 100
> boxes = total_items // 12   # 몇 박스가 필요한지 (8)
> remainder = total_items % 12  # 마지막 박스에 남는 개수 (4)
> ```
