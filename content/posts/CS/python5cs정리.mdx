## 1. 자료구조 기초

### 자료구조 (Data Structure)

**정의**: 여러 개의 값을 효율적으로 저장하고 다루기 위해 정리해둔 방식입니다.

**CS적 의의**:

- **단일 값 vs 자료구조**: 지금까지 배운 int, float, str, bool은 값 하나만 담는 자료형이고, 자료구조는 여러 값을 하나로 묶어 관리하는 상위 개념입니다.
- **선택의 기준**: 어떤 자료구조를 쓸지는 "값을 바꿀 일이 있는가", "순서가 중요한가", "중복을 허용할 것인가" 같은 기준에 따라 달라집니다. 오늘 배운 list/tuple은 그중 "순서가 있는" 자료구조에 속합니다.

---

### 리스트 (List)

**정의**: 여러 개의 값을 순서대로 담고, 나중에 추가·삭제·수정이 가능한 자료구조입니다.

**CS적 의의**:

- **동적 배열(Dynamic Array)**: 리스트는 내부적으로 크기가 자동으로 늘어나는 배열입니다. 값을 추가할 때마다 개발자가 크기를 직접 계산할 필요가 없습니다.
- **순서(Order) 보장**: 리스트에 담긴 값은 넣은 순서가 유지됩니다. 이는 나중에 배울 딕셔너리·셋과 리스트를 구분하는 핵심 특징 중 하나입니다.

---

### 인덱싱 (Indexing)

**정의**: 리스트 안의 특정 위치에 있는 값을, 번호(인덱스)로 지정해서 꺼내는 것입니다.

**CS적 의의**:

- **0-기반 인덱싱(Zero-based Indexing)**: 대부분의 프로그래밍 언어는 첫 번째 값을 0번으로 센다. 이는 컴퓨터가 메모리 주소를 "시작 위치로부터 몇 칸 떨어져 있는가"로 계산하는 방식에서 유래한 관례입니다.
- **음수 인덱스**: Python은 `-1`을 "맨 뒤에서 첫 번째"로 해석합니다. 리스트 길이를 몰라도 마지막 값에 바로 접근할 수 있게 해주는 편의 기능입니다.

```python
fruits = ["사과", "바나나", "포도"]
print(fruits[0])    # 사과
print(fruits[-1])   # 포도
```

## 2. 리스트 조작

### 슬라이싱 (Slicing)

**정의**: 리스트(또는 문자열)의 일부 구간만 잘라내어 새로운 리스트로 만드는 것입니다.

**CS적 의의**:

- **반개구간(Half-open Interval)**: `numbers[1:3]`은 인덱스 1부터 시작해서 3 "전까지"만 포함합니다. 시작은 포함, 끝은 미포함인 이 방식을 반개구간이라고 하며, `range()`의 끝값 미포함 규칙과 같은 원리입니다.
- **원본 불변**: 슬라이싱은 원본 리스트를 바꾸지 않고, 잘라낸 부분으로 이루어진 새로운 리스트를 만들어서 반환합니다.

```python
numbers = [10, 20, 30, 40, 50]
print(numbers[1:3])   # [20, 30]
```

---

### 중첩 리스트 (Nested List)

**정의**: 리스트의 각 원소가 또 다른 리스트인 구조입니다.

**CS적 의의**:

- **2차원 데이터 표현**: 행과 열로 구성된 표 형태 데이터(엑셀, 게임판 등)를 표현할 때 자연스럽게 대응됩니다. `grid[0][1]`처럼 인덱스를 두 번 사용해 접근합니다.

```python
grid = [[1, 2], [3, 4]]
print(grid[0][1])   # 2
```

---

### 리스트 메서드 (List Method)

**정의**: 리스트 객체에 딸려있는, 리스트를 조작하는 기능들입니다. `append()`, `insert()`, `pop()`, `sort()`, `count()`, `index()` 등이 있습니다.

**CS적 의의**:

- **제자리 변경(In-place Modification)**: `sort()`처럼 일부 메서드는 새 리스트를 만드는 게 아니라 원본 리스트 자체를 바로 바꿔버립니다. 이런 메서드는 반환값이 `None`인 경우가 많아서, `numbers = numbers.sort()`처럼 다시 대입하면 오히려 값을 잃어버리는 실수를 하기 쉽습니다.

```python
numbers = [3, 1, 2]
numbers.sort()        # 원본이 바로 바뀜 (반환값은 None)
print(numbers)         # [1, 2, 3]
```

---

### remove() vs pop()

**정의**: 둘 다 리스트에서 값을 삭제하는 메서드지만, `remove()`는 값을 기준으로, `pop()`은 인덱스(위치)를 기준으로 삭제한다는 차이가 있습니다.

**CS적 의의**:

- **탐색(Search) vs 접근(Access)**: `remove()`는 리스트를 처음부터 훑으며 값이 일치하는 첫 항목을 찾아야 하므로 탐색 과정이 필요합니다. `pop()`은 인덱스로 위치를 바로 알 수 있어 값을 찾을 필요 없이 즉시 접근합니다.
- **반환값의 유무**: `pop()`은 삭제한 값을 반환해서 그 값을 바로 다른 곳에 활용할 수 있지만, `remove()`는 삭제만 하고 아무것도 반환하지 않습니다(`None`). 이 차이 때문에 "삭제한 값을 다시 써야 하는가"가 둘 중 무엇을 쓸지 결정하는 기준이 됩니다.
- **스택(Stack) 자료구조와의 연결**: `pop()`으로 맨 뒤 값을 계속 꺼내는 방식은, 나중에 배울 스택(마지막에 넣은 것을 먼저 꺼내는 자료구조, LIFO)을 리스트로 흉내낼 때 그대로 쓰이는 패턴입니다.

```python
fruits = ["사과", "바나나", "포도"]
fruits.remove("바나나")   # 값으로 찾아서 삭제, 반환값 없음
last = fruits.pop()        # 인덱스(맨 뒤)로 삭제, 삭제한 값을 반환
```

## 3. 가변성과 참조

### 가변 자료형 vs 불변 자료형 (Mutable vs Immutable)

**정의**: 가변(Mutable) 자료형은 만든 후에도 내용을 바꿀 수 있고, 불변(Immutable) 자료형은 한 번 만들면 내용을 바꿀 수 없습니다.

**CS적 의의**:

- **리스트는 가변, 튜플은 불변**: 이 차이가 오늘 배운 두 자료구조를 구분하는 근본적인 기준입니다. int, float, str, bool, tuple은 모두 불변이고, list와 (다음에 배울) dict는 가변입니다.
- **불변 자료형의 안전성**: 값이 바뀌지 않는다는 게 보장되므로, 여러 곳에서 같은 값을 공유해도 예상치 못하게 바뀔 위험이 없습니다. 이것이 튜플이 좌표값처럼 "고정된 값"에 자주 쓰이는 이유입니다.

---

### 참조 (Reference) vs 복사 (Copy)

**정의**: 참조는 같은 데이터를 가리키는 또 다른 이름표를 만드는 것이고, 복사는 완전히 별도의 데이터를 새로 만드는 것입니다.

**CS적 의의**:

- **변수는 값이 아니라 값의 위치를 가리킨다**: `list2 = list1`은 list1이 담긴 실제 데이터를 복사하는 게 아니라, list1이 가리키는 메모리 위치를 list2도 함께 가리키게 만드는 것입니다. 그래서 list2를 바꾸면 list1도 함께 바뀝니다.
- **얕은 복사(Shallow Copy)**: `.copy()`나 `[:]`는 리스트 자체는 새로 만들지만, 그 안에 리스트 같은 가변 객체가 들어있다면 그 내부 객체까지는 복사하지 않고 여전히 참조를 공유합니다. (완전히 독립적인 복사가 필요하면 `copy.deepcopy()`를 쓰는데, 이는 나중에 필요할 때 다룰 심화 내용입니다.)

```python
list1 = [1, 2, 3]
list2 = list1          # 참조 — 같은 데이터를 가리킴
list2.append(4)
print(list1)            # [1, 2, 3, 4] — list1도 바뀜

list3 = list1.copy()   # 복사 — 별도의 데이터
list3.append(5)
print(list1)            # 영향 없음
```

---

### 튜플 언패킹 (Tuple Unpacking)

**정의**: 튜플 안에 담긴 값들을 여러 변수에 한 번에 나눠서 대입하는 것입니다.

**CS적 의의**:

- **구조 분해(Destructuring)의 Python식 구현**: 다른 언어에서 "구조 분해 할당"이라고 부르는 개념과 동일합니다. JS의 `const [x, y] = arr;`가 대응됩니다.
- **다중 반환값의 기반**: Python 함수가 "여러 값을 반환하는 것처럼 보이는" 문법(`return a, b`)은 사실 튜플 하나를 반환하는 것이고, 이를 받는 쪽에서 언패킹하는 구조입니다.

```python
point = (3, 5)
x, y = point   # 언패킹
```

---

### 불변 객체의 "수정" 우회법

**정의**: 튜플처럼 불변인 자료형도, 리스트로 변환해서 수정한 뒤 다시 그 자료형으로 되돌리면 마치 수정한 것처럼 결과를 얻을 수 있습니다.

**CS적 의의**:

- **제자리 수정이 아니라 새 객체 생성**: `tuple(list(point))`로 얻은 결과는 기존 튜플을 고친 게 아니라 완전히 새로운 튜플입니다. 변수 이름이 새 객체를 다시 가리키게 될 뿐, 불변성이라는 원칙 자체는 깨지지 않습니다.
- **타입 변환(Type Conversion)의 실용적 활용**: `list()`와 `tuple()`은 자료형을 서로 변환해주는 함수인데, 이 성질을 이용해 "불변 자료형을 잠깐 가변 자료형으로 바꿔서 작업한 뒤 다시 불변으로 되돌리는" 우회 패턴이 가능해집니다.

```python
point = (3, 5)
point_list = list(point)   # 튜플 → 리스트 (가변으로 전환)
point_list[0] = 10
point = tuple(point_list)   # 리스트 → 튜플 (다시 불변으로 전환, 사실상 새 튜플)
```

## Key Questions

**Q1.** 리스트는 가변인데 튜플은 왜 굳이 불변으로 만들었나요?

> **A.** 안전성과 성능, 두 가지 이유입니다.
> 좌표값처럼 "절대 바뀌면 안 되는 데이터"를 튜플로 만들어두면, 실수로 값을 바꾸는 코드를 작성해도 즉시 에러가 발생해 버그를 미리 잡을 수 있습니다.
> 또한 불변이라는 게 보장되면 Python 내부적으로 메모리를 더 가볍게 관리할 수 있어서, 같은 데이터라면 튜플이 리스트보다 처리 속도가 빠릅니다.

---

**Q2.** `list2 = list1`과 `list2 = list1.copy()`는 뭐가 다른가요?

> **A.** 전자는 참조를 공유하고, 후자는 완전히 별도의 데이터를 만듭니다.
> `list2 = list1`은 list2라는 이름표가 list1과 같은 실제 데이터를 가리키게 만드는 것이라, 하나를 바꾸면 둘 다 바뀐 것처럼 보입니다.
> `list2 = list1.copy()`는 list1의 내용을 그대로 옮겨 담은 새로운 데이터를 만들기 때문에, 이후로는 서로 독립적으로 움직입니다.

---

**Q3.** 슬라이싱에서 끝 인덱스가 왜 포함되지 않게 설계됐나요?

> **A.** `range()`가 끝값을 포함하지 않는 것과 같은 이유입니다.
> `numbers[a:b]`로 잘라낸 구간의 길이가 항상 `b - a`가 되도록 설계하기 위해서입니다.
> 만약 끝 인덱스를 포함했다면 길이를 계산할 때마다 1을 더하거나 빼야 하는 번거로움이 생겼을 것입니다.

---

**Q4.** 튜플이 불변인데, `list()`로 바꿔서 수정하고 다시 `tuple()`로 되돌리면 결국 수정한 거 아닌가요?

> **A.** 결과만 보면 수정한 것처럼 보이지만, 내부적으로는 완전히 다른 과정입니다.
> 기존 튜플은 그대로 메모리에 남아있고, `list()`가 그 값을 복사해 새로운 리스트를 만들며, 수정 후 `tuple()`이 또 다른 새로운 튜플을 만듭니다.
> 변수 이름(`point`)이 마지막에 새로 만들어진 튜플을 가리키도록 재할당된 것뿐이라, "기존 튜플 객체 자체가 바뀐 것"은 아닙니다.
> 이 구분이 중요한 이유는, 만약 다른 변수가 원래 튜플을 참조하고 있었다면 그 변수는 여전히 원래 값을 그대로 유지하기 때문입니다.
