## 1. 반복문 기초

### 반복문 (Loop)

**정의**: 같은 코드를 여러 번 반복해서 실행하게 해주는 제어 구조입니다.

**CS적 의의**:

- **제어 흐름(Control Flow)의 한 종류**: 조건문이 "실행 경로를 갈라지게" 한다면, 반복문은 "같은 경로를 되풀이해서" 실행하게 만드는 제어 흐름입니다.
- **코드 중복 제거**: 반복문이 없다면 같은 동작을 100번 실행하기 위해 같은 코드를 100번 써야 합니다. 반복문은 이런 중복을 없애고, 반복 횟수를 변수로 다룰 수 있게 해줍니다.

---

### 이터레이션 (Iteration)

**정의**: 반복문의 몸체(body)가 한 번 실행되는 단위입니다. "반복문이 5번 실행됐다"는 "이터레이션이 5번 일어났다"와 같은 뜻입니다.

**CS적 의의**:

- **반복(Loop)과 이터레이션의 관계**: 반복문(Loop)은 전체 반복 구조 자체를 가리키고, 이터레이션(Iteration)은 그 안에서 한 번 도는 개별 단위를 가리킵니다. "루프 안에서 3번째 이터레이션에 에러가 났다"처럼 특정 시점을 정확히 지칭할 때 씁니다.
- **이터러블(Iterable)과 이터레이터(Iterator)**: Python에서 `for` 문으로 순회할 수 있는 대상(리스트, 문자열 등)을 이터러블이라 하고, 실제로 값을 하나씩 꺼내주는 내부 객체를 이터레이터라고 부릅니다. 지금은 "순회 가능한 대상이 있고, 그걸 하나씩 꺼내는 장치가 따로 있다" 정도만 알아두면 충분합니다.

## 2. for문과 관련 도구

### for문 (For Loop)

**정의**: 리스트, 문자열처럼 여러 값이 모인 대상을 순서대로 하나씩 꺼내며 반복하는 문법입니다.

**CS적 의의**:

- **정해진 컬렉션을 순회(Traversal)**: for문은 "몇 번 반복할지 미리 알 수 있는" 상황에 적합합니다. 리스트의 길이만큼, 또는 range()로 지정한 횟수만큼 정확히 반복합니다.
- **JS `for...of`와의 유사성**: Python의 `for...in`은 JS의 `for...of`와 개념이 거의 같습니다. 다만 JS의 전통적인 3단계 for문(초기값, 조건, 증감)은 Python에 없습니다.

```python
for fruit in ["사과", "바나나"]:
    print(fruit)
```

---

### range()

**정의**: 연속된 숫자들을 만들어내는 함수로, `for`문과 함께 정해진 횟수만큼 반복할 때 주로 씁니다.

**CS적 의의**:

- **끝값 미포함(Exclusive End)**: `range(5)`는 0부터 4까지 다섯 개를 만들고, `range(2, 10, 2)`처럼 시작·끝·증가폭을 지정할 수도 있습니다. "끝값은 포함하지 않는다"는 규칙이 초보자가 가장 자주 헷갈리는 부분입니다.
- **지연 평가(Lazy Evaluation)**: `range()`는 모든 숫자를 미리 리스트로 만들어두는 게 아니라, 필요할 때마다 다음 숫자를 계산해서 내놓습니다. 그래서 `range(1000000)`처럼 큰 범위를 만들어도 메모리를 많이 차지하지 않습니다.

```python
for i in range(2, 10, 2):
    print(i)   # 2, 4, 6, 8
```

---

### enumerate()

**정의**: 반복문에서 값과 함께 순번(인덱스)을 동시에 꺼내주는 함수입니다.

**CS적 의의**:

- **인덱스 관리의 자동화**: `enumerate()`가 없다면 순번을 세는 변수를 직접 만들고 매 이터레이션마다 증가시켜야 합니다. `enumerate()`는 이 반복적인 패턴을 언어 차원에서 자동화해줍니다.
- **튜플 언패킹(Tuple Unpacking)**: `enumerate()`는 각 이터레이션마다 (순번, 값) 형태의 튜플을 반환하고, `for index, value in enumerate(...)`처럼 두 변수로 동시에 받는 것을 튜플 언패킹이라고 합니다. (튜플은 다음에 배울 자료구조)

```python
for index, fruit in enumerate(["사과", "바나나"]):
    print(index, fruit)
```

---

### 중첩 반복문 (Nested Loop)

**정의**: 반복문 안에 또 다른 반복문을 넣어, 바깥 반복이 한 번 돌 때마다 안쪽 반복이 전체를 도는 구조입니다.

**CS적 의의**:

- **시간 복잡도와의 관계**: 중첩 반복문은 바깥 반복 횟수 × 안쪽 반복 횟수만큼 전체 이터레이션이 발생합니다. 예를 들어 바깥 10번, 안쪽 10번이면 전체 100번(10×10) 실행됩니다. 이는 나중에 배울 "시간 복잡도(연산이 데이터 양에 따라 얼마나 느려지는가)" 개념과 직결됩니다.
- **행렬/표 데이터 처리**: 2차원 데이터(행과 열)를 다룰 때 바깥 반복이 행을, 안쪽 반복이 열을 순회하는 형태로 자연스럽게 대응됩니다.

## 3. while문과 흐름 제어

### while문 (While Loop)

**정의**: 정해진 횟수가 아니라, 조건이 참인 동안 계속 반복하는 문법입니다.

**CS적 의의**:

- **조건 기반 반복**: for문이 "정해진 대상을 순회"한다면, while문은 "조건이 유지되는 동안" 반복합니다. 반복 횟수를 미리 알 수 없는 상황(사용자 입력 대기, 재시도 로직 등)에 적합합니다.
- **JS와의 유사성**: while문은 Python과 JS 문법 구조가 거의 동일합니다(괄호·중괄호 유무만 다름).

```python
count = 0
while count < 3:
    print(count)
    count += 1
```

---

### 무한 루프 (Infinite Loop)

**정의**: 반복을 멈추는 조건이 없거나 잘못되어, 반복문이 영원히 끝나지 않는 상태입니다.

**CS적 의의**:

- **종료 조건(Termination Condition)의 중요성**: while문은 조건이 언젠가 거짓이 될 수 있어야 정상적으로 종료됩니다. `count += 1`처럼 조건을 변화시키는 코드를 빠뜨리면 무한 루프에 빠집니다.
- **의도적인 무한 루프**: `while True:`처럼 일부러 무한 루프를 만들고, 내부에서 `break`로 직접 종료 시점을 정하는 패턴도 실무에서 자주 씁니다. 이 경우는 버그가 아니라 의도된 설계입니다.

```python
while True:
    answer = input("종료하려면 q 입력: ")
    if answer == "q":
        break
```

---

### break / continue

**정의**: `break`는 반복문을 즉시 종료하고, `continue`는 현재 이터레이션만 건너뛰고 다음 이터레이션으로 넘어가게 하는 키워드입니다.

**CS적 의의**:

- **흐름 제어 키워드(Control Flow Keyword)**: 두 키워드 모두 반복문의 정상적인 순서를 벗어나게 만드는 예외적인 흐름 제어입니다. `break`는 반복문 자체를 빠져나가고, `continue`는 반복문 안에 머무르면서 현재 이터레이션만 건너뜁니다.
- **불필요한 연산 방지**: 원하는 값을 찾은 후 `break`로 즉시 멈추면, 남은 이터레이션을 실행하지 않아도 되므로 자원(시간)을 절약할 수 있습니다.

```python
for i in range(10):
    if i == 3:
        break       # 반복문 전체 종료
for i in range(5):
    if i == 2:
        continue    # 이번 이터레이션만 건너뜀
```

---

### for-else / while-else

**정의**: 반복문에 `else`를 붙이면, 그 반복문이 `break` 없이 끝까지 실행됐을 때만 실행되는 블록입니다.

**CS적 의의**:

- **JS에 없는 Python 고유 문법**: JS를 비롯한 대부분의 언어에는 이 문법이 없어서, Python을 처음 접할 때 가장 헷갈리는 구조 중 하나로 꼽힙니다.
- **if의 else와 이름은 같지만 의미가 다름**: if-else의 else는 "조건이 거짓일 때" 실행되지만, for-else/while-else의 else는 "반복문이 break 없이 정상적으로 끝났을 때" 실행됩니다. 조건의 참/거짓과는 무관합니다.

```python
for n in [1, 3, 5]:
    if n % 2 == 0:
        break
else:
    print("짝수가 없습니다")   # break 없이 끝까지 돌았으므로 실행됨
```

## Key Questions

**Q1.** "반복문(Loop)"과 "이터레이션(Iteration)"은 같은 말 아닌가요?

> **A.** 다릅니다. 반복문은 "구조 전체"를 가리키고, 이터레이션은 그 안에서 "한 번 도는 단위"를 가리킵니다.
> 예를 들어 `for i in range(5):`라는 반복문은 하나지만, 그 안에서 이터레이션은 5번 일어납니다.
> "이 반복문에서 에러가 났다"보다 "이 반복문의 3번째 이터레이션에서 에러가 났다"가 훨씬 정확한 표현이 되는 이유입니다.

---

**Q2.** `range()`는 왜 끝값을 포함하지 않게 설계됐나요?

> **A.** 시작값과 개수를 계산하기 편하게 만들기 위해서입니다.
> `range(0, n)`이라고 쓰면 정확히 n개의 숫자(0부터 n-1까지)가 나온다는 규칙이 항상 성립합니다.
> 만약 끝값을 포함했다면 `range(0, n)`이 n+1개를 만들게 되어, "몇 개가 나오는지" 계산할 때마다 1을 빼거나 더해야 하는 번거로움이 생깁니다.

---

**Q3.** `for-else`는 실무에서도 많이 쓰이나요?

> **A.** 문법 자체는 유효하지만, 실무에서는 오히려 잘 안 쓰이는 편입니다.
> 이름이 `else`인데 의미가 "break 없이 끝까지 돌았을 때"라서, 코드를 읽는 사람이 오해하기 쉽기 때문입니다.
> 그래서 많은 실무 코드는 `for-else` 대신, `found = False`처럼 별도의 플래그 변수를 두고 `break` 시점에 `found = True`로 바꾼 뒤, 반복문이 끝난 후 `if found:`로 확인하는 방식을 더 선호합니다.
