## 1. 함수 기초

### 함수 (Function)

**정의**: 특정 작업을 하나로 묶어 이름을 붙여두고, 필요할 때마다 호출해서 재사용할 수 있게 만든 코드 단위입니다.

**CS적 의의**:

- **추상화(Abstraction)**: 함수는 "어떻게 동작하는지"를 감추고 "무엇을 하는지"만 노출합니다. 함수를 쓰는 입장에서는 내부 구현을 몰라도 이름과 사용법만 알면 됩니다.
- **재사용성(Reusability)과 DRY 원칙**: 같은 로직을 여러 번 반복해서 쓰지 않고 한 곳에 모아두는 것을 "Don't Repeat Yourself(DRY)" 원칙이라 부르며, 함수는 이 원칙을 실현하는 가장 기본적인 도구입니다.

---

### 매개변수 (Parameter) vs 인자 (Argument)

**정의**: 매개변수는 함수를 정의할 때 선언하는 입력값의 이름이고, 인자는 함수를 호출할 때 실제로 전달하는 값입니다.

**CS적 의의**:

- **정의 시점 vs 호출 시점의 구분**: 이 둘을 구분하는 이유는 "함수가 무엇을 받을 수 있는가(정의)"와 "실제로 무엇을 받았는가(호출)"가 서로 다른 시점의 개념이기 때문입니다. 에러 메시지에서도 이 둘을 구분해서 설명합니다.

---

### 독스트링 (Docstring)

**정의**: 함수 바로 아래에 작성하는 공식 설명 문자열로, `함수.__doc__`이나 `help()`로 조회할 수 있습니다.

**CS적 의의**:

- **자기 문서화(Self-documenting Code)**: 코드 자체에 문서를 내장시켜, 별도의 문서 없이도 코드만으로 사용법을 파악할 수 있게 하는 관행입니다. 에디터의 자동완성 툴팁도 이 정보를 활용합니다.

## 2. 인자 전달 방식

### 기본값 매개변수와 가변 기본값 함정

**정의**: 매개변수에 기본값을 지정해두면 인자를 생략할 수 있습니다. 다만 리스트·딕셔너리 같은 가변 자료형을 기본값으로 쓰면 예상과 다르게 동작하는 함정이 있습니다.

**CS적 의의**:

- **평가 시점(Evaluation Time)의 문제**: 기본값은 함수가 호출될 때마다 새로 계산되는 게 아니라, 함수가 정의되는 시점에 단 한 번만 평가되어 메모리에 고정됩니다. 가변 객체를 기본값으로 쓰면, 모든 호출이 이 하나의 객체를 공유하게 되어 예상 밖의 부작용(Side Effect)이 생깁니다.
- **부작용(Side Effect)**: 함수가 자신의 입력 외의 상태(여기서는 공유된 기본값 리스트)를 변경하는 것을 부작용이라 부릅니다. 부작용이 있는 함수는 호출 순서나 횟수에 따라 결과가 달라질 수 있어 예측하기 어렵습니다.

```python
def add_item(item, cart=[]):   # cart는 함수 정의 시 딱 한 번만 생성됨
    cart.append(item)
    return cart
```

---

### 위치 인자 vs 키워드 인자

**정의**: 위치 인자는 순서에 따라 매개변수에 매칭되고, 키워드 인자는 매개변수 이름을 지정해서 순서와 무관하게 전달합니다.

**CS적 의의**:

- **가독성과 명시성**: 키워드 인자는 호출 코드만 보고도 "이 값이 어떤 의미인지" 바로 알 수 있게 해, 매개변수가 많은 함수일수록 실수를 줄여줍니다.

---

### `*args`와 `**kwargs`

**정의**: 정의하는 함수 쪽에서 개수가 정해지지 않은 위치 인자를 튜플로(`*args`), 키워드 인자를 딕셔너리로(`**kwargs`) 모아서 받는 문법입니다.

**CS적 의의**:

- **가변 인자(Variadic Argument)**: 매개변수 개수가 고정되지 않은 함수를 만들 수 있게 해주는 기능을 가변 인자라고 부르며, 여러 언어에 공통적으로 존재하는 개념입니다(JS의 Rest Parameter 등).
- **관례적 이름**: `args`, `kwargs`라는 이름 자체는 문법이 아니라 관례이며, 실제 문법 요소는 `*`와 `**` 기호입니다.

---

### 인자 언패킹 (Argument Unpacking)

**정의**: 함수를 호출하는 쪽에서, 이미 묶여있는 리스트나 딕셔너리를 `*`/`**`로 풀어서 개별 인자로 전달하는 것입니다.

**CS적 의의**:

- **`*`/`**`의 이중적 의미**: 같은 기호(`*`, `**`)가 함수를 정의할 때는 "여러 값을 하나로 모으는" 역할을, 함수를 호출할 때는 정반대로 "하나로 묶인 것을 여러 값으로 푸는" 역할을 합니다. 문맥(정의 vs 호출)에 따라 정반대 의미가 된다는 점이 중요합니다.

```python
add(*[1, 2, 3])       # 리스트를 낱개 위치 인자로 풀어서 전달
add(**{"a": 1, "b": 2})   # 딕셔너리를 낱개 키워드 인자로 풀어서 전달
```

## 3. 반환값과 스코프

### return vs print

**정의**: `print()`는 값을 화면에 출력만 하고, `return`은 값을 함수 밖으로 반환해서 다른 코드가 그 값을 사용할 수 있게 합니다.

**CS적 의의**:

- **부수 효과(Side Effect) vs 반환값(Return Value)**: `print()`는 화면에 출력하는 부수 효과만 있을 뿐 함수의 반환값에는 영향을 주지 않습니다. `return`이 없는 함수는 항상 `None`을 반환합니다. 이 구분은 함수형 프로그래밍에서 "순수 함수(입력에 대해 항상 같은 출력을 반환하고, 부수 효과가 없는 함수)" 개념과도 연결됩니다.

---

### 스코프 (Scope), UnboundLocalError, global

**정의**: 스코프는 변수가 유효한 범위를 뜻합니다. 함수 안에서 선언된 변수는 지역 스코프에 속해 함수 밖에서 접근할 수 없고, `global` 키워드로 전역 스코프의 변수를 함수 안에서 직접 수정할 수 있습니다.

**CS적 의의**:

- **렉시컬 스코프(Lexical Scope)**: Python은 변수가 코드상 어디에 "쓰여 있는가"를 기준으로 스코프를 결정합니다. 함수 안에 변수를 대입하는 코드가 있으면, 그 변수는 함수 전체에서 지역변수로 취급됩니다.
- **UnboundLocalError**: 함수 안에서 전역변수와 같은 이름에 대입하는 코드가 있으면, Python은 그 함수 전체에서 해당 이름을 지역변수로 미리 정해둡니다. 그래서 대입보다 먼저 그 이름을 읽으려 하면, "아직 값이 할당되지 않은 지역변수를 읽으려 했다"는 의미의 `UnboundLocalError`가 발생합니다.
- **전역 상태(Global State)의 위험성**: 여러 함수가 같은 전역변수를 자유롭게 수정할 수 있게 되면, 특정 시점에 그 변수의 값이 무엇인지 코드 전체를 추적해야만 알 수 있게 됩니다. 이런 이유로 실무에서는 `global`보다 `return`으로 상태를 명시적으로 주고받는 방식을 선호합니다.

```python
count = 0
def increase():
    global count   # 전역 스코프의 count를 가리킨다고 선언
    count += 1
```

## 4. 함수형 개념

### 일급 함수 (First-class Function)

**정의**: 함수를 숫자나 문자열 같은 일반 값처럼 변수에 저장하거나, 다른 함수의 인자로 넘기거나, 반환값으로 쓸 수 있는 성질입니다.

**CS적 의의**:

- **함수 참조 vs 함수 호출**: `함수이름`(괄호 없음)은 함수 자체(값)를 가리키고, `함수이름()`(괄호 있음)은 함수를 실행한 결과를 가리킵니다. 일급 함수 개념이 있기에 `sort(key=함수이름)`처럼 함수를 다른 함수의 인자로 그대로 넘기는 것이 가능합니다.
- **고차 함수(Higher-order Function)**: 함수를 인자로 받거나 함수를 반환하는 함수를 고차 함수라고 부릅니다. `sort()`가 `key` 인자로 함수를 받는 것이 대표적인 예시이며, Python 심화 단계에서 배울 데코레이터도 고차 함수의 일종입니다.

---

### 람다 (Lambda) — 익명 함수 (Anonymous Function)

**정의**: 이름 없이 `lambda 매개변수: 표현식` 형태로 만드는, 표현식 하나짜리 짧은 함수입니다.

**CS적 의의**:

- **익명 함수(Anonymous Function)**: 이름을 붙이지 않고 그 자리에서 바로 정의해서 쓰는 함수를 통칭하는 개념으로, 대부분의 현대 언어에 존재합니다(JS의 화살표 함수 등).
- **표현식(Expression) 제한**: 람다는 "문(statement)"이 아니라 "표현식(expression)" 하나만 담을 수 있어서, `if`나 `for` 같은 제어문을 넣을 수 없습니다. 이 제약 때문에 간단한 로직에만 한정적으로 쓰입니다.

```python
users.sort(key=lambda user: user["age"])   # 람다를 고차 함수의 인자로 전달
```

## Key Questions

**Q1.** `cart=[]`를 기본값으로 쓰면 왜 문제가 되나요?

> **A.** 기본값이 함수 정의 시점에 딱 한 번만 만들어지고, 이후 모든 호출이 그 객체 하나를 계속 공유하기 때문입니다.
> 리스트는 가변 자료형이라 한 번 만들어진 뒤에도 내용이 계속 바뀔 수 있는데, 기본값으로 쓰인 리스트는 함수가 여러 번 호출되어도 새로 만들어지지 않고 재사용됩니다.
> 그래서 이전 호출에서 추가한 값이 다음 호출까지 남아있는 부작용이 생깁니다. `None`을 기본값으로 두고 함수 안에서 새로 만드는 방식으로 이 문제를 피할 수 있습니다.

---

**Q2.** `*`와 `**`는 함수 정의와 호출에서 왜 반대 역할을 하나요?

> **A.** 같은 기호지만 "무엇을 하려는 상황인가"에 따라 필요한 동작이 정반대이기 때문입니다.
> 함수를 정의할 때는 "몇 개가 들어올지 모르는 값들을 하나로 모아둘" 필요가 있고, 함수를 호출할 때는 반대로 "이미 하나로 묶여있는 데이터를 낱개로 풀어서 전달할" 필요가 있습니다.
> Python은 이 두 가지 상황 모두에 같은 기호를 재사용하도록 설계했고, 문맥(정의부냐 호출부냐)으로 의미가 자동으로 정해집니다.

---

**Q3.** `global`을 쓰면 되는데 왜 실무에서는 `return`을 더 선호하나요?

> **A.** 코드의 예측 가능성 때문입니다.
> `return`을 쓰는 함수는 입력(매개변수)만 보고도 출력을 예상할 수 있고, 함수 밖의 다른 상태에 영향을 주지 않습니다.
> 반면 `global`로 전역변수를 직접 수정하면, 그 변수의 값이 코드의 어느 시점에 무엇인지 알려면 전역변수를 건드리는 모든 함수를 다 찾아봐야 합니다.
> 프로그램이 커질수록 이런 추적 비용이 기하급수적으로 늘어나기 때문에, 대부분의 실무 코드는 `global` 대신 함수가 값을 반환하고 호출한 쪽에서 다시 변수에 대입하는 방식을 씁니다.
