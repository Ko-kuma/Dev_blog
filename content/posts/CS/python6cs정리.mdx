## 1. 딕셔너리 기초

### 딕셔너리 (Dictionary)

**정의**: 값을 순서(인덱스)가 아니라 key라는 이름표로 찾아서 관리하는 자료구조입니다.

**CS적 의의**:

- **연관 배열(Associative Array)**: 컴퓨터과학 전반에서 이런 key-value 구조를 "연관 배열"이라고 부릅니다. 언어마다 이름이 다른데(Python: dict, JS: Object/Map, Java: HashMap), 개념 자체는 동일합니다.
- **동적 key 접근**: 변수는 이름이 코드에 고정되어 있지만, 딕셔너리는 key 자체를 실행 중에 변수처럼 다룰 수 있습니다(`person[field]`). 이는 변수로는 표현할 수 없는, 딕셔너리만의 근본적인 특징입니다.

```python
person = {"name": "혜진", "age": 30}
```

---

### KeyError

**정의**: 딕셔너리에 존재하지 않는 key로 값을 꺼내려 할 때 발생하는 에러입니다.

**CS적 의의**:

- **명시적 실패(Explicit Failure)**: 없는 key에 접근했을 때 조용히 `None`을 반환하는 대신 에러를 발생시키는 것은, 문제를 코드 작성 단계에서 바로 드러내기 위한 설계입니다.
- **안전한 접근과의 트레이드오프**: `get()` 메서드는 `KeyError` 대신 기본값을 반환하도록 만들어, "없을 수도 있는 데이터"를 다룰 때 프로그램이 멈추지 않게 해줍니다. 다만 이 경우 key가 실제로 없었다는 사실이 조용히 넘어갈 수 있다는 트레이드오프가 있습니다.

```python
person.get("phone", "없음")   # KeyError 대신 기본값 반환
```

## 2. 딕셔너리 조작

### dict.pop(key)

**정의**: 지정한 key의 값을 딕셔너리에서 꺼내면서 동시에 삭제하는 메서드입니다.

**CS적 의의**:

- **리스트 pop()과의 대응**: 리스트의 `pop()`이 인덱스를 기준으로 삭제·반환하듯, 딕셔너리의 `pop()`은 key를 기준으로 동일한 동작을 합니다. "값을 꺼내면서 동시에 제거한다"는 패턴 자체가 여러 자료구조에서 공통으로 나타나는 인터페이스입니다.

---

### update()

**정의**: 다른 딕셔너리의 내용을 현재 딕셔너리에 병합하는 메서드입니다. key가 겹치면 새 값으로 덮어씁니다.

**CS적 의의**:

- **병합(Merge) 연산**: 두 데이터 집합을 하나로 합치는 연산은 설정값 병합, 여러 소스에서 가져온 데이터 결합 등에서 자주 필요합니다. `update()`는 "기존 값 유지 + 새 값으로 덮어쓰기"라는 병합 규칙을 구현한 것입니다.

```python
default_settings.update(user_settings)   # user_settings가 우선권을 가짐
```

---

### 중첩 딕셔너리 (Nested Dictionary)

**정의**: 딕셔너리의 값(value) 자리에 또 다른 딕셔너리가 들어있는 구조입니다.

**CS적 의의**:

- **트리 구조(Tree Structure)와의 관계**: 중첩 딕셔너리는 사실상 트리 형태의 데이터입니다. JSON으로 표현되는 대부분의 실제 API 데이터가 이런 중첩 구조를 가지며, 이는 계층적인 정보(회원 정보 안에 주소 정보가 포함되는 등)를 표현하기에 자연스러운 방식입니다.

```python
user["address"]["city"]   # key를 연쇄적으로 사용해 중첩 구조에 접근
```

## 3. 해시 가능성과 key 제약

### 해시 가능(Hashable)

**정의**: 어떤 값이 변하지 않는다는 것이 보장되어, 딕셔너리의 key나 셋의 원소로 사용할 수 있는 성질입니다.

**CS적 의의**:

- **해시 테이블(Hash Table)의 원리**: 딕셔너리와 셋은 내부적으로 해시 테이블이라는 자료구조로 구현되어, key를 통해 값을 매우 빠르게 찾을 수 있습니다. 이 구조가 정상적으로 동작하려면 key의 값이 절대 바뀌지 않아야 합니다.
- **가변 자료형이 key가 될 수 없는 이유**: 리스트처럼 가변(Mutable) 자료형은 값이 언제든 바뀔 수 있어서, 한 번 계산해둔 "찾아가는 길(해시값)"이 무효화될 위험이 있습니다. 그래서 Python은 리스트를 key나 셋의 원소로 쓰는 것을 아예 금지합니다. 반면 튜플, 문자열, 숫자처럼 불변(Immutable) 자료형은 해시 가능합니다.

```python
{(1, 2): "좌표"}   # 튜플(불변) → 가능
{[1, 2]: "좌표"}   # 리스트(가변) → TypeError
```

---

### `{ }` 리터럴의 중의성

**정의**: 딕셔너리와 셋이 똑같이 중괄호 `{ }`를 사용하기 때문에, Python은 내용물에 콜론(`:`)이 있는지로 둘을 구분합니다.

**CS적 의의**:

- **문법적 모호성(Syntactic Ambiguity) 해소 규칙**: 같은 기호가 여러 의미로 쓰일 수 있을 때, 파서(코드를 해석하는 프로그램)는 정해진 규칙으로 의미를 결정해야 합니다. Python은 "콜론의 유무"라는 규칙으로 이 모호성을 해소합니다.
- **기본값 규칙**: 콜론이 있는지 확인할 대상 자체가 없는 빈 `{ }`는 딕셔너리로 취급됩니다. 이는 "예외적인 경우 무엇을 기본값으로 삼을지"를 언어 설계자가 미리 정해둔 규칙입니다. 빈 셋이 필요하면 `set()`을 명시적으로 써야 합니다.

## 4. 셋과 집합 연산

### 셋 (Set)

**정의**: 순서가 없고 중복을 허용하지 않는 자료구조입니다.

**CS적 의의**:

- **수학적 집합(Set)의 구현**: 프로그래밍의 셋은 수학 시간에 배우는 집합 개념을 그대로 코드로 옮긴 것입니다. 그래서 교집합, 합집합 같은 집합 연산을 기호 하나로 바로 계산할 수 있습니다.
- **빠른 존재 확인(Membership Test)**: 셋은 해시 테이블로 구현되어 있어, "이 값이 셋 안에 있는가"를 리스트보다 훨씬 빠르게 확인할 수 있습니다. 그래서 중복 확인이나 존재 여부 체크가 잦은 상황에 적합합니다.

---

### 집합 연산 (Set Operation)

**정의**: 두 셋 사이의 관계를 계산하는 연산으로, 교집합(`&`), 합집합(`|`), 차집합(`-`), 대칭차집합(`^`)이 있습니다.

**CS적 의의**:

- **각 연산의 의미**: 교집합은 "둘 다에 속하는 값", 합집합은 "둘 중 하나라도에 속하는 값", 차집합은 "한쪽에만 있고 다른 쪽엔 없는 값", 대칭차집합은 "겹치지 않는 값들만"을 의미합니다.
- **실무적 활용**: 두 그룹의 공통 권한 확인(교집합), 여러 목록을 하나로 합치기(합집합), 권한 차이 확인(차집합)처럼, 데이터 간의 "관계"를 다루는 로직에서 반복적으로 등장하는 패턴입니다.

```python
a & b   # 교집합
a | b   # 합집합
a - b   # 차집합
a ^ b   # 대칭차집합
```

---

### frozenset

**정의**: 셋의 불변(Immutable) 버전으로, 한 번 만들면 원소를 추가·삭제할 수 없습니다.

**CS적 의의**:

- **불변성과 해시 가능성의 연결**: 셋 자체는 가변 자료형이라 해시 가능하지 않으므로, 다른 셋이나 딕셔너리의 key 안에 넣을 수 없습니다. `frozenset`은 이 문제를 해결하기 위해 만들어진, 셋의 불변 버전입니다. 리스트-튜플의 관계와 정확히 같은 구조입니다.

```python
{frozenset(["a", "b"]), frozenset(["c", "d"])}   # 셋 안에 셋을 담을 때는 frozenset 필요
```

## Key Questions

**Q1.** 딕셔너리와 셋은 왜 리스트를 key/원소로 허용하지 않나요?

> **A.** 딕셔너리와 셋 내부는 해시 테이블로 구현되어 있어, key의 값을 근거로 "어디에 저장할지"를 미리 계산해두기 때문입니다.
> 만약 key로 쓰인 리스트의 내용이 나중에 바뀐다면, 미리 계산해둔 위치와 실제 값이 어긋나서 데이터를 찾을 수 없게 됩니다.
> 그래서 Python은 아예 처음부터 가변 자료형(리스트, 셋)을 key/원소로 쓰는 것을 금지하고, 불변 자료형(튜플, frozenset, 문자열, 숫자)만 허용합니다.

---

**Q2.** 딕셔너리와 셋 모두 `{ }`를 쓰는데 헷갈리지 않나요?

> **A.** Python은 콜론(`:`)의 유무로 구분합니다.
> `{"a": 1}`처럼 콜론이 있으면 딕셔너리, `{1, 2, 3}`처럼 콜론 없이 값만 나열하면 셋으로 해석됩니다.
> 다만 빈 `{ }`는 콜론이 있는지 확인할 대상 자체가 없어서, Python은 이를 기본값인 딕셔너리로 처리합니다. 빈 셋이 필요하면 `set()`이라고 명시적으로 써야 합니다.

---

**Q3.** 셋의 집합 연산(`&`, `|`, `-`, `^`)은 실무에서 정말 자주 쓰이나요?

> **A.** 두 그룹 사이의 관계를 다뤄야 하는 상황에서 자주 쓰입니다.
> 예를 들어 사용자가 가진 권한과 특정 기능에 필요한 권한을 각각 셋으로 만들어두면, 교집합(`&`)으로 "충족된 권한", 차집합(`-`)으로 "부족한 권한"을 한 줄로 바로 계산할 수 있습니다.
> 리스트로 이런 비교를 하려면 반복문과 조건문을 직접 짜야 하지만, 셋 연산은 이를 훨씬 간결하게 표현합니다.
