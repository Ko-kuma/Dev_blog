# HTML / CSS 기초

## 1. HTML?

HTML은 **HyperText Markup Language**의 약자입니다.
웹 페이지의 **뼈대(구조)**를 만드는 언어이며,
글자, 이미지, 버튼, 입력창 같은 콘텐츠의 구조를 정의합니다.

```
HTML  →  건물의 뼈대 (기둥, 벽, 문)
CSS   →  인테리어 (색상, 크기, 배치)
JS(JavaScript) →  전기 (움직임, 상호작용)
이 3개의 기술을 웹의 표준 기술이라고 부릅니다.
```

HTML은 **태그(Tag)**로 구성됩니다.
태그는 꺾쇠 괄호로 감싼 키워드입니다.

```html
<!-- 태그의 기본 형태 -->
<태그이름>내용</태그이름>

<!-- 예시 -->
<h1>제목입니다</h1>
<p>단락입니다</p>
<a href="https://google.com">구글로 이동</a>
```

| 용어 | 설명 |
|---|---|
| 태그 (Tag) | HTML의 기본 단위. `<h1>`, `<p>`, `<div>` 같은 것들 |
| 열린 태그 | 시작 태그. 예: `<p>` |
| 닫힌 태그 | 종료 태그. 예: `</p>` |
| 속성 (Attribute) | 태그에 추가 정보를 줌. 예: `href`, `src`, `class` |
| 요소 (Element) | 열린 태그 + 내용 + 닫힌 태그 전체 |

## 2. HTML 기본 구조

모든 HTML 파일은 아래 구조를 기본으로 합니다.

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>페이지 제목</title>
  </head>
  <body>
    <h1>안녕하세요!</h1>
    <p>첫 번째 웹 페이지입니다.</p>
  </body>
</html>
```

| 태그 | 역할 |
|---|---|
| `<!DOCTYPE html>` | 이 파일이 HTML5 문서임을 브라우저에 알림 |
| `<html>` | HTML 문서의 최상위 태그. 모든 내용을 감쌈 |
| `<head>` | 브라우저에 보이지 않는 메타 정보 영역 |
| `<meta charset="UTF-8">` | 한글 등 유니코드 문자를 올바르게 표시 |
| `<meta name="viewport">` | 모바일 반응형을 위한 뷰포트 설정 |
| `<title>` | 브라우저 탭에 표시되는 페이지 제목 |
| `<body>` | 실제로 화면에 보이는 콘텐츠 영역 |

## 3. 핵심 HTML 태그

### 3-1. 텍스트 태그

```html
<!-- 제목 태그 (h1이 가장 크고 h6이 가장 작음) -->
<h1>가장 큰 제목</h1>
<h2>두 번째 제목</h2>
<h3>세 번째 제목</h3>

<!-- 단락 -->
<p>이것은 단락입니다.</p>

<!-- 줄바꿈 (닫는 태그 없음) -->
<br>

<!-- 굵게, 기울임 -->
<strong>굵게</strong>
<em>기울임</em>

<!-- 구분선 -->
<hr>
```

### 3-2. 링크와 이미지

```html
<!-- 링크 -->
<a href="https://google.com">구글로 이동</a>

<!-- 새 탭에서 열기 -->
<a href="https://google.com" target="_blank">새 탭으로 열기</a>

<!-- 이미지 -->
<img src="이미지경로.jpg" alt="이미지 설명">

<!-- alt는 이미지 로딩 실패 시 표시될 텍스트 (접근성에도 중요) -->
```

### 3-3. 레이아웃 태그

```html
<!-- div — 블록 레벨 컨테이너 (줄 전체 차지) -->
<div>구역을 나누는 박스</div>

<!-- span — 인라인 컨테이너 (내용만큼만 차지) -->
<span>텍스트 중 일부만 스타일 적용할 때</span>

<!-- 시맨틱 태그 (의미있는 구조) -->
<header>상단 헤더</header>
<nav>네비게이션 메뉴</nav>
<main>주요 콘텐츠</main>
<section>섹션</section>
<article>독립적인 콘텐츠</article>
<aside>사이드바</aside>
<footer>하단 푸터</footer>
```

> **시맨틱 태그를 쓰는 이유**
> `<div>`만 써도 동작하지만 시맨틱 태그는 **의미**를 가집니다.
> 검색 엔진(SEO), 스크린 리더(접근성), 코드 가독성 모두 좋아집니다.

### 3-4. 목록 태그

```html
<!-- 순서 없는 목록 (불릿 포인트) -->
<ul>
  <li>항목 1</li>
  <li>항목 2</li>
</ul>

<!-- 순서 있는 목록 (번호) -->
<ol>
  <li>첫 번째</li>
  <li>두 번째</li>
</ol>
```

### 3-5. 테이블 태그

```html
<table>
  <thead>
    <tr>
      <th>이름</th>
      <th>나이</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>홍길동</td>
      <td>30</td>
    </tr>
  </tbody>
</table>
```

| 태그 | 역할 |
|---|---|
| `<table>` | 테이블 전체 감싸기 |
| `<thead>` | 테이블 헤더 영역 |
| `<tbody>` | 테이블 본문 영역 |
| `<tr>` | 행 (Table Row) |
| `<th>` | 헤더 셀 (굵게 표시) |
| `<td>` | 일반 셀 (Table Data) |

### 3-6. 폼 태그

```html
<!-- 폼 — 사용자 입력을 받는 영역 -->

  <!-- 드롭다운 -->

    <option value="kr">한국</option>
    <option value="us">미국</option>

  <button type="submit">제출</button>

</form>
```

## 4. CSS?

CSS는 **Cascading Style Sheets**의 약자입니다.
HTML로 만든 뼈대에 **색상, 크기, 배치, 폰트** 등 시각적 스타일을 입히는 언어예요.

>**Cascading(폭포처럼 흐르는)이란?**
> CSS 스타일은 위에서 아래로 적용되고, 더 구체적인 규칙이 우선순위를 가집니다.
> 부모 태그의 스타일이 자식 태그에 상속되는 것도 이 개념입니다.

CSS의 기본 문법:

```css
선택자 {
  속성: 값;
  속성: 값;
}

/* 예시 */
h1 {
  color: blue;
  font-size: 32px;
}

p {
  color: gray;
  line-height: 1.6;
}
```

---

## 5. CSS 연결 방법

| 방법 | 설명 | 추천 |
|---|---|---|
| Inline | 태그 안에 직접 style 속성으로 작성 | 비추천 |
| Internal | `<head>` 안에 `<style>` 태그로 작성 | 소규모만 가능 |
| External | 별도 .css 파일로 분리 후 연결 | 실무 표준 |

```html
<!-- 1. Inline (비추천) -->
<h1 style="color: blue; font-size: 32px;">제목</h1>

<!-- 2. Internal -->
<head>
  <style>
    h1 { color: blue; }
  </style>
</head>

<!-- 3. External (추천) — HTML에서 연결 -->
<head>
  <link rel="stylesheet" href="style.css">
</head>
```

```css
/* style.css 파일 내용 */
h1 {
  color: blue;
}
```

## 6. CSS 선택자

선택자는 **어떤 HTML 요소에 스타일을 적용할지** 지정하는 방법.

```css
/* 태그 선택자 — 해당 태그 전체에 적용 */
p { color: gray; }

/* 클래스 선택자 — .클래스명 (여러 요소에 재사용 가능) */
.highlight { background-color: yellow; }

/* ID 선택자 — #아이디명 (페이지에서 딱 하나) */
#header { background-color: navy; }

/* 자식 선택자 — 직접 자식만 */
div > p { color: red; }

/* 후손 선택자 — 안에 있는 모든 요소 */
div p { color: blue; }

/* 여러 선택자 동시에 */
h1, h2, h3 { font-family: sans-serif; }

/* 가상 클래스 — 특정 상태일 때 */
a:hover { color: red; }        /* 마우스 올렸을 때 */
button:active { opacity: 0.8; } /* 클릭했을 때 */
input:focus { border-color: blue; } /* 포커스됐을 때 */
```

| 선택자 | HTML 예시 | CSS 예시 |
|---|---|---|
| 태그 | `<p>텍스트</p>` | `p { }` |
| 클래스 | `<p class="red">텍스트</p>` | `.red { }` |
| ID | `<p id="title">텍스트</p>` | `#title { }` |

>  ID는 페이지에서 딱 하나만 써야 합니다.
> 여러 곳에 같은 스타일을 적용할 땐 클래스를 쓰면됩니다.

## 7. CSS 단위

```css
/* px — 고정 픽셀 */
font-size: 16px;

/* % — 부모 요소 기준 비율 */
width: 50%;

/* em — 현재 요소의 font-size 기준 */
padding: 1.5em; /* font-size가 16px이면 24px */

/* rem — 루트(html) 요소의 font-size 기준 */
font-size: 1.5rem; /* html이 16px이면 24px */

/* vw / vh — 뷰포트(화면) 너비/높이 기준 */
width: 100vw;   /* 화면 전체 너비 */
height: 100vh;  /* 화면 전체 높이 */
```

| 단위 | 기준 | 주로 쓰는 곳 |
|---|---|---|
| `px` | 고정 픽셀 | border, 작은 요소 |
| `%` | 부모 요소 | width, height |
| `em` | 현재 font-size | padding, margin |
| `rem` | 루트 font-size | font-size (실무 표준) |
| `vw` / `vh` | 화면 크기 | 전체 화면 섹션 |

## 8. 박스 모델 (Box Model)

HTML의 모든 요소는 **박스**로 이루어져 있고,
박스는 안쪽부터 `content → padding → border → margin` 4겹 구조입니다.

```
┌─────────────────────────────┐
│           margin            │
│   ┌─────────────────────┐   │
│   │       border        │   │
│   │   ┌─────────────┐   │   │
│   │   │   padding   │   │   │
│   │   │  ┌───────┐  │   │   │
│   │   │  │content│  │   │   │
│   │   │  └───────┘  │   │   │
│   │   └─────────────┘   │   │
│   └─────────────────────┘   │
└─────────────────────────────┘
```

| 속성 | 설명 |
|---|---|
| `content` | 실제 내용이 들어가는 영역 |
| `padding` | content와 border 사이의 안쪽 여백 |
| `border` | 박스의 테두리 |
| `margin` | border 바깥의 다른 요소와의 바깥쪽 여백 |

```css
div {
  width: 200px;
  height: 100px;
  padding: 20px;           /* 상하좌우 모두 20px */
  border: 2px solid black;
  margin: 10px;            /* 상하좌우 모두 10px */
}

/* 상하좌우 개별 지정 */
div {
  /* 단축 표기: 상 우 하 좌 (시계 방향) */
  padding: 10px 20px 10px 20px;
  margin: 10px 20px;       /* 상하 10px, 좌우 20px */
}
```

> **box-sizing: border-box**
> 기본적으로 width에 padding과 border가 포함되지 않으며.
> 실무에서는 아래 코드를 항상 CSS 맨 위에 씁니다:
> ```css
> * { box-sizing: border-box; }
> ```

## 9. position

요소를 **어떤 기준으로 어디에 배치할지** 지정합니다.

```css
/* static — 기본값. 일반 흐름대로 배치 */
div { position: static; }

/* relative — 원래 자기 위치를 기준으로 이동 */
div {
  position: relative;
  top: 10px;   /* 원래 위치에서 아래로 10px */
  left: 20px;  /* 원래 위치에서 오른쪽으로 20px */
}

/* absolute — 가장 가까운 position: relative 부모 기준으로 배치 */
.parent {
  position: relative;  /* 기준점 */
}
.child {
  position: absolute;
  top: 0;
  right: 0;  /* 부모의 오른쪽 상단에 붙이기 */
}

/* fixed — 화면(뷰포트) 기준으로 고정. 스크롤해도 안 움직임 */
.navbar {
  position: fixed;
  top: 0;
  width: 100%;
}

/* sticky — 스크롤 시 특정 위치에서 고정됨 */
.header {
  position: sticky;
  top: 0;
}
```

| 값 | 기준 | 특징 |
|---|---|---|
| `static` | 일반 흐름 | 기본값 |
| `relative` | 자기 원래 위치 | 주변 요소에 영향 없음 |
| `absolute` | relative 부모 | 일반 흐름에서 벗어남 |
| `fixed` | 화면(뷰포트) | 스크롤해도 고정 |
| `sticky` | 스크롤 위치 | 특정 지점에서 고정 |

> `absolute`는 부모에 `position: relative`가 없으면
> `<html>` 기준으로 배치돼요. 항상 부모에 `relative`를 설정합니다.

## 10. CSS 변수

```css
/* :root에 변수 선언 — 전역으로 사용 가능 */
:root {
  --primary-color: #3B82F6;

  --spacing-md: 16px;

/* 변수 사용 */
```

## 11. Flexbox — 1차원 레이아웃

Flexbox는 **가로 또는 세로 한 방향**으로 요소를 정렬하는 레이아웃 방식이며
네비게이션 바, 카드 나열, 중앙 정렬에 많이 씁니다.

```css
/* 부모 요소에 display: flex 설정 */
.container {
  display: flex;

  /* 방향 설정 */
  flex-direction: row;          /* 가로 방향 (기본값) */
  flex-direction: column;       /* 세로 방향 */

  /* 가로축 정렬 */
  justify-content: flex-start;  /* 왼쪽 정렬 (기본값) */
  justify-content: center;      /* 가운데 정렬 */
  justify-content: flex-end;    /* 오른쪽 정렬 */
  justify-content: space-between; /* 양 끝에 붙이고 나머지 균등 */
  justify-content: space-around;  /* 모두 균등한 여백 */

  /* 세로축 정렬 */
  align-items: flex-start;  /* 위쪽 정렬 */
  align-items: center;      /* 세로 가운데 정렬 */
  align-items: flex-end;    /* 아래쪽 정렬 */

  /* 줄 바꿈 */
  flex-wrap: wrap;   /* 넘치면 다음 줄로 */
  flex-wrap: nowrap; /* 한 줄 유지 (기본값) */

  /* 요소 간 간격 */
  gap: 16px;
}

/* 완벽한 가운데 정렬 (자주 쓰는 패턴) */
.center {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* 자식 요소 비율 설정 */
.item {
  flex: 1;  /* 남은 공간을 균등하게 나눔 */
}
```

> Flexbox는 **부모**에 `display: flex`를 설정하면
> **자식**들이 자동으로 flex 아이템이 됩니다.

## 12. Grid — 2차원 레이아웃

Grid는 **가로와 세로 동시에** 격자(Grid) 형태로 요소를 배치하는 레이아웃.
전체 페이지 레이아웃, 이미지 갤러리에 많이 씁니다.

```css
.container {
  display: grid;

  /* 열(column) 3개, 각각 1:1:1 비율 */
  grid-template-columns: 1fr 1fr 1fr;

  /* 단축 표기 */
  grid-template-columns: repeat(3, 1fr);

  /* 행(row) 높이 설정 */
  grid-template-rows: 100px auto 100px;

  /* 격자 사이 간격 */
  gap: 20px;
  column-gap: 20px;  /* 열 간격만 */
  row-gap: 10px;     /* 행 간격만 */
}

/* 반응형 자동 조절 (자주 쓰는 패턴) */
.container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 20px;
}
```

| 비교 | Flexbox | Grid |
|---|---|---|
| 차원 | 1차원 (가로 or 세로) | 2차원 (가로 and 세로) |
| 주로 쓰는 곳 | 네비게이션, 카드 나열 | 전체 레이아웃, 갤러리 |
| 기준 | 자식 요소 크기 기준 | 격자(칸) 기준 |

## 13. transition / animation

```css
/* transition — 상태 변화를 부드럽게 */
button {
  background-color: blue;
  transition: background-color 0.3s ease;
}

button:hover {
  background-color: darkblue; /* 0.3초 동안 부드럽게 변함 */
}

/* 여러 속성 동시에 */
.card {
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}

.card:hover {
  transform: translateY(-4px);  /* 위로 4px 이동 */
  box-shadow: 0 8px 24px rgba(0,0,0,0.1);
}

/* animation — 키프레임으로 복잡한 애니메이션 */
@keyframes fadeIn {
  from { opacity: 0; }
  to   { opacity: 1; }
}

.element {
  animation: fadeIn 0.5s ease;
}
```

## 14. 반응형 디자인

반응형 디자인은 **화면 크기에 따라 레이아웃이 자동으로 바뀌는** 디자인

```css
/* 기본 구조 */
@media (조건) {
  /* 조건이 맞을 때만 적용되는 스타일 */
}

/* 768px 이하 (모바일)일 때 */
@media (max-width: 768px) {
  .container {
    flex-direction: column;
  }
  .sidebar {
    display: none;
  }
}

/* 1024px 이상 (데스크탑)일 때 */
@media (min-width: 1024px) {
  .container {
    max-width: 1200px;
    margin: 0 auto;
  }
}
```

| 구간 | 범위 | 주요 대상 |
|---|---|---|
| 모바일 | ~ 768px | 스마트폰 |
| 태블릿 | 769px ~ 1024px | 태블릿 |
| 데스크탑 | 1025px ~ | 노트북, 데스크탑 |

> **모바일 퍼스트(Mobile First)** 전략
> 모바일 스타일을 기본으로 작성하고 화면이 커질수록 추가 스타일을 덮어씌우는 방식.
> `min-width`를 기준으로 미디어 쿼리를 작성합니다. 실무 표준입니다.
