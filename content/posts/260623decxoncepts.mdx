# 웹 개발 핵심 개념

> 웹 서비스는 식당과 똑같습니다.
> 손님(사용자)이 있고, 홀(화면)이 있고, 주방(서버)이 있고, 창고(데이터베이스)가 있어요.
> 각 역할이 뭔지 하나씩 살펴봅시다.

---

## 한눈에 보기

| 역할 | 식당 비유 | 실제 기술 | 한 줄 요약 |
| --- | --- | --- | --- |
| 프론트엔드 | 홀, 메뉴판 | Next.js / React | 사용자 눈에 보이는 모든 것 |
| 백엔드 | 주방, 요리사 | FastAPI / Python | 실제 로직이 돌아가는 곳 |
| 데이터베이스 | 냉장고, 창고 | PostgreSQL | 데이터를 저장하는 곳 |
| 보안 | 입구 경비원 | JWT / CORS | 신원 확인 + 이상한 요청 차단 |
| DevOps | 건물주, 관리자 | Docker / Vercel | 서버를 짓고 운영하는 것 |

---

## 1. 프론트엔드 (Frontend)

> **한 줄 정의:** 사용자 눈에 보이는 모든 것을 만드는 영역

### 식당 비유

식당에서 손님이 보는 것들 — 메뉴판, 테이블, 의자, 조명, 서빙 직원.
손님이 "김치찌개 주세요" 하고 말하는 행위 자체도 프론트엔드입니다.
**주방에서 뭔 일이 일어나는지는 손님이 몰라도 됩니다.**

### 실제 웹에서는?

지금 이 글을 읽고 있는 화면 자체가 프론트엔드입니다.

- 버튼
- 글자
- 색상
- 이미지
- 클릭하면 반응하는 것들
- 로딩 스피너

이 모든 걸 프론트엔드가 만들어요.

### 어떻게 만드나요?

```
HTML  → 뼈대 (어떤 요소가 있는가)
CSS   → 디자인 (어떻게 생겼는가)
JS    → 동작 (클릭하면 어떻게 되는가)
```

추후 배울 **Next.js**는 이 셋을 훨씬 편하게 만들어주는 도구입니다.

<Callout>
💡 비유: HTML은 건물 뼈대(철골), CSS는 인테리어(벽지, 조명), JS는 자동문·엘리베이터(동작하는 것들)
</Callout>

### 프론트엔드 개발자가 하는 일

- 화면 레이아웃 짜기
- 버튼 클릭 시 동작 만들기
- 백엔드에서 데이터 받아서 화면에 표시하기
- 모바일/PC 화면에 맞게 반응형 만들기

## 2. 백엔드 (Backend)

> **한 줄 정의:** 사용자 눈에 안 보이지만, 실제 일이 일어나는 곳

### 식당 비유

주방입니다. 손님은 주방을 볼 수 없지만, 실제 요리는 여기서 일어나요.
"김치찌개 주세요" 라는 주문이 들어오면 → 주방에서 재료를 꺼내고 → 요리해서 → 내보내는 것.

**백엔드 = 요리사 + 주방**

### 실제 웹에서는?

- 로그인 버튼을 누르면 → 백엔드가 "이 사람 회원 맞나?" 확인
- 상품 목록을 열면 → 백엔드가 데이터베이스에서 상품 정보를 꺼내서 전달
- 결제하면 → 백엔드가 결제 처리하고 주문 내역 저장

프론트엔드가 "이거 해줘!" 하고 요청하면, 백엔드가 처리하고 결과를 돌려줘요.
이 요청/응답 방식을 **API**라고 부릅니다.

<Callout>
API란?
Application Programming Interface의 약자로 
프론트엔드와 백엔드가 대화하는 규칙입니다.
프론트: "회원가입 시켜줘 (이름: 홍길동, 이메일: ...)"
백엔드: "OK, 완료했어 / 아니면 이미 있는 이메일이야"
이런 주고받음의 형식을 API라고 해요.

</Callout>

### 추후 배울 FastAPI

Python으로 만드는 백엔드 프레임워크입니다.
"프레임워크"란 개발을 편하게 해주는 틀(뼈대)이에요.
FastAPI를 쓰면 API를 빠르고 쉽게 만들 수 있습니다.

```python
# FastAPI 예시 — 이렇게 쓰면 API 하나가 만들어져요
@app.get("/hello")
def say_hello():
    return {"message": "안녕하세요!"}
```

## 3. 데이터베이스 (Database)

> **한 줄 정의:** 데이터를 저장하고 꺼내오는 창고

### 식당 비유

냉장고 + 창고입니다.
요리사(백엔드)가 재료(데이터)가 필요하면 창고에서 꺼내오고,
새로운 재료가 들어오면 창고에 보관합니다.

### 실제 웹에서는?

- 회원 정보 (이름, 이메일, 비밀번호)
- 게시글 목록
- 주문 내역
- 좋아요 누른 것들

이런 것들이 전부 데이터베이스에 저장돼요.
서버를 껐다 켜도 데이터가 사라지지 않는 이유가 바로 데이터베이스 덕분입니다.

<Callout>
데이터베이스가 없다면? 서버를 껐다 켜는 순간 모든 데이터가 사라집니다.
마치 냉장고 없는 식당에서 재료를 그냥 바닥에 놔두는 것과 같아요.
</Callout>

### 추후 배울 PostgreSQL

가장 널리 쓰이는 관계형 데이터베이스 중 하나입니다.
엑셀처럼 행(row)과 열(column)로 데이터를 저장해요.

```
users 테이블 (표)
┌────┬──────────┬──────────────────┐
│ id │   name   │      email       │
├────┼──────────┼──────────────────┤
│  1 │  홍길동  │ hong@example.com │
│  2 │  김철수  │ kim@example.com  │
└────┴──────────┴──────────────────┘
```

## 4. 보안 (Security)

> **한 줄 정의:** 허가된 사람만 들어오게 하고, 이상한 요청을 차단하는 것

### 식당 비유

입구의 경비원입니다.
- 예약한 손님만 입장 가능한 식당이라면 → 예약 확인
- 술 취한 손님은 입장 거부
- 직원 전용 구역은 직원만 출입 가능

### 실제 웹에서는?

**인증(Authentication):** "이 사람이 누구인가?" 확인
→ 로그인이 바로 인증입니다.

**인가(Authorization):** "이 사람이 이걸 할 권한이 있나?" 확인
→ 일반 회원은 관리자 페이지에 못 들어가게 하는 것.

<Callout>
인증 vs 인가 차이
인증: 신분증 확인 (당신이 홍길동 맞나요?)
인가: 권한 확인 (홍길동은 이 방에 들어갈 수 있나요?)
</Callout>

### 추후 배울 보안 기술들

**JWT (JSON Web Token)**
로그인 성공 시 발급받는 "디지털 신분증"입니다.
이걸 가지고 있으면 로그인된 상태로 인정받아요.

**httpOnly Cookie**
JWT를 안전하게 보관하는 방법입니다.
JavaScript로 접근 못하게 막아서 해킹을 어렵게 만들어요.

**CORS (Cross-Origin Resource Sharing)**
"다른 주소에서 오는 요청을 허용할지 말지" 결정하는 규칙입니다.
내 백엔드에 아무 사이트나 접근 못하게 막는 것이에요.

## 5. DevOps

> **한 줄 정의:** 서비스가 실제로 돌아가도록 환경을 만들고 관리하는 것

### 식당 비유

건물주 + 시설 관리자입니다.
아무리 요리를 잘해도 식당 건물이 없으면 장사를 못 하죠.
건물을 짓고(서버 세팅), 전기·수도를 연결하고(인프라), 항상 불이 켜져 있게(무중단 운영) 유지하는 것.

### 실제 웹에서는?

코드를 짜는 것만으로는 서비스를 출시할 수 없어요.
코드가 실제 인터넷에서 돌아가게 하려면 DevOps가 필요합니다.

- 서버 컴퓨터에 코드 올리기
- 24시간 꺼지지 않게 유지하기
- 사용자가 많아지면 서버 늘리기
- 새 버전 배포할 때 서비스 안 끊기게 하기

### 추후 배울 DevOps 기술들

**Docker**
코드를 "박스"에 포장해서, 어떤 컴퓨터에서든 똑같이 돌아가게 하는 도구입니다.
"내 컴퓨터에서는 됐는데 서버에서는 안 돼요" 문제를 해결해줘요.

**Vercel**
Next.js(프론트엔드)를 아주 쉽게 인터넷에 올릴 수 있는 서비스입니다.
코드를 GitHub에 올리면 자동으로 배포까지 해줘요.

**무중단 배포 (Zero-downtime Deploy)**
서비스를 업데이트할 때, 사용자에게 "지금 점검 중" 메시지 없이 조용히 바꾸는 기술.

<Callout>
DevOps는 처음엔 어렵게 느껴지지만, 사실 가장 나중에 배워도 됩니다.
코드를 먼저 잘 짜는 법을 익히고, 그다음에 배포하는 법을 배워도 늦지 않아요.
</Callout>

## 전체 흐름 정리

사용자가 로그인 버튼을 누르면 어떤 일이 일어날까요?

```
1. 사용자가 로그인 버튼 클릭
        ↓
2. 프론트엔드(Next.js)가 백엔드에 요청 전송
   "이 이메일 + 비밀번호가 맞나요?"
        ↓
3. 보안이 요청을 검사
   "이 요청 허가된 곳에서 온 건가?"
        ↓
4. 백엔드(FastAPI)가 처리
   "데이터베이스에서 이 사람 찾아봐"
        ↓
5. 데이터베이스(PostgreSQL)가 확인
   "맞아, 이 사람 회원 맞아"
        ↓
6. 백엔드가 JWT(디지털 신분증) 발급
        ↓
7. 프론트엔드가 JWT 받아서 보관
   → 로그인 완료 화면 표시
        ↓
8. 이 모든 것이 DevOps가 만든 서버 위에서 돌아감
```
