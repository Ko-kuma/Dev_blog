## 함수란?

특정 작업을 하나로 묶어서, 필요할 때마다 이름으로 불러 쓸 수 있게 만든 코드 뭉치다.

실생활 비유: 자판기와 같다. 동전(입력)을 넣고 버튼을 누르면, 정해진 과정(내부 로직)을 거쳐 음료(출력)가 나온다. 자판기 내부가 어떻게 동작하는지 몰라도, "동전 넣고 버튼 누르면 음료가 나온다"는 사용법만 알면 누구나 쓸 수 있다.

```python
def greet(name):        # greet이라는 이름의 함수를 정의
    return f"안녕, {name}!"   # 결과를 돌려줌(반환)

message = greet("혜진")   # 함수를 호출(사용)
print(message)              # 안녕, 혜진!
```

### 독스트링 (Docstring) — 함수에 설명 달기

함수 바로 아래 줄에 큰따옴표 세 개(`"""`)로 감싼 문자열을 적으면, 그 함수가 무슨 일을 하는지 설명하는 공식 문서(Docstring)가 된다.

```python
def greet(name):
    """이름을 받아서 인사말 문자열을 만들어 반환한다."""
    return f"안녕, {name}!"

print(greet.__doc__)   # 이름을 받아서 인사말 문자열을 만들어 반환한다.
print(help(greet))      # 독스트링을 포함한 함수 설명을 보여줌
```

실생활 비유: 자판기 옆에 붙어있는 "사용 설명서" 스티커와 같다. 자판기를 직접 열어보지 않아도, 설명서만 읽으면 어떻게 쓰는지 알 수 있다.

**실무에서는 이렇게 쓴다**: 협업할 때 다른 사람(또는 나중의 나)이 함수 내부 코드를 다 읽지 않고도 "이 함수가 뭘 하는지"를 바로 파악하게 해준다. VSCode 같은 에디터에서 함수에 마우스를 올렸을 때 뜨는 설명도 이 독스트링에서 가져오는 것이다.

## 매개변수(Parameter)와 인자(Argument)

두 용어가 비슷해 보이지만 가리키는 대상이 다르다.

- **매개변수**: 함수를 "정의할 때" 괄호 안에 적어두는 이름표. 위 예시의 `name`.
- **인자**: 함수를 "호출할 때" 실제로 넣어주는 값. 위 예시의 `"혜진"`.

실생활 비유: 매개변수는 자판기에 붙어있는 "동전 투입구"라는 빈 자리이고, 인자는 실제로 그 자리에 넣는 진짜 동전이다.

**실무에서는 이렇게 쓴다**: 코드 리뷰나 에러 메시지에서 "매개변수 개수가 안 맞다"는 말을 자주 듣게 되는데, 이건 함수 정의할 때 정한 자리(매개변수) 개수와 실제로 넣은 값(인자) 개수가 다르다는 뜻이다.

## 기본값 매개변수 (Default Parameter)

매개변수에 미리 기본값을 정해두면, 인자를 안 넘겨도 그 값이 자동으로 쓰인다.

```python
def greet(name, greeting="안녕"):   # greeting은 기본값 "안녕"을 가짐
    return f"{greeting}, {name}!"

print(greet("혜진"))            # 안녕, 혜진! (greeting 생략 → 기본값 사용)
print(greet("혜진", "반가워"))   # 반가워, 혜진! (직접 지정하면 기본값 무시)
```

실생활 비유: 카페에서 "사이즈를 안 말하면 기본으로 톨 사이즈를 준다"는 규칙과 같다. 특별히 다른 사이즈를 요청하면 그걸 우선한다.

> 함정: 기본값이 있는 매개변수는 반드시 기본값이 없는 매개변수보다 뒤에 와야 한다. `def greet(greeting="안녕", name):`처럼 순서를 바꾸면 `SyntaxError`가 발생한다.

### 가변 기본값의 함정 (Mutable Default Argument)

리스트나 딕셔너리처럼 가변 자료형을 기본값으로 쓰면, 예상과 전혀 다르게 동작하는 유명한 함정이 있다.

```python
def add_item(item, cart=[]):   # 기본값으로 빈 리스트를 지정
    cart.append(item)
    return cart

print(add_item("사과"))    # ['사과']
print(add_item("바나나"))  # ['사과', '바나나'] — 예상: ['바나나']만 나와야 할 것 같은데!
```

**왜 이런 일이 생길까**: 기본값(`cart=[]`)은 함수가 호출될 때마다 새로 만들어지는 게 아니라, 함수를 정의하는 시점에 딱 한 번만 만들어져서 계속 재사용된다. 그래서 이전 호출에서 추가한 값이 다음 호출에도 그대로 남아있게 된다.

실생활 비유: 자판기에 "손님이 안 가져가고 남긴 물건을 담아두는 공용 바구니"가 딱 하나만 있는 것과 같다. 새 손님이 올 때마다 새 바구니를 주는 게 아니라, 이전 손님이 두고 간 물건이 그대로 남아있는 바구니를 계속 같이 쓰게 되는 것이다.

**올바른 해결법**: 기본값을 `None`으로 두고, 함수 안에서 진짜 빈 리스트를 새로 만든다.

```python
def add_item(item, cart=None):
    if cart is None:
        cart = []          # 호출될 때마다 새로운 리스트가 만들어짐
    cart.append(item)
    return cart

print(add_item("사과"))    # ['사과']
print(add_item("바나나"))  # ['바나나'] — 이제 의도대로 동작함
```

> 함정: 이건 Python에서 가장 유명한 초보자 버그 중 하나로 꼽힌다. 기본값에 리스트나 딕셔너리를 직접 쓰지 말고, `None`으로 둔 다음 함수 안에서 새로 만드는 패턴을 습관화하는 게 안전하다.

## 위치 인자 vs 키워드 인자

함수를 호출할 때 값을 넘기는 두 가지 방식이다.

```python
def introduce(name, age, job):
    return f"{name}, {age}세, {job}"

# 위치 인자: 순서대로 값이 매칭됨
print(introduce("혜진", 30, "developer"))

# 키워드 인자: 이름을 지정해서 순서 상관없이 값을 넘김
print(introduce(job="developer", name="혜진", age=30))
```

실생활 비유: 위치 인자는 "첫 번째 칸엔 이름, 두 번째 칸엔 나이"처럼 정해진 순서대로 서류 양식을 채우는 것이고, 키워드 인자는 각 칸에 "이름: ___", "나이: ___"라고 라벨을 붙여서 순서 상관없이 채우는 것이다.

**실무에서는 이렇게 쓴다**: 매개변수가 많은 함수를 호출할 때, 키워드 인자를 쓰면 "이 값이 무엇을 의미하는지" 코드만 봐도 바로 알 수 있어서 가독성이 좋아진다. FastAPI 코드에서도 키워드 인자를 쓰는 관례가 흔하다.

## *args — 개수가 정해지지 않은 위치 인자

몇 개의 값이 들어올지 미리 알 수 없을 때, `*args`로 여러 개의 인자를 하나로 묶어서 받을 수 있다.

```python
def total(*args):        # 몇 개가 들어오든 args라는 튜플로 묶임
    return sum(args)

print(total(1, 2, 3))       # 6
print(total(1, 2, 3, 4, 5)) # 15
```

실생활 비유: 뷔페 식당에서 "몇 접시를 가져오든 상관없이 한 번에 계산대에서 처리"하는 것과 같다. 접시 개수를 미리 정해두지 않아도 알아서 다 받아준다.

> `args`라는 이름 자체는 관례일 뿐, 별표(`*`)가 진짜 문법이다. `*numbers`라고 써도 똑같이 동작한다.

## **kwargs — 개수가 정해지지 않은 키워드 인자

`*args`가 여러 개의 값을 받는다면, `**kwargs`는 이름이 붙은(키워드) 값을 여러 개 받아서 딕셔너리로 묶어준다.

```python
def print_profile(**kwargs):   # 어떤 키워드 인자든 kwargs라는 딕셔너리로 묶임
    for key, value in kwargs.items():
        print(f"{key}: {value}")

print_profile(name="혜진", age=30, job="developer")
# name: 혜진
# age: 30
# job: developer
```

실생활 비유: 회원가입 양식에 "추가로 원하는 정보가 있으면 자유롭게 적어주세요"라는 빈 칸을 여러 개 열어둔 것과 같다. 몇 개를 적을지, 어떤 항목일지 미리 정해두지 않아도 전부 받을 수 있다.

**실무에서는 이렇게 쓴다**: 함수를 만들 때 "앞으로 매개변수가 늘어날 수도 있다"는 상황에 대비하거나, 여러 설정값을 유연하게 받아야 하는 함수(로깅, 설정 관리 등)에서 자주 쓰인다.

JS와 비교하면:

| Python | JavaScript |
|---|---|
| `def func(*args):` | `function func(...args) { }` (전개 구문, Rest Parameter) |
| `def func(**kwargs):` | 전용 문법 없음 — 마지막 매개변수를 객체로 받아서 흉내냄 |

> 함정: JS의 `...args`(Rest Parameter)는 Python의 `*args`와 개념이 비슷하지만, JS에는 `**kwargs`처럼 이름 붙은 인자만 따로 묶어주는 전용 문법이 없다. 보통 함수의 마지막 매개변자리를 객체 하나로 받아서(`function func(options) { }`) 비슷하게 흉내낸다.

## 함수를 호출하는 여러 가지 방법

지금까지는 `*args`/`**kwargs`로 "여러 인자를 하나로 모아서 받는" 정의 쪽을 봤는데, 반대로 호출하는 쪽에서 리스트나 딕셔너리를 낱개로 풀어서 넘길 수도 있다.

### 인자 언패킹으로 호출하기

```python
def add(a, b, c):
    return a + b + c

numbers = [1, 2, 3]
print(add(*numbers))   # add(1, 2, 3)과 완전히 같음 — 리스트를 낱개로 풀어서 전달

data = {"a": 1, "b": 2, "c": 3}
print(add(**data))     # add(a=1, b=2, c=3)과 완전히 같음 — 딕셔너리를 키워드 인자로 풀어서 전달
```

실생활 비유: 선물 세트(리스트/딕셔너리)를 통째로 건네는 대신, 포장을 뜯어서 개별 물건(낱개 인자)으로 하나씩 손에 쥐어주는 것과 같다. `*`와 `**`는 정의할 때는 "여러 개를 하나로 모으는" 역할이지만, 호출할 때는 정반대로 "하나로 뭉쳐있는 걸 여러 개로 풀어주는" 역할을 한다.

**실무에서는 이렇게 쓴다**: API로 받은 딕셔너리 데이터를 함수의 각 매개변수에 그대로 대응시켜 넘길 때(`create_user(**request_data)`), 리스트로 관리하던 값들을 한꺼번에 함수 인자로 넘길 때 자주 쓰인다.

### 인자 개수가 안 맞으면?

매개변수 개수와 인자 개수가 다르면 `TypeError`가 발생한다.

```python
def introduce(name, age):
    return f"{name}, {age}세"

introduce("혜진")   # TypeError: introduce() missing 1 required positional argument: 'age'
introduce("혜진", 30, "developer")   # TypeError: introduce() takes 2 positional arguments but 3 were given
```

> 함정: 에러 메시지를 읽으면 "무엇이 부족한지" 또는 "몇 개가 초과됐는지"를 정확히 알려준다. 에러가 나면 당황하지 말고 메시지를 그대로 읽으면 원인을 바로 찾을 수 있다.

### 함수 자체를 값으로 넘기기 (괄호의 유무가 중요하다)

`함수이름`(괄호 없이)은 함수 자체를 가리키고, `함수이름()`(괄호 있이)은 함수를 실행한 결과를 가리킨다. 이 둘을 헷갈리면 완전히 다른 동작을 하게 된다.

```python
def greet():
    return "안녕!"

a = greet     # 함수 자체를 변수에 저장 (아직 실행 안 됨)
b = greet()   # 함수를 실행한 "결과값"을 저장

print(a)   # <function greet at 0x...> — 함수 자체를 가리킴
print(b)   # 안녕! — 실행 결과
print(a())  # 안녕! — 나중에 필요할 때 a()로 실행
```

실생활 비유: `greet`(괄호 없이)은 자판기 버튼 자체를 가리키는 것이고, `greet()`(괄호 있이)은 그 버튼을 실제로 눌러서 나온 음료를 가리키는 것이다.

**실무에서는 이렇게 쓴다**: 오늘 배운 `users.sort(key=lambda user: user["age"])`처럼, 함수를 다른 함수의 인자로 넘길 때는 괄호 없이 함수 자체(또는 람다)를 넘긴다. 만약 실수로 `key=lambda user: user["age"]()`처럼 괄호를 붙이면, "나중에 실행할 함수"가 아니라 "지금 당장 실행한 결과"를 넘기게 되어 원하는 동작을 하지 않는다.

## return과 print의 차이

초보자가 정말 자주 헷갈리는 부분이다. `print()`는 화면에 값을 "보여주기"만 하고, `return`은 그 값을 함수 밖으로 "돌려줘서" 다른 코드에서 쓸 수 있게 한다.

```python
def add_wrong(a, b):
    print(a + b)      # 화면에 출력만 함, 함수 밖으로 값을 안 돌려줌

def add_right(a, b):
    return a + b       # 값을 진짜로 돌려줌

result = add_wrong(3, 4)   # 화면에 7이 출력되지만
print(result)                # None! (return이 없어서 아무것도 못 받음)

result2 = add_right(3, 4)
print(result2)                # 7 (진짜로 값을 받음)
```

실생활 비유: `print()`는 자판기 화면에 "음료 나왔습니다"라는 문구만 띄우는 것이고, `return`은 실제로 음료를 손에 쥐어주는 것이다. 화면 문구만 보고 손엔 아무것도 없다면 그 다음 아무것도 할 수 없다.

> 함정: `print()`가 있다고 함수가 값을 돌려주는 게 아니다. 함수의 결과를 다른 변수에 저장하거나 다른 함수에 넘기려면 반드시 `return`이 있어야 한다. `return`이 없는 함수는 항상 `None`을 반환한다.

**실무에서는 이렇게 쓴다**: 디버깅할 때는 `print()`로 중간 값을 눈으로 확인하고, 실제 로직에서 계산 결과를 다음 단계로 넘길 때는 반드시 `return`을 쓴다.

## 여러 값 반환하기

앞서 배운 튜플과 언패킹이 여기서 그대로 쓰인다. Python 함수는 콤마로 여러 값을 이어 쓰면 자동으로 튜플 하나를 반환한다.

```python
def get_min_max(numbers):
    return min(numbers), max(numbers)   # 사실 튜플 (min(numbers), max(numbers))를 반환

low, high = get_min_max([3, 1, 4, 1, 5])   # 튜플 언패킹으로 두 변수에 나눠 받음
print(low, high)   # 1 5
```

JS와 비교하면:

| Python | JavaScript |
|---|---|
| `return a, b` (튜플 하나를 반환) | 전용 문법 없음 — 배열이나 객체로 감싸서 반환 (`return [a, b]`) |

## 스코프 — 함수 안과 밖의 변수는 다르다

함수 안에서 만든 변수(지역변수)는 함수 밖에서 접근할 수 없다.

```python
def calculate():
    result = 100   # 지역변수, calculate 함수 안에서만 존재

calculate()
print(result)   # NameError! 함수 밖에서는 result가 존재하지 않음
```

실생활 비유: 자판기 내부에서 잠깐 쓰는 부품은 자판기 문을 열어야만 볼 수 있고, 문 밖(함수 바깥)에서는 그 부품이 아예 존재하지 않는 것처럼 접근할 수 없다.

### global — 함수 안에서 전역변수를 진짜로 수정하기

```python
count = 0   # 전역변수

def increase_wrong():
    count = count + 1   # UnboundLocalError! count를 지역변수로 취급하는데, 아직 값이 없음

def increase_right():
    global count          # "이 함수 안에서 count는 전역변수를 가리킨다"고 선언
    count = count + 1

increase_right()
increase_right()
print(count)   # 2 — 전역변수가 실제로 바뀜
```

`global count`라고 선언하는 순간, 함수 안에서 `count`에 값을 대입해도 새로운 지역변수를 만드는 게 아니라 진짜 전역변수를 수정하게 된다.

실생활 비유: `global`은 "이 부품은 자판기 밖에 있는 공용 재고 창고 것을 가리킨다"고 미리 선언해두는 것과 같다. 이 선언이 없으면 자판기는 자기 안에 새 부품을 몰래 만들어버리고, 밖의 진짜 창고는 건드리지 않는다.

> 함정: `global` 없이 함수 안에서 전역변수와 같은 이름에 값을 대입하면, Python은 그 이름을 "새로운 지역변수"로 취급한다. 이때 대입 전에 그 지역변수를 먼저 읽으려고 하면(`count + 1`처럼) `UnboundLocalError`가 발생한다. "값을 읽기만 하는 것"과 "값을 대입하는 것"은 완전히 다른 취급을 받는다는 게 핵심이다.

**왜 실무에서 잘 안 쓸까**: `global`을 여기저기서 쓰면, 어떤 함수가 그 변수를 언제 바꾸는지 코드 전체를 다 훑어봐야 알 수 있게 된다. 함수가 입력(매개변수)을 받아서 출력(`return`)만 내놓는 구조를 유지하면, 그 함수만 봐도 동작을 예측할 수 있어 훨씬 안전하다. 그래서 대부분의 실무 코드는 `global` 대신 `return`으로 값을 넘기고 다시 변수에 대입하는 방식을 쓴다.

## 람다 함수 (Lambda) — 이름 없는 한 줄 함수

아주 짧고 간단한 함수를 `def` 없이 한 줄로 만드는 문법이다.

```python
# 일반 함수
def square(x):
    return x ** 2

# 같은 기능을 람다로
square_lambda = lambda x: x ** 2

print(square(4))          # 16
print(square_lambda(4))   # 16
```

`lambda 매개변수: 반환할 표현식` 형태로, `def`와 `return`을 따로 안 쓰고 한 줄에 압축한 것이다.

실생활 비유: 정식 이름과 간판을 갖춘 가게(일반 함수) 대신, 길거리에서 잠깐 물건을 파는 노점(람다)과 같다. 한 번 쓰고 말 간단한 용도라면 정식 가게를 차릴 필요 없이 노점으로 충분하다.

**어디에 주로 쓰일까**: 람다 자체를 변수에 저장해서 쓰기보다는, `sort()`처럼 "정렬 기준을 잠깐 지정해줘야 하는" 다른 함수 안에 바로 끼워넣는 용도로 훨씬 많이 쓰인다.

```python
# 실무에서 자주 보는 형태: 정렬 기준을 람다로 지정
users = [
    {"name": "혜진", "age": 30},
    {"name": "민수", "age": 25},
]

users.sort(key=lambda user: user["age"])   # "age" 값을 기준으로 정렬
print(users)   # 민수(25)가 먼저, 혜진(30)이 나중
```

JS와 비교하면:

| Python | JavaScript |
|---|---|
| `lambda x: x ** 2` | `x => x ** 2` (화살표 함수) |
| `users.sort(key=lambda u: u["age"])` | `users.sort((a, b) => a.age - b.age)` |

Python의 람다와 JS의 화살표 함수는 "짧은 함수를 한 줄로 표현한다"는 목적은 같지만, 문법과 쓰는 방식에 차이가 있다. JS의 화살표 함수는 일반 함수 대신으로도 자유롭게 쓰이지만, Python의 람다는 관례상 정말 간단한 표현식 하나짜리에만 제한적으로 쓴다.

> 함정: 람다 안에는 `if`, `for` 같은 문(statement)을 넣을 수 없고, 오직 표현식 하나만 담을 수 있다. 로직이 조금이라도 복잡해지면 억지로 람다에 욱여넣지 말고 `def`로 일반 함수를 만드는 게 낫다.

## 요약

- 함수는 코드를 이름으로 묶어 재사용하게 해주는 도구이며, `def`로 정의하고 `return`으로 결과를 돌려준다
- 독스트링(`"""설명"""`)으로 함수 바로 아래에 설명을 달아두면, `help()`나 에디터 툴팁으로 바로 확인할 수 있다
- 매개변수(정의할 때 이름표)와 인자(호출할 때 실제 값)는 서로 다른 개념이다
- 기본값 매개변수로 인자를 생략할 수 있고, 기본값 없는 매개변수보다 뒤에 와야 한다
- 함정: 리스트·딕셔너리 같은 가변 자료형을 기본값으로 쓰면, 호출할 때마다 값이 공유되어 예상과 다르게 동작한다. 기본값은 `None`으로 두고 함수 안에서 새로 만드는 게 안전하다
- 위치 인자는 순서대로, 키워드 인자는 이름을 지정해서 값을 넘긴다
- `*args`는 여러 개의 위치 인자를 튜플로, `**kwargs`는 여러 개의 키워드 인자를 딕셔너리로 묶어 받는다
- 반대로 호출할 때는 `*리스트`, `**딕셔너리`로 이미 묶여있는 데이터를 낱개 인자로 풀어서 넘길 수 있다
- 매개변수와 인자 개수가 안 맞으면 `TypeError`가 발생하며, 에러 메시지가 부족한지 초과됐는지 알려준다
- 함정: `함수이름`(괄호 없음)은 함수 자체를, `함수이름()`(괄호 있음)은 실행 결과를 가리킨다. `sort(key=...)`처럼 함수를 값으로 넘길 때는 괄호를 붙이지 않는다
- 함정: `print()`는 화면에 보여주기만 하고, `return`이 있어야 값을 함수 밖으로 진짜로 돌려준다. `return` 없는 함수는 항상 `None` 반환
- `return a, b`처럼 콤마로 여러 값을 반환하면 튜플이 되고, 언패킹으로 여러 변수에 나눠 받을 수 있다
- 함정: 함수 안에서 만든 변수(지역변수)는 함수 밖에서 접근할 수 없다. 반대로 함수 안에서 전역변수와 같은 이름에 값을 대입하면 새로운 지역변수가 만들어진다
- `global 변수명`을 선언하면 함수 안에서 전역변수를 진짜로 수정할 수 있지만, 코드 추적이 어려워져 실무에서는 `return`으로 값을 넘기는 방식을 더 선호한다
- 람다(`lambda 매개변수: 표현식`)는 이름 없는 한 줄 함수로, `sort()`의 정렬 기준처럼 다른 함수 안에 바로 끼워넣는 용도로 주로 쓰인다. 표현식 하나만 담을 수 있어 복잡한 로직에는 적합하지 않다
