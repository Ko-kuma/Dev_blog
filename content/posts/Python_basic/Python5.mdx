## 자료구조란?
 
여러 개의 값을 하나로 묶어서 관리하는 방법이다.
 
지금까지는 `age = 25`처럼 변수 하나에 값 하나만 담았는데, 실제로는 "학생 30명의 이름"처럼 여러 값을 한꺼번에 다뤄야 하는 경우가 훨씬 많다. 이럴 때 쓰는 게 자료구조다. 오늘은 그중 가장 기본인 ** 리스트(list) **와 ** 튜플(tuple) **을 배운다.
 
## 리스트 (List)
 
**정의**: 여러 개의 값을 순서대로 담아두는 상자다. 담긴 값은 나중에 추가·삭제·수정이 가능하다.
 
실생활 비유: 장바구니와 같다. 물건을 담고, 담은 순서가 유지되고, 마음이 바뀌면 물건을 빼거나 새로 담을 수도 있다.
 
```python
fruits = ["사과", "바나나", "포도"]   # 대괄호 [ ]로 리스트를 만듦
```
 
### 인덱싱 (Indexing) — 순서로 값 꺼내기
 
리스트 안의 각 값은 0부터 시작하는 번호(인덱스)로 위치가 매겨진다.
 
```python
fruits = ["사과", "바나나", "포도"]
print(fruits[0])   # 사과 (첫 번째)
print(fruits[1])   # 바나나 (두 번째)
print(fruits[-1])  # 포도 (맨 뒤에서 첫 번째, 음수 인덱스는 뒤에서부터 셈)
```
 
> 함정: 사람은 보통 "첫 번째"를 1번으로 세지만, 프로그래밍에서는 0번부터 센다. `fruits[1]`이 두 번째 값이라는 게 초보자가 가장 많이 헷갈리는 부분이다.
 
### 값 추가/삭제/수정
 
```python
fruits.append("수박")     # 맨 뒤에 추가
fruits.remove("바나나")   # 값으로 찾아서 삭제
fruits[0] = "딸기"        # 인덱스로 값 수정
print(fruits)             # ['딸기', '포도', '수박']
```
 
### 슬라이싱 (Slicing) — 일부만 잘라내기
 
```python
numbers = [10, 20, 30, 40, 50]
print(numbers[1:3])   # [20, 30] (인덱스 1부터 3 전까지)
print(numbers[:2])    # [10, 20] (처음부터 인덱스 2 전까지)
print(numbers[2:])    # [30, 40, 50] (인덱스 2부터 끝까지)
```
 
실생활 비유: 김밥을 자를 때 "1번째 칸부터 3번째 칸 직전까지만 도시락에 담아줘"라고 지정하는 것과 같다.
 
### len() — 개수 세기
 
리스트에 값이 몇 개 들어있는지는 `len()` 함수로 확인한다.
 
```python
fruits = ["사과", "바나나", "포도"]
print(len(fruits))   # 3
```
 
**실무에서는 이렇게 쓴다**: "장바구니가 비어있는지" 확인할 때(`len(cart) == 0`), 페이지네이션에서 전체 항목 개수를 셀 때 거의 항상 쓰인다. (Day3에서 배운 Truthy/Falsy를 활용하면 `if cart:`만으로도 비슷하게 확인할 수 있다.)
 
### 자주 쓰는 리스트 메서드
 
`append()`, `remove()` 외에도 실무에서 자주 보는 메서드들이 있다.
 
```python
numbers = [3, 1, 4, 1, 5]
 
numbers.insert(0, 100)   # 0번 위치에 100을 끼워넣기 → [100, 3, 1, 4, 1, 5]
numbers.pop()             # 맨 뒤 값을 꺼내면서 삭제 → 5가 반환되고 리스트에서 사라짐
numbers.sort()            # 오름차순 정렬 (리스트 자체가 바뀜)
numbers.count(1)          # 값 1이 몇 개 있는지 셈
numbers.index(4)          # 값 4가 몇 번째 인덱스에 있는지 찾음
```
 
### remove()와 pop()의 차이
 
둘 다 리스트에서 값을 지운다는 점은 같지만, **무엇을 기준으로 지우는지**가 다르다.
 
| 구분 | `remove()` | `pop()` |
|---|---|---|
| 기준 | 값으로 찾아서 삭제 | 인덱스(위치)로 찾아서 삭제 |
| 반환값 | 없음 (`None`) | 삭제한 값을 반환함 |
| 인덱스 생략 시 | 해당 없음 (값을 반드시 지정) | 맨 뒤 값을 삭제 |
| 값/위치가 없으면 | `ValueError` 발생 | `IndexError` 발생 |
 
```python
fruits = ["사과", "바나나", "포도"]
fruits.remove("바나나")   # "바나나"라는 값을 찾아서 삭제
print(fruits)              # ['사과', '포도']
 
fruits = ["사과", "바나나", "포도"]
last = fruits.pop()        # 인덱스 생략 시 맨 뒤(포도)를 삭제하고, 그 값을 반환
print(last)                 # 포도
first = fruits.pop(0)      # 0번 인덱스(사과)를 삭제하고 반환
print(first)                 # 사과
```
 
실생활 비유: `remove()`는 "빨간 옷을 찾아서 빼줘"처럼 물건 자체를 지목하는 것이고, `pop()`은 "맨 위에 있는 것(또는 3번째 칸에 있는 것)을 꺼내줘"처럼 위치를 지목하는 것이다.
 
JS와 비교하면:
 
| Python | JavaScript |
|---|---|
| `fruits.remove("바나나")` | 전용 메서드 없음 — `fruits.splice(fruits.indexOf("바나나"), 1)`로 흉내내야 함 |
| `fruits.pop()` | `fruits.pop()` — 동일하게 맨 뒤 값을 반환하며 삭제 |
 
> 함정: JS의 `pop()`은 Python과 똑같이 동작하지만, JS에는 값으로 바로 삭제하는 `remove()` 같은 전용 메서드가 없다. 인덱스를 먼저 찾은 다음 `splice()`로 지워야 해서 한 단계가 더 필요하다.
 
**실무에서는 이렇게 쓴다**: 장바구니에서 "이 상품을 빼주세요"처럼 값을 아는 경우엔 `remove()`, 큐/스택처럼 "맨 앞(또는 맨 뒤) 작업을 하나씩 꺼내서 처리하기" 같은 로직에는 `pop()`을 쓴다.
 
### 중첩 리스트 (Nested List)
 
리스트 안에 또 다른 리스트를 넣을 수도 있다. 표(행과 열) 형태의 데이터를 표현할 때 쓴다.
 
```python
grid = [
    [1, 2, 3],
    [4, 5, 6],
]
print(grid[0])       # [1, 2, 3] (0번째 행 전체)
print(grid[0][1])    # 2 (0번째 행의 1번째 값)
```
 
실생활 비유: 서랍장 안에 여러 개의 작은 칸막이 상자가 들어있는 것과 같다. 서랍(바깥 리스트) 하나를 열면, 그 안에 또 작은 칸(안쪽 리스트)들이 나뉘어 있다.
 
### 리스트를 복사할 때 반드시 알아야 할 함정
 
리스트를 다른 변수에 그대로 대입(`=`)하면, **복사가 아니라 같은 리스트를 가리키는 이름표가 하나 더 생기는 것**이다.
 
```python
list1 = [1, 2, 3]
list2 = list1        # 복사가 아니라, list2도 list1과 같은 리스트를 가리킴
 
list2.append(4)
print(list1)          # [1, 2, 3, 4] — list1도 같이 바뀜!
print(list2)          # [1, 2, 3, 4]
```
 
실생활 비유: 집 열쇠를 복사해서 두 개를 만든 것과 같다. 열쇠(변수 이름)는 두 개지만, 그 열쇠로 여는 집(실제 리스트)은 하나뿐이다. 한쪽 열쇠로 집 안 가구를 옮기면, 다른 쪽 열쇠로 들어가도 똑같이 바뀐 모습을 보게 된다.
 
**진짜로 복사하고 싶다면** 아래처럼 명시적으로 복사해야 한다.
 
```python
list1 = [1, 2, 3]
list2 = list1.copy()   # 진짜 복사본 생성 (list1[:]도 같은 효과)
 
list2.append(4)
print(list1)   # [1, 2, 3] — 원본은 안 바뀜
print(list2)   # [1, 2, 3, 4]
```
 
> 함정: 이건 초보자가 실무에서 정말 자주 만나는 버그의 원인이다. "분명 list2만 바꿨는데 왜 list1도 바뀌었지?"라는 상황을 겪는다면 대부분 이 문제다. 리스트, 딕셔너리처럼 값이 변경 가능한(Mutable) 자료구조는 항상 이 점을 주의해야 한다. (반대로 숫자나 문자열, 그리고 튜플처럼 변경 불가능한 자료형은 이런 문제가 생기지 않는다 — 튜플이 안전한 이유 중 하나이기도 하다.)
 
JS와 비교하면:
 
| 구분 | JavaScript | Python |
|---|---|---|
| 자료구조 이름 | Array (배열) | List (리스트) |
| 선언 | `const fruits = ["사과", "바나나"]` | `fruits = ["사과", "바나나"]` |
| 값 추가 | `fruits.push("수박")` | `fruits.append("수박")` |
| 슬라이싱 | `fruits.slice(1, 3)` | `fruits[1:3]` |
| 길이 확인 | `fruits.length` | `len(fruits)` |
 
리스트/배열 개념 자체는 JS와 Python이 거의 동일하다. 함수 이름(`push` vs `append`)과 슬라이싱 문법(`slice()` vs `[:]`)만 다르다고 생각하면 된다.
 
**실무에서는 이렇게 쓴다**: DB에서 여러 회원 정보를 한 번에 가져올 때, API 응답으로 게시글 목록을 주고받을 때 등 여러 개의 데이터를 하나로 묶어 다루는 거의 모든 상황에서 리스트가 쓰인다.
 
## 튜플 (Tuple)
 
**정의**: 리스트와 거의 똑같이 여러 값을 순서대로 담지만, 한 번 만들면 내용을 바꿀 수 없는(불변, Immutable) 자료구조다.
 
실생활 비유: 인쇄된 영수증과 같다. 장바구니(리스트)는 담긴 물건을 계속 바꿀 수 있지만, 영수증(튜플)은 한 번 발행되면 그 안의 내용을 고칠 수 없다.
 
```python
point = (3, 5)   # 소괄호 ( )로 튜플을 만듦
print(point[0])   # 3 (인덱싱은 리스트와 동일하게 사용 가능)
 
point[0] = 10     # TypeError! 튜플은 값을 수정할 수 없음
```
 
### 왜 굳이 못 바꾸는 자료구조를 쓸까
 
- **실수 방지**: 좌표(x, y)처럼 절대 바뀌면 안 되는 값을 튜플로 만들어두면, 나중에 실수로 값을 바꾸는 코드를 짜도 즉시 에러가 나서 바로 알아챌 수 있다.
- **속도**: 튜플은 리스트보다 내부적으로 더 가볍고 빠르게 처리된다.
```python
# 실무 예시: 여러 값을 한 번에 반환할 때 튜플을 자주 씀
def get_min_max(numbers):
    return min(numbers), max(numbers)   # 괄호 없이 콤마만 써도 튜플이 됨
 
low, high = get_min_max([3, 1, 4, 1, 5])   # 튜플을 두 변수에 바로 나눠 담음(언패킹)
print(low, high)   # 1 5
```
 
### 그래도 튜플 내용을 "고치고" 싶다면 — 리스트로 변환하는 우회법
 
튜플 자체를 직접 수정하는 방법은 없지만, **리스트로 변환해서 고친 다음 다시 튜플로 되돌리는** 우회법은 있다.
 
```python
point = (3, 5)
 
point_list = list(point)   # 튜플 → 리스트로 변환 (이제 수정 가능해짐)
point_list[0] = 10          # 리스트니까 인덱스로 수정 가능
point = tuple(point_list)   # 다시 리스트 → 튜플로 변환
 
print(point)   # (10, 5)
```
 
실생활 비유: 인쇄된 영수증(튜플)의 글자를 직접 지우고 다시 쓸 수는 없지만, 영수증 내용을 그대로 베껴서 메모장(리스트)에 옮긴 다음 메모장에서 수정하고, 그걸 다시 새 영수증(새 튜플)으로 인쇄하는 것과 같다.
 
> 함정: 이건 원래 튜플을 "수정"하는 게 아니라, 완전히 새로운 튜플을 만들어서 같은 변수 이름에 다시 담는 것이다. 기존 튜플 자체는 여전히 바뀌지 않았고, `point`라는 이름표가 이제 새로 만든 튜플을 가리키게 됐을 뿐이다. 튜플의 "불변성"이라는 원칙 자체는 깨지지 않는다.
 
**간단히 값을 추가만 하고 싶다면** `+` 연산자로 새 튜플을 이어붙일 수도 있다.
 
```python
point = (3, 5)
point = point + (7,)   # (3, 5, 7) — 역시 새 튜플을 만드는 것
```
 
**실무에서는 이렇게 쓴다**: 설정값처럼 "평소엔 안 바뀌지만, 아주 가끔 관리자가 값을 하나 고쳐야 하는" 데이터에 이 패턴을 쓴다. 자주 값이 바뀌는 데이터라면 애초에 리스트로 관리하는 게 낫고, 이 우회법은 "원래는 불변이어야 하는데 예외적으로 한 번 바꿔야 할 때"에만 쓰는 게 좋다.
 
### 튜플 언패킹 (Tuple Unpacking)
 
방금 예시에서 `low, high = get_min_max(...)`처럼 튜플 안의 값을 여러 변수에 한 번에 나눠 담는 것을 **언패킹**이라고 부른다.
 
```python
point = (3, 5)
x, y = point   # 튜플 안의 값을 각각 x와 y에 나눠 담음
print(x)   # 3
print(y)   # 5
```
 
실생활 비유: 택배 상자(튜플) 하나를 열어서, 안에 든 물건 두 개를 각각 다른 서랍(변수)에 바로 나눠 넣는 것과 같다. 상자를 열고 → 물건을 하나씩 꺼내서 → 각자 자리에 넣는 과정을 한 줄로 끝낸다.
 
JS와 비교하면:
 
| Python | JavaScript |
|---|---|
| `x, y = point` | `const [x, y] = point;` (배열 구조 분해 할당) |
 
JS의 "구조 분해 할당(Destructuring)"과 개념이 동일하다. Python은 대괄호 없이 바로 변수명을 나열하고, JS는 대괄호로 감싼다는 문법 차이만 있다.
 
불변성 관점에서도 JS와 비교하면:
 
| Python | JavaScript |
|---|---|
| `point = (3, 5)` (튜플, 불변) | 전용 튜플 타입 없음 — 배열을 그대로 쓰거나 `Object.freeze()`로 흉내냄 |
 
> 함정: JS에는 Python의 튜플 같은 전용 불변 자료구조가 없다. 비슷하게 흉내내려면 `Object.freeze()`를 쓰거나 그냥 관례적으로 "이 배열은 안 바꾼다"고 약속하는 정도다. Python은 이걸 아예 언어 차원에서 강제한다는 점이 다르다.
 
**실무에서는 이렇게 쓴다**: 함수가 값을 두 개 이상 반환할 때(`return low, high`), 위도·경도 좌표처럼 절대 바뀌면 안 되는 값을 담을 때 튜플을 쓴다.
 
## 리스트 vs 튜플 — 언제 뭘 쓰나
 
| 구분 | 리스트 (List) | 튜플 (Tuple) |
|---|---|---|
| 괄호 | `[ ]` | `( )` |
| 값 변경 | 가능 (Mutable) | 불가능 (Immutable) |
| 쓰는 경우 | 계속 추가/삭제될 데이터 (장바구니, 할 일 목록) | 고정된 값 묶음 (좌표, 설정값) |
 
## 요약
 
- 자료구조는 여러 값을 하나로 묶어 관리하는 방법이며, 오늘은 리스트와 튜플을 배웠다
- 리스트(`[ ]`)는 값 변경이 자유로운 자료구조. 인덱스는 0부터 시작하며, 음수 인덱스는 뒤에서부터 센다
- 슬라이싱(`[1:3]`)으로 리스트 일부만 잘라낼 수 있다. 끝 인덱스는 포함되지 않는다
- 함정: 인덱스가 0부터 시작한다는 점, 슬라이싱 끝값이 미포함이라는 점을 늘 주의해야 한다
- `len()`으로 리스트(또는 튜플)의 길이를 확인할 수 있다
- `insert()`, `pop()`, `sort()`, `count()`, `index()` 등 자주 쓰는 리스트 메서드가 있다
- `remove()`는 값으로, `pop()`은 인덱스(위치)로 삭제한다는 차이가 있다. `pop()`은 삭제한 값을 반환하지만 `remove()`는 반환값이 없다
- 중첩 리스트로 표(행과 열) 형태의 데이터를 표현할 수 있다 (`grid[0][1]`처럼 이중 인덱싱)
- **가장 중요한 함정**: `list2 = list1`은 복사가 아니라 같은 리스트를 가리키는 이름표가 하나 더 생기는 것. 진짜 복사하려면 `.copy()`나 `[:]`를 써야 한다. 이 문제는 튜플처럼 변경 불가능한 자료형에는 생기지 않는다
- 튜플(`( )`)은 한 번 만들면 값을 바꿀 수 없는 자료구조. 실수 방지와 속도 이점이 있다
- 튜플을 "고치고" 싶다면 `list(튜플)`로 변환해 수정한 뒤 `tuple(리스트)`로 되돌리는 우회법이 있다. 이건 수정이 아니라 새 튜플을 만드는 것이라 불변성 원칙은 깨지지 않는다
- 튜플 언패킹(`x, y = point`)으로 튜플 안의 값을 여러 변수에 한 번에 나눠 담을 수 있다. JS의 구조 분해 할당과 같은 개념
- 함정: JS에는 Python 튜플 같은 전용 불변 자료구조가 없다
- 실무에서는 여러 값을 반환할 때(튜플), 계속 변하는 데이터 목록을 다룰 때(리스트) 구분해서 쓴다
