## Python 소개 & 실행 방식

### 프로그래밍 언어 기초 개념

#### 저수준 언어 vs 고수준 언어

| 구분 | 저수준(low-level) 언어 | 고수준(high-level) 언어 |
|---|---|---|
| 예시 | 어셈블리어, C | Python, JavaScript, Java |
| 특징 | 컴퓨터(하드웨어)에 가까움, 메모리 주소를 직접 다룸 | 사람이 이해하기 쉬움, 메모리 관리를 자동으로 해줌 |
| 실행 속도 | 빠름 | 상대적으로 느림 |
| 배우기 | 어려움 | 쉬움 |

비유: 저수준 언어는 "재료 g수, 온도, 시간을 초 단위까지 정확히 지시하는 레시피"고, 고수준 언어는 "적당히 볶아줘"라고만 말해도 알아서 처리해주는 것과 비슷하다. Python은 후자에 가까워서, 사람이 편하게 쓸 수 있는 대신 컴퓨터 입장에서는 "적당히"를 해석하는 과정이 필요해서 조금 느리다.

#### 인터프리터 언어 vs 컴파일 언어

| 구분 | 컴파일 언어 | 인터프리터 언어 |
|---|---|---|
| 예시 | C, C++ | Python, JavaScript |
| 실행 방식 | 전체 코드를 미리 기계어로 통째로 번역(컴파일)한 후 실행 | 코드를 한 줄씩 번역하면서 즉시 실행 |
| 비유 | 책 전체를 번역기로 미리 통째로 번역해 놓고 읽는 것 | 통역사가 옆에서 한 문장씩 즉석 통역해주는 것 |
| 장점 | 실행 속도가 빠름 | 수정하고 바로 실행할 수 있어 개발이 빠름 |
| 단점 | 코드 수정할 때마다 다시 컴파일해야 함 | 매번 번역하면서 실행하므로 상대적으로 느림 |

이 두 개념을 알고 나면, 왜 Python이 "배우기 쉽지만 실행 속도는 느린" 언어인지가 자연스럽게 이해된다. 고수준 언어 + 인터프리터 언어라는 두 가지 특성이 겹쳐서 그렇다.

### Python이란?

Python은 1991년 네덜란드 개발자 **Guido van Rossum**이 발표한 **고수준(high-level), 인터프리터 방식**의 범용 프로그래밍 언어다.

방금 배운 개념으로 다시 말하면: Python은 사람이 이해하기 쉬운 문법(고수준)을 가졌고, 코드를 한 줄씩 즉시 실행(인터프리터)하는 언어다.

```python
# 저수준 언어(C)라면 메모리 할당까지 신경써야 하지만
# Python은 그냥 이렇게 쓰면 끝
numbers = [1, 2, 3]
total = sum(numbers)
print(total)  # 6
```

### 누가, 언제 만들었나?

- 창시자: Guido van Rossum (귀도 반 로섬)
- 발표 시점: 1991년 (Python 0.9.0)
- 탄생 배경: 반 로섬은 네덜란드 CWI(국립 수학전산학연구소)에서 ABC언어라는 교육용 언어 개발에 참여했다. ABC는 배우기 쉬웠지만 확장성이 부족했고, 그 아쉬움을 해결하기 위해 크리스마스 연휴 동안 취미로 만들기 시작한 것이 Python이다.
- 2018년까지 반 로섬은 Python의 모든 규칙을 최종적으로 결정하는 "BDFL(종신 자애로운 독재자)"이었다. 회사로 치면 "최종 결재권자" 같은 역할이었다가, 이후에는 여러 명이 함께 결정하는 커뮤니티 운영위원회 체제로 바뀌었다.

### 이름의 유래

Python이라는 이름은 뱀이나 그리스 신화와 무관하다. 반 로섬이 즐겨 보던 영국 코미디 그룹의 방송 "Monty Python's Flying Circus"에서 따온 이름이다. 그는 "뭔가 짧고, 신비롭고, 약간 엉뚱한 이름을 원했다"고 밝힌 바 있다.

다만 이후 공식 로고에 뱀 두 마리가 들어가고 커뮤니티에서 뱀 이미지를 적극적으로 채택하면서, 지금은 뱀 상징이 훨씬 유명해진 상태다. (PyPI 패키지 아이콘, 컨퍼런스 이름 "PyCon" 등에도 뱀/코미디 유머가 자주 섞여 나온다.)

### 특징

| 특징 | 쉽게 설명하면 |
|---|---|
| 인터프리터 언어 | 코드를 미리 통째로 변환(컴파일)하지 않고, 한 줄씩 바로 실행함 |
| 동적 타이핑 | 변수의 종류(숫자, 글자 등)를 미리 정해두지 않아도 됨 (`x = 5`만 써도 숫자로 인식) |
| 들여쓰기 기반 문법 | `{}` 대신 들여쓰기로 코드 블록(묶음)을 구분함 |
| 다중 패러다임 | 절차형/객체지향/함수형 등 여러 방식으로 코드를 짤 수 있음 |
| 방대한 표준 라이브러리 | "배터리 포함" 철학 — 별도 설치 없이 기본으로 쓸 수 있는 기능이 많음 |
| 플랫폼 독립적 | Windows/Mac/Linux 어디서나 똑같이 실행됨 |

### 장점

- 가독성 높은 문법: 영어 문장과 비슷해서 코드 자체가 설명서 역할을 함
- 빠른 개발 속도: 적은 코드로 같은 기능을 구현할 수 있음
- 방대한 생태계: PyPI에 60만 개 이상의 패키지(남이 만들어 둔 기능 모음)가 있음
- 낮은 진입장벽: 비전공자도 배우기 쉬운 언어로 설계되어 교육용으로 세계적으로 채택됨
- 커뮤니티: 사용자가 압도적으로 많아서 막히는 부분을 검색하면 대부분 답이 나옴

### 단점

- 실행 속도: 인터프리터 언어라 C/Java 같은 컴파일 언어보다 느림 (FastAPI는 비동기 처리로 이를 상당 부분 보완함)
- GIL(Global Interpreter Lock): 쉽게 말하면, 한 프로그램 안에서 여러 작업을 진짜 동시에 처리하지 못하고 번갈아가며 처리한다는 뜻이다. 계산이 많은 작업에서 느려지는 원인 중 하나다. (지금은 "이런 게 있구나" 정도만 알아두면 된다)
- 모바일 앱 개발에는 부적합: iOS/Android 네이티브 앱 개발 생태계가 약함
- 동적 타이핑의 양면성: 빠르게 짤 수 있지만, 프로젝트가 커지면 타입 관련 버그가 늦게 발견될 수 있음 (→ Type Hint 기능, FastAPI/Pydantic이 이를 적극 활용)

### "Life is too short, you need Python"

Python 커뮤니티에서 유명한 밈 문구다. 핵심 메시지는 다음과 같다.

> 인생은 짧으니, 굳이 어렵고 장황한 코드를 쓰며 시간 낭비하지 말자.

C나 Java로 몇 줄씩 써야 하는 작업을 Python은 한두 줄로 끝낼 수 있는 경우가 많다. 예를 들어 파일 하나를 열어서 읽는 작업:

```python
# Python: 단 2줄
with open("file.txt") as f:
    content = f.read()
```

이런 짧고 직관적인 문법이 Python 철학의 핵심이며, 실무에서 프로토타이핑이나 자동화 스크립트에 Python이 많이 쓰이는 이유이기도 하다.

### Python으로 무엇을 만들 수 있나

| 분야 | 쉽게 설명하면 |

> 참고: C/C++ 결합과 GUI 프로그래밍은 "Python이 할 수 있는 일"에 포함시켜 정보로만 알아두는 것이다. 지금 직접 공부하는 항목은 아니고,지금은 Next.js/FastAPI 같은 웹 스택에 계속 집중한다.(추후 시간이 있을 시 배워볼 예정)

### JS와 실행 방식 비교

| 구분 | JavaScript | Python |
|---|---|---|
| 실행 주체 | 브라우저 엔진(V8 등) 또는 Node.js | Python 인터프리터(CPython) |
| 파일 확장자 | .js | .py |
| 실행 명령 | node app.js | python app.py |
| 세미콜론 | 선택 (자동 삽입) | 사용 안 함 |
| 들여쓰기 | 스타일 문제 (강제 아님) | 문법 규칙 (강제) |
| 변수 선언 | let, const | 선언 키워드 없이 바로 대입 |

### 들여쓰기가 문법이다

JS에서는 중괄호 `{ }`로 코드 블록을 구분했다.

```javascript
if (age >= 18) {
  console.log("성인");
}
```

Python은 중괄호가 없다. 들여쓰기(보통 스페이스 4칸) 자체가 블록의 경계다.

```python
if age >= 18:        # 콜론(:)으로 블록 시작을 알림
    print("성인")     # 들여쓰기가 곧 "이 줄은 if 안에 속함"이라는 의미
```

들여쓰기가 한 칸이라도 틀리면 `IndentationError`가 발생한다. JS처럼 "일관성 없어도 일단 돌아가는" 게 아니라, 들여쓰기가 곧 코드 구조라는 점이 가장 큰 사고방식 전환 포인트다.

#### 왜 Tab이 아니라 Space를 쓰는가

Python 공식 스타일 가이드 PEP 8에서는 들여쓰기에 탭(Tab) 대신 스페이스(Space) 4칸을 사용할 것을 권장한다. 이유는 다음과 같다.

1. 탭의 너비가 에디터마다 다르게 보임: 어떤 에디터는 탭을 2칸으로, 어떤 에디터는 4칸이나 8칸으로 표시한다. 같은 코드인데 보는 사람마다 다르게 보여서 혼란이 생긴다.
2. 탭과 스페이스를 섞어 쓰면 에러 발생: Python 3에서는 탭과 스페이스를 한 파일에 섞어 쓰면 `TabError`가 발생하며 아예 실행이 안 된다. (Python 2는 허용했지만 Python 3부터 엄격해짐)
3. 협업 시 충돌 원인: Git 같은 버전관리 도구로 코드 변경사항을 비교(diff)할 때 탭은 표시가 애매해서, 여러 사람이 같이 작업할 때 충돌이 자주 생긴다.
4. 대부분의 에디터가 자동 변환해줌: VSCode 같은 최신 에디터는 Python 파일에서 Tab 키를 눌러도 설정에 따라 자동으로 스페이스 4칸으로 바꿔준다.

> VSCode에서 Python 파일을 열면 화면 하단 상태바에 "Spaces: 4"라고 표시되는지 확인해보면 좋다.

### 예시

```python
# hello.py
name = "혜진"          # 변수 선언 키워드 없음 (let, const 같은 것 없음)
print(f"안녕, {name}!") # f-string: JS의 템플릿 리터럴(`${name}`)과 같은 역할
```

터미널에서 실행:

```bash
python hello.py
```

### 요약

- 저수준 언어는 컴퓨터에 가깝고 빠르지만 어렵고, 고수준 언어는 사람이 이해하기 쉽지만 상대적으로 느림. Python은 고수준 언어
- 컴파일 언어는 전체를 미리 번역 후 실행, 인터프리터 언어는 한 줄씩 즉시 번역하며 실행. Python은 인터프리터 언어
- Python은 Guido van Rossum이 1991년 발표한 언어이며, ABC 언어의 한계를 보완하려는 시도에서 시작됨
- 이름은 신화가 아니라 코미디 쇼 "Monty Python"에서 유래 (뱀 로고는 이후에 생긴 상징)
- 특징: 동적 타이핑, 들여쓰기 기반 문법, 방대한 표준 라이브러리
- 장점: 가독성, 빠른 개발 속도, 거대한 생태계 / 단점: 실행 속도, GIL, 모바일 개발 약함
- "Life is too short, you need Python" — 짧고 직관적인 코드로 생산성을 높이는 철학
- 웹 백엔드, 데이터 분석, AI, 자동화, C/C++ 연동, 데스크탑 GUI까지 폭넓게 쓰이지만, 지금 로드맵은 웹 백엔드(FastAPI)에 집중함
- 변수 선언 키워드가 없고, 콜론(`:`)으로 블록 시작을 표시함
- 들여쓰기는 Tab이 아니라 Space 4칸을 써야 함 (PEP 8 권장, 섞어 쓰면 TabError 발생)
