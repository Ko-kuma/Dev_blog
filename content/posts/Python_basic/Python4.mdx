## 반복문이란?

같은 동작을 여러 번 반복해서 실행하고 싶을 때 쓰는 문법이다.

실생활 비유: 세탁기의 "헹굼" 기능과 같다. "물 채우기 → 헹구기 → 물 빼기"를 사람이 몇 번씩 손으로 반복하는 대신, "3번 반복해줘"라고 한 번만 지시하면 세탁기가 알아서 그 과정을 3번 되풀이한다.

Python의 반복문은 크게 두 가지다: `for`문과 `while`문.

**용어 하나 먼저 짚고 가기 — 이터레이션(Iteration)**: 반복문이 한 바퀴 도는 것, 즉 반복문 몸체(body)가 한 번 실행되는 단위를 **이터레이션**이라고 부른다. "5번 반복한다"는 말은 "이터레이션이 5번 일어난다"는 뜻과 같다.

```python
for fruit in ["사과", "바나나", "포도"]:
    print(fruit)
# 이 반복문은 3번의 이터레이션으로 이루어짐:
# 1번째 이터레이션: fruit = "사과"
# 2번째 이터레이션: fruit = "바나나"
# 3번째 이터레이션: fruit = "포도"
```

실생활 비유: 세탁기가 헹굼을 3번 한다고 할 때, "1차 헹굼", "2차 헹굼", "3차 헹굼" 각각이 하나의 이터레이션이다. "헹굼을 반복한다"는 전체 과정과, "1차 헹굼" 같은 개별 한 번은 서로 다른 개념이다.

**실무에서는 이렇게 쓴다**: "몇 번째 이터레이션에서 에러가 났다", "이터레이션마다 API를 한 번씩 호출한다"처럼, 반복문 실행 중 어느 시점을 콕 집어 이야기할 때 쓰는 표준 용어다. 디버깅할 때 "루프의 3번째 이터레이션에서 값이 이상해진다"는 식으로 정확하게 문제 지점을 설명할 수 있다.

## for문 — 정해진 대상을 순서대로 반복

**정의**: 리스트, 문자열처럼 여러 개의 값이 모인 것을 순서대로 하나씩 꺼내면서 반복하는 문법이다.

```python
fruits = ["사과", "바나나", "포도"]

for fruit in fruits:      # fruits 안의 값을 하나씩 꺼내서 fruit에 담음
    print(fruit)           # "사과" → "바나나" → "포도" 순서로 출력
```

**정해진 횟수만큼 반복하기**: 리스트 없이 "그냥 5번 반복해줘"라고 하고 싶을 때는 `range()`를 쓴다.

```python
for i in range(5):        # 0, 1, 2, 3, 4 다섯 번 반복
    print(i)
```

> `range(5)`는 0부터 4까지(5는 포함 안 됨) 다섯 개의 숫자를 만들어낸다. "5번 반복"이라고 할 때 왜 4까지만 나오는지 헷갈리기 쉬운데, "5개를 만든다"고 기억하면 된다.

JS와 비교하면:

| 구분 | JavaScript | Python |
|---|---|---|
| 배열 순회 | `for (const fruit of fruits) { }` | `for fruit in fruits:` |
| 횟수 반복 | `for (let i = 0; i < 5; i++) { }` | `for i in range(5):` |

Python의 `for`문은 JS의 `for...of`와 거의 똑같은 느낌이다. 다만 JS의 전통적인 `for (let i=0; i<5; i++)` 같은 3단계 구성(초기값, 조건, 증감) 자체가 Python에는 없고, `range()`가 그 역할을 대신한다.

**실무에서는 이렇게 쓴다**: DB에서 가져온 여러 개의 회원 데이터를 하나씩 화면에 뿌릴 때(`for user in users:`), 게시글 목록을 순서대로 처리할 때 등 리스트 형태 데이터를 다루는 거의 모든 곳에서 쓰인다.

**`range()`의 세 가지 인자**: 시작값, 끝값, 증가폭까지 지정할 수 있다.

```python
for i in range(2, 10, 2):   # 2부터 시작, 10 전까지, 2씩 증가
    print(i)                 # 2, 4, 6, 8 출력
```

실생활 비유: 계단을 오를 때 "1층부터 시작해서, 10층 전까지, 2층씩 건너뛰며 올라가라"는 지시와 같다. 시작(2), 끝(10, 미포함), 보폭(2) 세 가지를 정확히 지정하는 것이다.

**문자열도 순회할 수 있다**: 리스트뿐 아니라 문자열도 한 글자씩 꺼내며 반복할 수 있다.

```python
for char in "안녕":
    print(char)   # "안" → "녕" 순서로 출력
```

**실무에서는 이렇게 쓴다**: `range(0, 100, 10)`처럼 페이지네이션에서 10개씩 건너뛰며 처리하거나, 짝수만 확인할 때(`range(0, 10, 2)`) 자주 쓰인다.

## enumerate() — 순번과 값을 함께 꺼내기

**정의**: 리스트를 순회할 때, 값뿐 아니라 몇 번째인지(인덱스)도 동시에 꺼내주는 함수다.

```python
fruits = ["사과", "바나나", "포도"]

for index, fruit in enumerate(fruits):
    print(index, fruit)
# 0 사과
# 1 바나나
# 2 포도
```

`enumerate()`가 없다면 순번을 직접 세는 변수를 따로 만들어야 하는데, `enumerate()`는 그 과정을 자동으로 해준다.

```python
# enumerate() 없이 순번을 직접 관리하면 이렇게 번거로움
index = 0
for fruit in fruits:
    print(index, fruit)
    index += 1   # 매번 직접 1씩 증가시켜야 함
```

실생활 비유: 줄 서있는 사람들에게 이름표를 나눠주면서 동시에 "1번, 2번, 3번..."이라고 순서를 매겨주는 것과 같다.

**실무에서는 이렇게 쓴다**: 게시글 목록을 화면에 뿌릴 때 순번을 같이 표시하거나(`1. 첫 게시글`, `2. 두 번째 게시글`), 리스트의 특정 위치(인덱스)에 있는 값을 조건에 따라 바꿀 때 흔히 쓰인다.

## 중첩 반복문 (Nested Loop)

조건문에 중첩 if가 있었듯, 반복문 안에 또 다른 반복문을 넣을 수도 있다.

```python
for i in range(1, 3):        # 바깥 반복: 1, 2
    for j in range(1, 3):    # 안쪽 반복: 각 i마다 1, 2
        print(i, j)
# 1 1
# 1 2
# 2 1
# 2 2
```

실생활 비유: 옷장을 정리할 때 "각 서랍(바깥 반복)마다, 그 안의 칸(안쪽 반복)을 하나씩 확인한다"는 이중 확인 과정과 같다. 서랍 하나를 다 확인해야 다음 서랍으로 넘어간다.

**실무에서는 이렇게 쓴다**: 표(행과 열) 형태의 데이터를 다룰 때(`for row in rows: for cell in row:`), 두 리스트의 모든 조합을 확인해야 할 때 쓰인다.

## while문 — 조건이 참인 동안 반복

**정의**: 정해진 횟수가 아니라, 조건이 참인 동안 계속 반복하는 문법이다.

실생활 비유: "손님이 없을 때까지 계속 청소한다"처럼, 몇 번 할지 미리 정해두지 않고 특정 상태가 끝날 때까지 반복하는 것이다.

```python
count = 0

while count < 3:        # count가 3보다 작은 동안 계속 반복
    print(count)
    count += 1           # count = count + 1과 같음 (반복마다 1씩 증가)
```

> 주의: `count += 1`을 깜빡하면 조건이 영원히 참이 되어 **무한 루프(Infinite Loop)**에 빠진다. while문을 쓸 때는 "언젠가 조건이 거짓이 될 수 있는가"를 항상 확인해야 한다.

JS와 비교하면:

| Python | JavaScript |
|---|---|
| `while count < 3:` | `while (count < 3) { }` |
| `count += 1` | `count += 1` — 동일 |

while문 자체의 개념과 문법 구조는 Python과 JS가 거의 동일하다. 괄호·중괄호 유무만 다르다.

**실무에서는 이렇게 쓴다**: 사용자가 올바른 값을 입력할 때까지 계속 물어보는 로직, 서버가 응답할 때까지 재시도하는 로직처럼 몇 번 반복할지 미리 알 수 없는 상황에 쓰인다.

**`while True:` + `break` 패턴**: 조건을 아예 "항상 참"으로 만들어놓고, 원하는 시점에 `break`로 직접 빠져나오는 방식도 실무에서 아주 자주 쓴다.

```python
while True:              # 조건이 항상 True라 겉보기엔 무한 반복
    answer = input("종료하려면 q를 입력하세요: ")
    if answer == "q":
        break             # 조건을 만족하면 여기서 직접 빠져나감
```

실생활 비유: "정지 신호가 올 때까지 계속 걷는다"는 지시와 같다. 몇 걸음을 걸을지 미리 정하지 않고, 멈추라는 신호(break 조건)가 올 때까지 계속 진행한다.

## break와 continue

**break**: 반복문을 그 자리에서 즉시 멈추고 빠져나온다.

**continue**: 이번 반복만 건너뛰고, 다음 반복으로 넘어간다.

```python
for i in range(10):
    if i == 3:
        break            # i가 3이 되는 순간 반복문 전체를 종료
    print(i)             # 0, 1, 2만 출력됨

for i in range(5):
    if i == 2:
        continue         # i가 2일 때만 건너뛰고
    print(i)             # 0, 1, 3, 4 출력됨 (2만 빠짐)
```

실생활 비유: `break`는 찾던 물건을 발견하면 그 자리에서 수색을 완전히 멈추는 것, `continue`는 이번 서랍만 건너뛰고 다음 서랍을 계속 확인하는 것과 같다.

**실무에서는 이렇게 쓴다**: 리스트에서 원하는 항목을 찾으면 더 이상 검색할 필요가 없을 때 `break`로 멈추고(불필요한 반복 방지), 특정 조건(예: 삭제된 게시글)만 건너뛰고 나머지는 계속 처리할 때 `continue`를 쓴다.

```python
# 실무에서 자주 보는 형태 (조건에 맞는 첫 번째 사용자를 찾으면 즉시 종료)
users = ["철수", "영희", "혜진", "민수"]

for user in users:
    if user == "혜진":
        print(f"{user}님을 찾았습니다")
        break             # 찾은 순간 나머지는 확인할 필요 없으니 종료
```

## for-else / while-else (Python만의 독특한 문법)

**정의**: 반복문에도 `else`를 붙일 수 있는데, 이 `else`는 **반복문이 `break`로 중간에 끊기지 않고 끝까지 다 돌았을 때만** 실행된다.

JS에는 이런 문법 자체가 없어서, Python을 처음 접하면 반드시 헷갈리는 부분이다.

```python
numbers = [1, 3, 5, 7]

for n in numbers:
    if n % 2 == 0:
        print("짝수 발견!")
        break
else:
    print("짝수가 없습니다")   # break 없이 반복문이 끝까지 돌았으므로 실행됨
```

실생활 비유: 서랍을 처음부터 끝까지 다 뒤졌는데도 찾는 물건이 안 나왔을 때만 "없다"고 말하는 것과 같다. 중간에 물건을 찾아서(`break`) 수색을 멈췄다면 "없다"는 말은 하지 않는다.

> 함정: 이름은 `else`지만 "조건이 거짓일 때"가 아니라 "`break`를 안 만나고 끝까지 돌았을 때" 실행된다는 뜻이다. if문의 else와 이름은 같지만 의미가 완전히 다르니 헷갈리지 않아야 한다.

**실무에서는 이렇게 쓴다**: 리스트 안에서 특정 값을 찾다가 못 찾았을 때만 "존재하지 않습니다" 같은 안내를 보여주고 싶을 때 쓴다. 다만 코드 가독성이 떨어진다는 이유로, 실무에서는 이 문법 대신 별도의 `found` 플래그 변수를 두는 방식을 더 선호하기도 한다.

## 요약

- 반복문은 같은 동작을 여러 번 실행하게 해주며, `for`(정해진 값들을 순회)와 `while`(조건이 참인 동안 반복) 두 종류가 있다
- **이터레이션(Iteration)**: 반복문 몸체가 한 번 실행되는 단위. "5번 반복"은 "이터레이션 5번"과 같은 말이며, 디버깅 시 "몇 번째 이터레이션에서 문제가 생겼는지" 콕 집어 말할 때 쓰는 표준 용어다
- `for fruit in fruits:`처럼 리스트를 순회하거나, `range(5)`로 횟수를 지정해서 반복한다. `range(5)`는 0~4까지 5개를 만든다는 점 주의
- `range(시작, 끝, 증가폭)`처럼 세 인자를 지정하면 더 세밀하게 반복 범위를 조절할 수 있다. 문자열도 한 글자씩 순회할 수 있다
- `enumerate()`로 순번과 값을 동시에 꺼낼 수 있어, 직접 순번 변수를 관리하지 않아도 된다
- 중첩 반복문(반복문 안의 반복문)으로 표 형태 데이터나 두 리스트의 모든 조합을 다룰 수 있다
- JS의 `for...of`와 Python의 `for...in`은 거의 동일하지만, JS의 전통적 3단계 for문은 Python에 없고 `range()`가 대신한다
- 함정: while문에서 조건을 거짓으로 만드는 코드(`count += 1` 등)를 빠뜨리면 무한 루프에 빠진다
- `while True:` + `break` 패턴으로 "멈출 조건이 생길 때까지 반복"하는 로직을 표현할 수 있다
- `break`는 반복문을 완전히 종료, `continue`는 이번 반복만 건너뛰고 계속 진행
- 함정: `for-else`/`while-else`는 JS에 없는 Python만의 문법으로, "break 없이 끝까지 돌았을 때"만 else가 실행된다. if문의 else와 이름은 같지만 의미가 다르다
- 실무에서는 DB 데이터 순회, 재시도 로직, 검색 후 즉시 종료, 페이지네이션 같은 곳에서 반복문이 핵심적으로 쓰인다
