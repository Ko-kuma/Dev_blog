## 연산자란?

연산자는 값과 값 사이에서 계산이나 비교를 해주는 기호다.

실생활로 치면 저울과 비슷하다. 저울에 두 물건을 올리면 "어느 쪽이 더 무거운가"를 알려주듯이, 연산자는 두 값을 놓고 "더하면 얼마인가", "둘이 같은가", "둘 다 참인가" 같은 결과를 알려준다.

Python의 연산자는 크게 네 종류로 나뉜다: 산술 연산자, 비교 연산자, 논리 연산자, 멤버십 연산자.

## 산술 연산자 (Arithmetic Operators)

숫자를 계산할 때 쓰는 연산자다. 슈퍼마켓에서 물건값을 계산기로 두드리는 것과 같다.

| 연산자 | 의미 | 예시 | 결과 |
|---|---|---|---|
| + | 더하기 | 3 + 2 | 5 |
| - | 빼기 | 3 - 2 | 1 |
| * | 곱하기 | 3 * 2 | 6 |
| / | 나누기 (항상 float) | 7 / 2 | 3.5 |
| // | 정수 나누기 (몫만) | 7 // 2 | 3 |
| % | 나머지 | 7 % 2 | 1 |
| ** | 거듭제곱 | 2 ** 3 | 8 |
어
이 중 `/`, `//`, `%`, `**`는 자료형 공부할 때 이미 다뤘던 내용이다. 

오늘은 여기에 `+`, `-`, `*`를 더해서 산술 연산자 전체를 완성한다.

```python
price = 1000
quantity = 3
total = price * quantity   # 3000원
print(total)
```

**실무에서는 이렇게 쓴다**: 장바구니 총액 계산(`가격 * 수량`), 할인율 적용(`가격 * 0.9`), 페이지네이션에서 전체 페이지 수 구하기(`전체_개수 // 페이지당_개수`) 같은 곳에 흔히 쓰인다.

JS와 비교하면:

| 연산자 | JavaScript | Python |
|---|---|---|
| 정수 나누기(몫만) | 없음 — `Math.floor(a / b)`로 직접 계산 | `//` 연산자로 바로 계산 |
| 거듭제곱 | `**` (ES2016부터 지원) | `**` — 동일 |

> 함정: JS에는 Python의 `//`(정수 나누기) 같은 전용 연산자가 없다. `Math.floor()`를 따로 호출해야 해서, Python이 이 부분만큼은 조금 더 간결하다.

## 비교 연산자 (Comparison Operators)

두 값을 비교해서 참(True) 또는 거짓(False)으로 답해주는 연산자다. 저울에 두 물건을 올려서 "왼쪽이 더 무겁나요?"라고 묻는 것과 같다 — 답은 항상 "네" 아니면 "아니오"다.

| 연산자 | 의미 | 예시 | 결과 |
|---|---|---|---|
| == | 같다 | 3 == 3 | True |
| != | 다르다 | 3 != 2 | True |
| > | 크다 | 3 > 2 | True |
| < | 작다 | 3 < 2 | False |
| >= | 크거나 같다 | 3 >= 3 | True |
| <= | 작거나 같다 | 3 <= 2 | False |

```python
age = 20
print(age >= 18)   # True
```

> 주의: `=`는 "대입"(변수에 값을 넣기)이고, `==`는 "비교"(두 값이 같은지 확인)다. 초보자들이 자주 헷갈리는 부분이다. `age = 20`은 age에 20을 저장하라는 뜻이고, `age == 20`은 age가 20과 같은지 물어보는 것이다.

JS와 비교하면:

| 상황 | JavaScript | Python |
|---|---|---|
| 값 비교 | `==` (타입 변환 후 비교), `===` (타입까지 비교) | `==`만 존재 (Python은 값과 타입을 함께 비교) |

Python의 `==`는 JS의 `===`처럼 동작한다고 생각하면 된다. Python에는 JS의 `==`처럼 타입을 몰래 바꿔서 비교하는 느슨한 비교 연산자가 없다.

```python
print(1 == "1")   # False (Python은 타입이 다르면 애초에 다른 값으로 봄)
```

**실무에서는 이렇게 쓴다**: 로그인 시 비밀번호 일치 확인(`입력값 == 저장된값`), 재고 확인(`재고 > 0`), 나이 제한 검사(`age >= 19`) 등에 쓰인다.

## 논리 연산자 (Logical Operators)

여러 개의 참/거짓 값을 조합해서 하나의 참/거짓 결과로 만들어주는 연산자다. "이 조건도 맞고, 저 조건도 맞아야 통과"처럼 여러 조건을 한꺼번에 확인할 때 쓴다.

실생활 비유: 놀이공원 놀이기구를 탈 때 "키가 120cm 이상**이고** 몸무게가 100kg 이하**여야** 탈 수 있음" 같은 안내문과 같다. 두 조건을 동시에 만족해야 통과된다.

| 연산자 | 의미 | 예시 |
|---|---|---|
| and | 둘 다 참이어야 참 | height >= 120 and weight <= 100 |
| or | 둘 중 하나만 참이어도 참 | is_admin or is_owner |
| not | 참/거짓을 뒤집음 | not is_deleted |

```python
height = 130
weight = 45
can_ride = height >= 120 and weight <= 100
print(can_ride)   # True
```

JS와 비교하면:

| Python | JavaScript |
|---|---|
| and | && |
| or | \|\| |
| not | ! |

Python은 영어 단어(and, or, not)를 그대로 쓰고, JS는 기호(&&, ||, !)를 쓴다는 점이 가장 큰 차이다.

**실무에서는 이렇게 쓴다**: 회원가입 시 유효성 검사(`나이 >= 19 and 이메일_형식_맞음`), 권한 체크(`is_admin or is_owner`일 때만 삭제 버튼 보여주기) 등에서 자주 쓰인다.

## 멤버십 연산자 (Membership Operators)

어떤 값이 리스트나 문자열 같은 데이터 묶음 안에 들어있는지 확인하는 연산자다. (리스트는 내일 배울 자료구조지만, 오늘은 개념만 살짝 맛본다.)

실생활 비유: 명단을 보고 "이 사람 이름이 여기 있나?"를 확인하는 것과 같다.

| 연산자 | 의미 | 예시 |
|---|---|---|
| in | 포함되어 있으면 참 | "a" in "apple" |
| not in | 포함되어 있지 않으면 참 | "z" not in "apple" |

```python
fruits = ["사과", "바나나", "포도"] #리스트는 아직 배우지 않았으므로 리스트가 있고 인덱스가 0부터 시작이라는 것만 기억 추후 배울 내용 
print("사과" in fruits)       # True
print("수박" not in fruits)   # True
```

**실무에서는 이렇게 쓴다**: 화이트리스트 검사(`허용된_이메일_도메인 in 도메인_목록`), 금지어 필터링(`금지단어 not in 게시글_내용`) 등에 쓰인다.

JS와 비교하면:

| Python | JavaScript |
|---|---|
| `"사과" in fruits` (리스트 값 안에 있는지) | `fruits.includes("사과")` |
| `in` | 배열에는 없음 (객체의 key 확인 용도로만 `in` 사용) |

> 함정: JS에도 `in` 연산자가 있지만 의미가 다르다. JS의 `in`은 배열의 "값"이 아니라 객체의 "키(key)"가 있는지 확인할 때 쓴다. 배열 안에 특정 값이 있는지 확인하려면 `includes()`를 써야 한다. Python처럼 `in`으로 값을 바로 검사할 수 있다고 착각하면 안 된다.

## 조건문 (if / elif / else)

조건문은 "만약 이렇다면 이렇게 하고, 아니라면 저렇게 해라"라고 프로그램에게 알려주는 문법이다.

실생활 비유: 신호등과 같다. "초록불이면 건너고, 빨간불이면 멈춘다"처럼, 상황(조건)에 따라 다른 행동을 하도록 미리 정해두는 것이다.

```python
age = 17

if age >= 19:
    print("성인입니다")       # 조건이 참일 때 실행
else:
    print("미성년자입니다")   # 조건이 거짓일 때 실행
```

조건이 여러 개일 때는 `elif`(else if의 줄임말)를 추가로 쓴다.

```python
score = 85

if score >= 90:
    print("A등급")
elif score >= 80:
    print("B등급")      # 여기 해당됨
elif score >= 70:
    print("C등급")
else:
    print("F등급")
```

Python은 위에서부터 순서대로 조건을 확인하다가, 처음으로 참이 되는 조건 하나만 실행하고 나머지는 건너뛴다. `score = 85`는 `score >= 90`은 거짓이라 넘어가고, `score >= 80`이 참이 되는 순간 "B등급"만 출력하고 조건문 전체를 빠져나간다.

JS와 비교하면:

| 구분 | JavaScript | Python |
|---|---|---|
| 조건 감싸기 | `if (조건) { }` — 괄호와 중괄호 필요 | `if 조건:` — 괄호 없이 콜론만 |
| else if | `else if (조건) { }` | `elif 조건:` |
| 블록 구분 | 중괄호 `{ }` | 들여쓰기 (Day1에서 배운 개념) |

Python은 괄호(`()`)와 중괄호(`{}`) 없이 콜론(`:`)과 들여쓰기만으로 조건문을 표현한다는 게 JS와 가장 큰 차이다.

**실무에서는 이렇게 쓴다**: 회원 등급별 할인율 적용, 로그인 상태에 따라 다른 화면 보여주기, 입력값 검증 후 에러 메시지 다르게 보여주기 등 거의 모든 로직에서 조건문이 기본 뼈대로 쓰인다.

```python
# 실무에서 자주 보는 형태 (회원 등급 할인)
grade = "VIP"

if grade == "VIP":
    discount = 0.2
elif grade == "GOLD":
    discount = 0.1
else:
    discount = 0
```

## 조건문 더 알아보기

앞서 배운 if/elif/else가 조건문의 기본이라면, 아래 세 가지는 실무에서 자주 마주치는 확장편이다.

### 중첩 조건문 (Nested if)

조건문 안에 또 다른 조건문을 넣을 수 있다. "1차 조건을 통과한 사람만, 2차 조건도 확인한다"는 흐름이다.

실생활 비유: 놀이공원 입장 심사와 같다. 먼저 "티켓이 있는가"를 확인하고, 티켓이 있는 사람 중에서만 다시 "키가 120cm 이상인가"를 확인하는 것처럼, 조건을 단계별로 나눠서 검사한다.

```python
has_ticket = True
height = 130

if has_ticket:
    if height >= 120:
        print("입장 가능")       # 여기 해당됨
    else:
        print("보호자 동반 필요")
else:
    print("티켓을 구매하세요")
```

**실무에서는 이렇게 쓴다**: 로그인 여부를 먼저 확인하고, 로그인된 사용자 중에서만 관리자 권한을 다시 확인하는 식으로 "여러 단계의 검증"을 표현할 때 자주 쓴다. 다만 중첩이 3~4단계 이상 깊어지면 코드가 읽기 어려워지므로, 그럴 땐 `and`로 조건을 합치거나 함수로 쪼개는 것이 더 낫다.

```python
# 중첩 대신 and로 합쳐서 더 간결하게 쓸 수도 있음
if has_ticket and height >= 120:
    print("입장 가능")
```

### 조건부 표현식 (삼항 연산자)

if/else를 한 줄로 줄여서, 조건에 따라 다른 값을 바로 변수에 담을 때 쓰는 문법이다.

실생활 비유: 자판기에서 "학생 카드를 대면 할인 가격, 아니면 정상 가격"이 즉시 하나로 정해지는 것과 같다. 두 단계를 거치지 않고, 조건을 확인하는 즉시 최종 가격표 하나만 딱 나오는 것이다.

```python
age = 20

# 기존 방식 (4줄)
if age >= 19:
    status = "성인"
else:
    status = "미성년자"

# 조건부 표현식 (1줄, 결과는 완전히 동일)
status = "성인" if age >= 19 else "미성년자"
```

읽는 순서가 독특한데, "조건이 참일 때의 값 → if 조건 → else 조건이 거짓일 때의 값" 순서로 왼쪽에서 오른쪽으로 읽으면 된다.

JS와 비교하면:

| Python | JavaScript |
|---|---|
| `"성인" if age >= 19 else "미성년자"` | `age >= 19 ? "성인" : "미성년자"` |

JS는 `조건 ? 참일때값 : 거짓일때값` 순서로 물음표(`?`)와 콜론(`:`)을 쓰는데, Python은 기호 대신 `if`/`else` 단어를 그대로 쓰되 순서가 앞뒤로 바뀐다는 점이 다르다.

**실무에서는 이렇게 쓴다**: 짧은 조건 하나로 값을 정할 때 코드를 간결하게 만들어준다. 예를 들어 `discount = 0.1 if is_member else 0`처럼 회원 여부에 따라 할인율을 한 줄로 정하는 식이다. 단, 조건이 복잡해지면 오히려 가독성이 떨어지므로 간단한 경우에만 쓰는 게 좋다.

### Truthy와 Falsy

Python은 True/False가 아닌 값도 조건문 안에서는 자동으로 참/거짓 취급을 한다. 이렇게 "참으로 취급되는 값"을 **Truthy**, "거짓으로 취급되는 값"을 **Falsy**라고 부른다.

**Falsy로 취급되는 대표값**: `0`, `0.0`, `""`(빈 문자열), `None`, `[]`(빈 리스트), `{}`(빈 딕셔너리)

**그 외 대부분의 값**은 Truthy로 취급된다 (`1`, `"안녕"`, `[1, 2]` 등).

```python
name = ""

if name:
    print(f"이름: {name}")
else:
    print("이름이 비어있습니다")   # 빈 문자열은 Falsy라서 여기 실행됨
```

실생활 비유: 창고에 물건이 하나도 없으면(빈 상태) "없다"고 자동으로 인식하고, 뭐라도 하나 있으면 "있다"고 인식하는 것과 같다. 굳이 "물건이 0개인가요?"라고 정확히 안 물어봐도, 창고가 비었는지 아닌지만으로 자연스럽게 판단이 되는 것이다.

**실무에서는 이렇게 쓴다**: `if user_list:` 한 줄로 "리스트가 비어있지 않으면"을 표현할 수 있어서, `if len(user_list) > 0:`이라고 길게 쓰는 것보다 짧고 Python다운 코드가 된다. API 응답값이 비어있는지 확인할 때(`if response_data:`) 아주 흔하게 쓰는 패턴이다.

JS와 비교하면:

| 값 | Python | JavaScript |
|---|---|---|
| `0` | Falsy | Falsy — 동일 |
| `""` (빈 문자열) | Falsy | Falsy — 동일 |
| `[]` (빈 배열/리스트) | **Falsy** | **Truthy** — 다름! |
| `{}` (빈 객체/딕셔너리) | **Falsy** | **Truthy** — 다름! |

> 함정: 가장 중요한 차이. Python에서는 빈 리스트(`[]`)와 빈 딕셔너리(`{}`)가 Falsy(거짓 취급)지만, JS에서는 빈 배열(`[]`)과 빈 객체(`{}`)가 오히려 **Truthy(참 취급)**다. JS에서 넘어온 습관대로 `if ([])`를 "비어있으니까 거짓이겠지"라고 생각하면 틀린다. Python과 JS를 넘나들며 코드를 짤 때 가장 흔히 걸리는 실수 중 하나다.

### 한 줄 if문 (조건부 표현식과는 다른 것)

앞서 배운 조건부 표현식(삼항 연산자)은 **값을 만들어서 변수에 저장**하는 것이었다. 반면 지금 소개할 한 줄 if문은 **조건에 따라 코드를 그냥 실행만** 하는 것이다. 둘은 생김새가 비슷해 보여도 목적이 다르다.

```python
age = 20

# 한 줄 if문: 값을 만드는 게 아니라 그냥 실행만 함
if age >= 19: print("성인입니다")

# 지금까지 쓰던 여러 줄 방식과 완전히 같은 동작
if age >= 19:
    print("성인입니다")
```

| 구분 | 조건부 표현식(삼항 연산자) | 한 줄 if문 |
|---|---|---|
| 하는 일 | 값을 만들어서 변수에 저장 | 코드를 그냥 실행만 함 |
| 예시 | `status = "성인" if age >= 19 else "미성년"` | `if age >= 19: print("성인")` |
| else 없이 쓰기 | 불가능 (반드시 else 필요) | 가능 (else 생략 가능) |

> 주의: 한 줄 if문은 문법적으로 가능하지만, PEP 8은 이 스타일을 권장하지 않는다. 아주 단순한 조건 하나짜리가 아니라면, 지금까지 배운 여러 줄 방식이 훨씬 읽기 쉽다. 실무 코드 리뷰에서도 "가독성을 해친다"는 이유로 자주 지적받는 스타일이다.

## 예시

```python
# operators_conditions.py
age = 22
has_ticket = True

if age >= 19 and has_ticket:
    print("입장 가능합니다")
elif age >= 19 and not has_ticket:
    print("티켓을 구매해주세요")
else:
    print("미성년자는 입장할 수 없습니다")
```

## 요약

- 연산자는 값 사이의 계산·비교를 담당하며, 산술(+,-,*,/,//,%,**)·비교(==,!=,>,<,>=,<=)·논리(and,or,not)·멤버십(in, not in) 네 종류가 있다
- 함정: JS에는 `//`(정수 나누기) 전용 연산자가 없어서 `Math.floor(a/b)`로 계산해야 함
- `=`는 대입(값을 저장), `==`는 비교(같은지 확인). Python의 `==`는 JS의 `===`처럼 타입까지 함께 비교한다
- 논리 연산자는 Python이 and/or/not 영어 단어를, JS는 &&/||/! 기호를 쓴다
- 함정: JS의 `in`은 배열 값이 아니라 객체의 키를 확인하는 연산자다. 배열 값 확인은 `includes()`를 써야 함
- 조건문은 신호등처럼 상황에 따라 다른 코드를 실행하게 해준다. `if` → `elif` → `else` 순서로 위에서부터 확인하며, 처음 참이 되는 조건 하나만 실행된다
- Python은 괄호·중괄호 없이 콜론과 들여쓰기만으로 조건문을 표현한다
- 중첩 조건문(if 안의 if)으로 단계별 검증을 표현할 수 있지만, 너무 깊어지면 and로 합치거나 함수로 분리하는 게 낫다
- 조건부 표현식(`값1 if 조건 else 값2`)으로 짧은 if/else를 한 줄로 줄일 수 있다. JS의 삼항 연산자(`조건 ? 값1 : 값2`)와 순서가 다르다
- 한 줄 if문(`if 조건: 실행문`)은 조건부 표현식과 달리 값을 만들지 않고 코드를 그냥 실행만 한다. 문법적으로 가능하지만 PEP 8은 권장하지 않음
- Truthy/Falsy: `0`, `""`, `None`, `[]`, `{}`는 자동으로 거짓 취급되고 나머지는 참 취급됨
- **가장 중요한 함정**: Python은 빈 리스트(`[]`)·빈 딕셔너리(`{}`)가 Falsy지만, JS는 빈 배열·빈 객체가 오히려 Truthy다. 두 언어를 오갈 때 가장 흔히 실수하는 부분
- 실무에서는 회원 등급별 로직, 권한 체크, 입력값 검증 등 거의 모든 곳에서 연산자와 조건문이 기본 뼈대로 쓰인다
