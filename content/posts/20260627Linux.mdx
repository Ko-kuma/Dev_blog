## Linux?

### 운영체제(OS)란?

컴퓨터를 켜면 가장 먼저 실행되는 소프트웨어입니다.
하드웨어(CPU, 메모리, 저장장치)와 우리가 쓰는 프로그램 사이를 중간에서 연결해줍니다.
Windows, macOS, Linux 전부 운영체제입니다.

### Linux란?

1991년 핀란드 대학생 리누스 토르발스(Linus Torvalds)가 만든 **오픈소스 운영체제**입니다.
누구나 소스코드를 보고, 수정하고, 배포할 수 있어요.

| | Windows | macOS | Linux |
|---|---|---|---|
| 만든 곳 | Microsoft | Apple | 오픈소스 커뮤니티 |
| 비용 | 유료 | 무료 (Mac 구매 시) | 무료 |
| 주로 쓰는 곳 | 개인 PC | 개인 PC, 개발자 | 서버, 클라우드, 개발 환경 |
| 점유율 (서버) | 낮음 | 낮음 | **90% 이상** |

### 왜 개발자는 Linux를 써야 하나?

인터넷에 있는 서버의 90% 이상이 Linux로 돌아갑니다.
AWS, Google Cloud, Docker 컨테이너 전부 Linux 기반이 이고
즉, **내가 만든 코드가 실제로 돌아가는 환경이 Linux** 입니다.

Mac은 Linux와 같은 계열(Unix 기반)이라 터미널 명령어가 거의 동일하며
Windows는 Git Bash나 WSL(Windows Subsystem for Linux)을 쓰면 Linux 명령어를 그대로 쓸 수 있습니다.

### 커널(Kernel)이란?

Linux의 핵심 부품입니다.
운영체제에서 하드웨어를 직접 제어하는 가장 낮은 층의 소프트웨어이며,
터미널에서 명령어를 치면 → 쉘(Shell)이 해석 → 커널이 실행 → 하드웨어가 동작하는 구조입니다.

```
사용자 입력
    ↓
쉘 (Shell) — 명령어를 해석하는 프로그램 (bash, zsh 등)
    ↓
커널 (Kernel) — 하드웨어를 직접 제어
    ↓
하드웨어 (CPU, 메모리, 디스크)
```

### 쉘(Shell)이란?

사용자가 입력한 명령어를 커널이 이해할 수 있도록 해석해주는 프로그램입니다.
터미널을 열면 실행되는 게 쉘입니다.

| 쉘 종류 | 설명 |
|---|---|
| bash | Linux 기본 쉘. 가장 널리 쓰임 |
| zsh | macOS 기본 쉘 (Catalina 이후). bash 확장판 |
| fish | 자동완성이 강력한 현대적 쉘 |

현재 쓰는 쉘 확인:
```bash
echo $SHELL
# 출력 예시: /bin/zsh 또는 /bin/bash
```

## Linux 명령어를 왜 배우는가?

서버, Docker, 터미널 환경은 전부 Linux 기반입니다.
마우스로 클릭하는 GUI 대신 **텍스트 명령어로 컴퓨터를 조작**하는 게 CLI(Command Line Interface)입니다.

처음엔 낯설지만 익숙해지면 마우스보다 훨씬 빠릅니다.
지금 배울 명령어 7개만 알아도 대부분의 터미널 작업을 할 수 있습니다.

---

## 터미널 열기

- **Mac**: Terminal 앱 또는 iTerm2
- **Windows**: Git Bash, PowerShell, 또는 WSL (Windows Subsystem for Linux)
- **Linux**: Ctrl + Alt + T

## 핵심 명령어 7개

### 1. `pwd` — 현재 위치 확인

**P**rint **W**orking **D**irectory의 약자입니다.
"나 지금 어디 있어?"를 물어보는 명령어예요.

```bash
pwd
# 출력 예시: /Users/사용자/Documents
```

> 터미널을 열면 항상 어딘가에 위치해 있습니다.
> 길을 잃었다 싶으면 `pwd`부터 치는 게 습관이 됩니다.

---

### 2. `ls` — 현재 폴더 내용 보기

**L**i**s**t의 약자입니다.
현재 폴더 안에 뭐가 있는지 보여줍니다.

```bash
# 기본 사용법
ls

# 숨김 파일까지 전부 보기 (. 으로 시작하는 파일들)
ls -a

# 파일 크기, 권한, 날짜까지 자세하게 보기
ls -l

# 둘 다 합치기 (가장 자주 씀)
ls -al
```

> `-a`, `-l` 같은 것을 **옵션(flag)** 이라고 합니다.
> 명령어의 동작 방식을 조정하는 스위치입니다.

---

### 3. `cd` — 폴더 이동

**C**hange **D**irectory의 약자입니다.
다른 폴더로 이동할 때 씁니다.

```bash
# 특정 폴더로 이동
cd Documents

# 한 단계 위 폴더로 이동 (상위 폴더)
cd ..

# 두 단계 위로 이동
cd ../..

# 홈 디렉토리로 바로 이동 (어디서든 집으로)
cd ~

# 루트 디렉토리로 이동 (최상위)
cd /

# 절대 경로로 이동
cd /Users/사용자/Documents/project
```

> **절대 경로 vs 상대 경로**
> - 절대 경로: `/Users/heejin/Documents` — 루트(/)부터 시작하는 전체 주소
> - 상대 경로: `Documents` — 현재 위치 기준의 주소
> 집 주소로 비유하면 절대 경로는 "서울시 강남구 테헤란로 123", 상대 경로는 "여기서 두 블록 앞"

---

### 4. `mkdir` — 폴더 만들기

**M**a**k**e **Dir**ectory의 약자입니다.

```bash
# 폴더 만들기
mkdir my-project

# 여러 폴더 한 번에 만들기
mkdir frontend backend database

# 중간 폴더가 없어도 한 번에 만들기 (-p 옵션)
mkdir -p projects/2026/june
```

---

### 5. `touch` — 파일 만들기

빈 파일을 생성합니다.
파일이 이미 있으면 수정 날짜만 업데이트해요.

```bash
# 빈 파일 만들기
touch README.md

# 여러 파일 한 번에 만들기
touch index.html style.css script.js

# .gitignore 파일 만들기
touch .gitignore
```

> 터미널에서 파일을 만들 땐 `touch`, 폴더를 만들 땐 `mkdir`을 씁니다.

---

### 6. `cat` — 파일 내용 보기

Con**cat**enate의 약자입니다.
파일 내용을 터미널에 바로 출력해줘요.

```bash
# 파일 내용 출력
cat README.md

# 여러 파일 내용 이어서 출력
cat file1.txt file2.txt

# 줄 번호와 함께 출력
cat -n README.md
```

> 파일이 너무 길면 `cat` 대신 `less` 명령어를 씁니다.
> `less 파일명`으로 열고, 스크롤하다가 `q`를 누르면 나옵니다.

---

### 7. `rm` — 파일/폴더 삭제

**R**e**m**ove의 약자입니다.
**주의: 휴지통 없이 즉시 삭제됩니다. 복구 불가능해요.**

```bash
# 파일 삭제
rm 파일명.txt

# 여러 파일 삭제
rm file1.txt file2.txt

# 폴더 삭제 (-r: 폴더 안 내용까지 재귀적으로 삭제)
rm -r 폴더명

# 확인 없이 강제 삭제 (-f: force)
rm -rf 폴더명
```

> `rm -rf`는 복구 불가능한 강제 삭제입니다.
> 특히 `rm -rf /` (루트 전체 삭제)는 절대 입력하면 안 됩니다.
> 항상 `pwd`로 위치 확인 후 삭제하세요.

## 자주 쓰는 추가 명령어

### `cp` — 파일/폴더 복사

```bash
# 파일 복사
cp 원본파일.txt 복사본파일.txt

# 폴더 복사 (-r 옵션)
cp -r 원본폴더 복사본폴더
```

### `mv` — 파일/폴더 이동 또는 이름 변경

```bash
# 파일 이동
mv 파일명.txt 다른폴더/

# 파일 이름 변경 (이동 명령어를 이름 변경에도 활용)
mv 옛날이름.txt 새이름.txt
```

### `echo` — 텍스트 출력 또는 파일에 쓰기

```bash
# 터미널에 텍스트 출력
echo "Hello, World!"

# 파일에 텍스트 쓰기 (파일이 없으면 생성)
echo "# My Project" > README.md

# 파일에 텍스트 추가 (덮어쓰기 아닌 이어쓰기)
echo "새로운 줄" >> README.md
```

### `clear` — 터미널 화면 정리

```bash
clear
# 단축키: Ctrl + L
```

## 전체 흐름 따라하기

아래 순서대로 따라하면 폴더 생성부터 파일 작성까지 한 번에 연습가능

```bash
# 1. 현재 위치 확인
pwd

# 2. 홈 디렉토리로 이동
cd ~

# 3. 실습 폴더 만들기
mkdir linux-practice
cd linux-practice

# 4. 현재 위치 재확인
pwd

# 5. 파일 3개 만들기
touch README.md index.html style.css

# 6. 파일 목록 확인
ls -al

# 7. README.md에 내용 쓰기
echo "# Linux 실습" > README.md

# 8. README.md 내용 확인
cat README.md

# 9. 하위 폴더 만들기
mkdir assets

# 10. 구조 확인
ls -al

# 11. 상위 폴더로 이동
cd ..

# 12. 실습 폴더 삭제
rm -r linux-practice

# 13. 삭제 확인
ls
```

## 유용한 터미널 단축키

| 단축키 | 하는 일 |
|---|---|
| `Tab` | 명령어/파일명 자동완성 |
| `↑` / `↓` | 이전/다음 명령어 불러오기 |
| `Ctrl + C` | 실행 중인 명령어 강제 종료 |
| `Ctrl + L` | 화면 정리 (clear) |
| `Ctrl + A` | 커서를 줄 맨 앞으로 |
| `Ctrl + E` | 커서를 줄 맨 뒤로 |

> `Tab` 자동완성은 정말 많이 씁니다. 폴더명/파일명 두 글자 치고 Tab 누르는 게 습관이 됩니다.
