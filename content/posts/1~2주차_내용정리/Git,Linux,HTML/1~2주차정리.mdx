1~2주차 동안 개발환경 셋업, Git, Linux, HTML/CSS, JavaScript를 공부하면서 한 가지를 느꼈습니다.

요즘 에이전트 모델의 발전으로 프로그래밍의 문턱이 많이 낮아진 건 사실입니다. 비전공자, 비개발자도 AI의 도움을 받아 코드를 작성하고 서비스를 만들 수 있는 시대가 됐습니다.

그런데 공부를 하다 보니, 아무리 에이전트 모델이 코드를 짜주고 LLM이 오류를 찾아줘도 결국 본인이 코드가 어떤 방식으로 동작하는지, 어떤 값이 반환되는지 대략적으로라도 이해하고 있어야 한다는 걸 느꼈습니다.

AI가 작성한 코드를 그대로 붙여넣는 것과, 그 코드가 무엇을 하는지 읽으면서 붙여넣는 것은 완전히 다른 이야기입니다. AI에만 의존하다가 문제가 생기면 그때는 어떻게 해결할 것인가. 토큰 사용량이 초과돼서 그날 작업을 못 하게 된다면 어떻게 할 것인가. 결국 문제를 해결하고 답을 찾아가려면 AI를 의존의 대상이 아닌 협업자이자 도구로 활용할 줄 알아야 한다고 생각합니다.

그래서 블로그 정리를 시작했습니다. 완벽하게 외우기 위해서가 아니라, 필요할 때 언제든 꺼내볼 수 있는 나만의 레퍼런스를 만들기 위해서입니다. 기억하지 못해도 괜찮습니다. 기록해두면 언제든 찾아볼 수 있고, 다시 복습할 수 있고, 언제든 다시 시작할 수 있습니다. 자신만의 정리 공간이 있다는 것 자체가 중요하다고 생각합니다.
