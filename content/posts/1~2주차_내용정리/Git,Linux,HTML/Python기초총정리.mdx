## Python 소개 & 실행 방식

[Python 기초 - Python 소개 & 실행 방식](/posts/Python1)

- 저수준 언어(빠르지만 어려움)와 고수준 언어(쉽지만 느림), 컴파일 언어와 인터프리터 언어의 차이
- Python은 Guido van Rossum이 1991년 발표한 고수준 인터프리터 언어. 이름은 신화가 아니라 코미디 쇼 "Monty Python"에서 유래
- 특징: 동적 타이핑, 들여쓰기 기반 문법, 방대한 표준 라이브러리
- 들여쓰기는 Tab이 아니라 Space 4칸 (PEP 8 권장, 섞으면 TabError)
- 활용 분야: 웹 백엔드, 데이터 분석, AI, 자동화 등 폭넓게 쓰이며 로드맵은 웹 백엔드(FastAPI)에 집중

## 변수와 자료형

[Python 기초 - 변수와 자료형](/posts/Python2)

- 변수는 라벨 붙은 서랍. `이름 = 값`으로 선언하며 snake_case가 Python 변수/함수 명명 규칙
- 변수 이름 규칙: 숫자로 시작 불가, 예약어 사용 불가, 특수문자 불가, 대소문자 구분
- 기본 자료형: int, float, str, bool, NoneType
- 부동소수점 함정: 컴퓨터는 2진법으로 저장하기 때문에 `0.1 + 0.2`가 정확히 0.3이 아님. 금액 계산은 decimal 모듈 사용
- `type()`으로 자료형 확인, `int()`/`str()`/`float()`로 형변환

## 연산자와 조건문

[Python 기초 - 연산자와 조건문](/posts/Python3)

- 연산자 네 종류: 산술(`+,-,*,/,//,%,**`), 비교(`==,!=,>,<,>=,<=`), 논리(`and,or,not`), 멤버십(`in, not in`)
- `=`는 대입, `==`는 비교. Python의 `==`는 JS의 `===`처럼 타입까지 비교
- 조건문은 `if → elif → else` 순서로 확인하며 처음 참인 조건 하나만 실행
- 중첩 조건문, 조건부 표현식(삼항연산자), 한 줄 if문
- **가장 중요한 함정**: Python은 빈 리스트·딕셔너리가 Falsy지만, JS는 빈 배열·객체가 Truthy

## 반복문 (for, while)

[Python 기초 - 반복문 (for, while)](/posts/Python4)

- 이터레이션은 반복문이 한 번 도는 단위. `for`는 정해진 대상을 순회, `while`은 조건이 참인 동안 반복
- `range(시작, 끝, 증가폭)`, 문자열도 순회 가능, `enumerate()`로 순번+값 동시 추출
- 중첩 반복문, `while True:` + `break` 패턴
- `break`는 반복문 전체 종료, `continue`는 이번 반복만 건너뜀
- `for-else`/`while-else`는 JS에 없는 문법으로, break 없이 끝까지 돌았을 때만 실행

## 자료구조 (list, tuple)

[Python 기초 - 자료구조 (list, tuple)](/posts/Python5)

- 리스트(`[ ]`)는 값 변경이 자유로운 자료구조. 인덱스는 0부터 시작, 음수 인덱스는 뒤에서부터
- 슬라이싱(`[1:3]`)은 끝 인덱스 미포함. `len()`, `insert()/pop()/sort()/count()/index()` 등 메서드
- **가장 중요한 함정**: `list2 = list1`은 복사가 아니라 같은 리스트를 가리키는 참조. 진짜 복사는 `.copy()`
- 튜플(`( )`)은 불변 자료구조. 튜플 언패킹(`x, y = point`)으로 여러 변수에 나눠 받기
- 튜플도 `list()` → 수정 → `tuple()`로 우회해서 "새로 만드는" 방식으로 값을 바꿀 수 있음

## 자료구조 (dict, set)

[Python 기초 - 자료구조 (dict, set)](/posts/Python6)

- 딕셔너리는 key-value 쌍으로 값을 저장. 없는 key 접근 시 `KeyError`, `get()`으로 안전하게 접근
- `pop(key)`, `update()`, 중첩 딕셔너리로 API/DB 데이터와 동일한 구조 표현
- 딕셔너리 key와 셋의 원소는 불변 자료형만 가능 (해시 가능성 때문)
- 셋은 순서 없고 중복 불가. `&`(교집합), `|`(합집합), `-`(차집합), `^`(대칭차집합)
- **함정**: `{ }`는 콜론 유무로 딕셔너리/셋을 구분하며, 빈 `{ }`는 딕셔너리로 취급됨 (빈 셋은 `set()`)
- 변수 대신 딕셔너리를 쓰는 이유는 key를 실행 중에 동적으로 다룰 수 있기 때문

## 함수 (def, 매개변수, 리턴)

[Python 기초 - 함수 (def, 매개변수, 리턴)](/posts/Python7)

- 매개변수(정의 시 이름표)와 인자(호출 시 실제 값)는 다른 개념
- 기본값 매개변수, 위치 인자 vs 키워드 인자, `*args`(튜플로 수집)/`**kwargs`(딕셔너리로 수집)
- 호출할 때는 반대로 `*리스트`/`**딕셔너리`로 언패킹해서 전달 가능
- **함정**: `def func(items=[]):`처럼 가변 자료형을 기본값으로 쓰면 호출마다 값이 공유됨. `None` 패턴으로 해결
- `print()`는 출력만, `return`이 있어야 값을 진짜로 돌려줌. `return` 없으면 항상 `None`
- 지역변수는 함수 밖에서 접근 불가. `global`로 전역변수 수정 가능하지만 실무에서는 `return`을 선호
- 람다(`lambda x: 표현식`)는 이름 없는 한 줄 함수, `sort(key=...)`처럼 다른 함수 인자로 자주 쓰임

## 예외처리 (try, except)

[Python 기초 - 예외처리 (try, except)](/posts/20260709-python-exception-handling)

- `try` 블록에서 에러가 나면 즉시 `except`로 흐름이 넘어감. 종류를 지정하지 않은 `except:`는 지양
- 모든 예외는 `Exception` 아래 계층 구조를 가짐. `except` 순서는 구체적인 것부터 먼저 써야 함
- `except (A, B):`로 여러 예외 동시 처리, `as e`로 에러 메시지 확인
- `else`는 성공했을 때만, `finally`는 항상 실행
- `raise`로 직접 예외 발생, `Exception` 상속으로 커스텀 예외 생성
- `assert 조건, 메시지`로 가정 검증 (최적화 모드에서 무시될 수 있음)
- 트레이스백은 아래(결론)에서 위(호출 경로)로 읽는다

## 모듈, 패키지, 가상환경

[Python 기초 - 모듈, 패키지, 가상환경 (import, venv, pip)](/posts/Python9)

- 모듈은 파일 하나, 패키지는 모듈을 묶은 폴더(`__init__.py` 포함)
- 같은 모듈은 여러 번 import해도 코드가 한 번만 실행됨
- `if __name__ == "__main__":`으로 직접 실행과 import를 구분
- `import`(통째로) vs `from ... import`(콕 집어서) vs `as`(별칭)
- `pip`으로 PyPI 패키지 설치, `requirements.txt`로 의존성 목록 관리, `==`로 버전 고정
- 가상환경(venv)은 프로젝트별 독립 공간. 활성화 안 하면 컴퓨터 전체에 설치되는 함정 주의
- `venv`, `__pycache__`는 `.gitignore`에 등록하고 `requirements.txt`로만 환경을 재현

## 파일 입출력과 클래스 기초

[Python 기초 - 파일 입출력과 클래스 기초](/posts/Python10)

- `open()`/`close()`로 파일을 열고 닫으며, `with`문이 자동으로 닫아줘서 더 안전 (예외처리의 `finally`와 같은 원리)
- 파일 모드: `r`(읽기), `w`(덮어쓰기, 기존 내용 삭제), `a`(이어쓰기). `encoding="utf-8"`로 한글 깨짐 방지
- `read()`, `for line in file`, `readlines()`로 상황에 맞게 읽기. 없는 파일은 `FileNotFoundError`
- 클래스는 설계도(붕어빵 틀), 인스턴스는 그 결과물(붕어빵). `__init__`은 생성자, `self`는 인스턴스 자기 자신
- 속성(데이터)과 메서드(행동), `__str__`로 `print()` 결과를 사람이 읽기 쉽게 정의
- **함정**: 클래스 변수(공유됨)에 가변 자료형을 쓰면 함수 기본값 함정과 같은 문제가 생김. 인스턴스별 값은 `self.`로
- 상속으로 부모 클래스 기능을 자식이 물려받음. 예외처리의 `Exception` 상속이 실제 사례였음

## 전체 핵심 테마

파이썬의 기초를 돌아보면 반복해서 등장한 굵직한 테마 세 가지로 요약된다.

1. **가변(Mutable) vs 불변(Immutable)**: 리스트-튜플, 딕셔너리 key 제약, 함수 기본값 함정, 클래스 변수 공유 문제까지 전부 이 하나의 원리에서 파생됐다
2. **참조(Reference)와 공유**: `list2 = list1`, 클래스 변수 공유, 가변 기본값 함정 모두 "변수는 값이 아니라 값이 있는 곳을 가리킨다"는 원리로 연결된다
3. **Python다운 안전장치**: `with`문, `finally`, `get()`, `None` 기본값 패턴처럼, Python은 실수를 줄이기 위한 관용구가 곳곳에 마련되어 있다
